# Evidence Priority

Use this ranking whenever the sources disagree.

1. Exact git date-window history
2. Same-window merge history
3. `origin/main` coverage of the work
4. Branch divergence counts
5. Release tags
6. Dated changelog headings
7. Current-state docs
8. Plan/progress/worknote artifacts
9. Memory or prior summaries

## Notes

- Changelog and tags are strong release hints, but not enough by themselves.
- Plan docs are useful for naming and scoping, but they do not prove shipping.
- Memory is for consistency, not for overriding current evidence.
