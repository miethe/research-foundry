# MeatyWiki — Project Context

This project uses **MeatyWiki** as its knowledge compilation layer. The vault lives at:

```
{{VAULT_PATH}}
```

Project intent and scope are declared in `{{VAULT_PATH}}/wiki-spec.md`. Read that file before making decisions about what to ingest or compile.

## When to Drive MeatyWiki

Invoke the `meatywiki` skill whenever the user:

- Adds, references, or asks about a source (URL, PDF, note, transcript, export)
- Asks a question that should be answered from project knowledge ("what did we decide about X?")
- Asks to compile, classify, search, or synthesize over the vault
- Mentions lens scores (fidelity, freshness, clarity) or workflow runs
- Wants a decision-table evaluation against an artifact

Do NOT use MeatyWiki for: code search (use grep/symbols), git history, CI/CD ops, or anything outside the vault scope declared in `wiki-spec.md`.

## Core Commands (project-scoped)

All commands take `--vault {{VAULT_PATH}}` (or set `MEATYWIKI_VAULT_ROOT={{VAULT_PATH}}` once per shell):

```bash
meatywiki --vault {{VAULT_PATH}} ingest <source>            # add to raw/
meatywiki --vault {{VAULT_PATH}} compile --pending          # raw/ → wiki/
meatywiki --vault {{VAULT_PATH}} query "<question>"         # FTS-backed answer with citations
meatywiki --vault {{VAULT_PATH}} search "<keywords>"        # full-text search
meatywiki --vault {{VAULT_PATH}} doctor                     # health check
meatywiki --vault {{VAULT_PATH}} stats                      # counts + freshness
```

For the full command map, decision tree, and guardrails, the canonical reference is the globally-installed `meatywiki` skill at `.claude/skills/meatywiki/SKILL.md` (or wherever the skill is installed on this machine).

## Guardrails (project-local)

1. **Vault writes go through the engine.** Never `cat > {{VAULT_PATH}}/wiki/...` or edit `_meta/` files manually. Always use `meatywiki` commands.
2. **`_meta/` is gitignored and engine-owned.** Do not commit `_meta/meatywiki.db` or `_meta/compile_state.json`.
3. **`wiki-spec.md` is canonical for intent.** When the user asks "what's this vault for?", read `{{VAULT_PATH}}/wiki-spec.md`, not this file.
4. **Source URLs / PDFs go through `ingest`, not direct file copy.** This ensures classification, frontmatter, and index round-trip.
5. **Semantic / lens / workflow / decision commands require `PORTAL_DATABASE_URL`.** If unset, fall back to FTS5-only search and skip lens/workflow/decision surfaces.
