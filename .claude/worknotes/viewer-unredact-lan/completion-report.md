Ran the static data copy step via a node one-liner (same logic as `prebuild-static-data.mjs` minus the rf export), then ran `pnpm run build` (tsc + vite). The worktree having no `.venv` is a known constraint stated in the sprint instructions ("NO prior run.json artifacts").

## Memory Candidates Captured

- The worktree `.venv` path is not created by the worktree setup; `rf` must be on PATH globally (it is). The `prebuild-static-data.mjs` script hardcodes `.venv/bin/python` relative to repo root — any worktree without a `.venv` symlink will fail the prebuild step. Workaround: run the copy step manually or create a `.venv` symlink in the worktree.
- `ProvenanceModal` had 3 untracked render sites (ClaimAuditWorkbench, RunDetailModal, ReportOverlay) where `sensitivityThreshold` was not being passed. This was a latent re-masking bug even before this contract.
