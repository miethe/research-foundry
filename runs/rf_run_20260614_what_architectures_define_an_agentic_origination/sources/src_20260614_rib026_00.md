# Source Card: Building Effective AI Agents

## Summary
Anthropic's engineering guidance "Building Effective AI Agents" defines a vocabulary and set of composable patterns for agentic systems. It distinguishes workflows (LLMs and tools orchestrated through predefined code paths) from agents (LLMs that dynamically direct their own processes and tool usage), and establishes the "augmented LLM" — a model enhanced with retrieval, tools, and memory — as the foundational building block. It then catalogs concrete workflow patterns including routing and orchestrator-workers.

## Key evidence
- (ev_001) Workflows orchestrate LLMs/tools through predefined code paths; agents dynamically direct their own processes and tool usage.
- (ev_002) The foundational building block is the augmented LLM, enhanced with retrieval, tools, and memory.
- (ev_003) Routing classifies an input and directs it to a specialized followup task; classification can be done by an LLM or a traditional classification model/algorithm.
- (ev_004) Routing example: easy/common questions to cost-efficient models like Claude Haiku, hard/unusual ones to more capable models like Claude Sonnet, to optimize cost/performance.
- (ev_005) Orchestrator-workers: a central LLM dynamically breaks down tasks, delegates to worker LLMs, and synthesizes results; subtasks are determined at runtime, not pre-defined (the key difference from parallelization).

## Limitations
- Originally published 2024-12-19; foundational and stable for pattern definitions but should be treated as tertiary for freshness. The live page has been updated since original publication (e.g., model-name references such as Claude Haiku 4.5 / Sonnet 4.5 reflect a later revision than the 2024 publish date), so model-specific examples are time-sensitive even though the architectural pattern definitions are durable.
