# Source Card: aurelio-labs/semantic-router — README (official repo)

## Summary
The official README for aurelio-labs/semantic-router describes the library as a "superfast decision-making layer" that performs intent routing in embedding/semantic vector space rather than via LLM generation, thereby avoiding slow LLM round-trips for the routing decision itself. Routes are declared as named objects with example "utterances," and the system supports pluggable encoders, fully local execution, and "Dynamic Routes" that generate parameters and trigger function calls. As the maintainer's own README it is a primary description of intended capability rather than independent benchmark evidence.

## Key evidence
- (ev_001) Positions itself as a fast decision-making/routing layer in front of LLMs and agents.
- (ev_002) Uses semantic vector space (embeddings) instead of slow LLM generations to make routing/tool-use decisions.
- (ev_003) Routes are defined as objects with a name and example utterances matched by semantic similarity.
- (ev_004) Supports "Dynamic Routes" for parameter generation and function calls, not only static classification.
- (ev_005) Can run fully locally via HuggingFaceEncoder and LlamaCppLLM.
- (ev_006) Provides pluggable encoders (Cohere, OpenAI, HuggingFace, FastEmbed, multi-modal) and vector-DB integrations (Pinecone, Qdrant).
