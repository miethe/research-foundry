# Source Card: Create multi-agent collaboration - Amazon Bedrock User Guide

## Summary
This official AWS Bedrock User Guide page documents how to build a multi-agent collaboration team: deploy task-specific collaborator agents, designate a supervisor agent, associate collaborator aliases (max 10), then prepare, deploy, and invoke. The supervisor can either coordinate collaborator responses ("Supervisor") or route information to the appropriate collaborator to send the final response ("Supervisor with routing"), the latter reducing latency. Per-collaborator "Collaboration instructions" drive the supervisor's dispatch decision, and an optional conversation-history toggle shares full session context with a collaborator.

## Key evidence
- (ev_001) Supervisor can be configured to coordinate collaborator responses or to route information to the appropriate collaborator for the final response.
- (ev_002) Assigning the supervisor the routing task reduces latency.
- (ev_003) A supervisor can associate a maximum of 10 collaborator agents.
- (ev_004) Console modes are explicit: "Supervisor" (coordinate) vs "Supervisor with routing" (route to appropriate collaborator).
- (ev_005) "Collaboration instructions" specify when a given collaborator should be used by the supervisor.
- (ev_006) Optional "Enable conversation history" shares the full current-session history (user input and supervisor responses per turn) with a collaborator.
- (ev_007) Deployment is via setting up the supervisor agent to make an InvokeAgent request.

## Limitations
- Vendor-authored product documentation describing as-designed behavior, not independent benchmark or empirical evaluation; latency reduction is asserted qualitatively without quantified figures. The "maximum of 10 collaborator agents" is explicitly noted as a current ("at this time") limit subject to change.
