# Source Card: Agent orchestration — OpenAI Agents SDK

## Summary
This is the official OpenAI Agents SDK documentation page on agent orchestration. It distinguishes orchestrating via the LLM (the model autonomously plans, uses tools, and delegates via handoffs) from orchestrating via code (deterministic flow built from structured outputs, chaining, while loops, and parallel execution). It defines two composable SDK patterns: agents-as-tools (a manager agent calls specialists via `Agent.as_tool()` and owns the final answer) and handoffs (a triage agent routes to a specialist that becomes the active agent for the turn).

## Key evidence
- (ev_001) An SDK agent is an LLM with instructions, tools, and handoffs that can autonomously plan, act, and delegate to sub-agents.
- (ev_002) Orchestrating via code is more deterministic and predictable in speed, cost, and performance.
- (ev_003) Code-based orchestration uses structured outputs, chaining, while loops, and parallel execution (e.g., asyncio.gather).
- (ev_004) The manager pattern has a manager agent keep conversation control and call specialists via `Agent.as_tool()`.
- (ev_005) Handoffs route the conversation to a specialist that becomes the active agent for the rest of the turn.
- (ev_006) Agents-as-tools and handoffs are composable within a single workflow.
