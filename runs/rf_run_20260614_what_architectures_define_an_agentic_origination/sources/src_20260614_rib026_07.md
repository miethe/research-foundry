# Source Card: Microsoft Agent Framework Version 1.0

## Summary
This first-party Microsoft Dev Blogs post announces the version 1.0 general availability (April 3, 2026) of the Microsoft Agent Framework for .NET and Python, presented as a production-ready, long-term-supported open-source SDK that unifies Semantic Kernel's enterprise foundations with AutoGen's orchestration patterns. It establishes the framework's architecture: a graph-based workflow engine, five multi-agent orchestration patterns, durability features (checkpointing/hydration, human-in-the-loop, pause/resume), declarative YAML agent/workflow definitions, and first-party connectors to multiple model providers.

## Key evidence
- (ev_001) Microsoft Agent Framework reached v1.0 GA for .NET and Python on April 3, 2026 as a production-ready release with stable APIs and long-term support.
- (ev_002) The framework unifies Semantic Kernel's enterprise foundations with AutoGen's orchestrations into a single open-source SDK.
- (ev_003) Five orchestration patterns are supported: sequential, concurrent, handoff, group chat, and Magentic-One.
- (ev_004) A graph-based workflow engine composes agents and functions into deterministic, repeatable processes and is now stable.
- (ev_005) Checkpointing and hydration let long-running processes survive interruptions.
- (ev_006) First-party service connectors span Microsoft Foundry, Azure OpenAI, OpenAI, Anthropic Claude, Amazon Bedrock, Google Gemini, and Ollama.
- (ev_007) Agents and workflows can be defined in version-controlled YAML and loaded/run with a single API call.

## Limitations
- Vendor-authored product announcement: capability claims (e.g., "production-ready," "stable," durability guarantees) are promotional and not independently benchmarked; treat performance/reliability assertions as the vendor's framing rather than verified evidence.
