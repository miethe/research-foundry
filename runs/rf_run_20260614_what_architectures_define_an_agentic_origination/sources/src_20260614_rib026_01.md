# Source Card: Handoffs — OpenAI Agents SDK

## Summary
The official OpenAI Agents SDK "Handoffs" documentation establishes that a handoff is a mechanism for one agent to delegate a task to another specialized agent, surfaced to the LLM as a tool named `transfer_to_<agent_name>`. It documents the `handoff()` customization function and its parameters (agent, tool_name_override, tool_description_override, on_handoff, input_type, input_filter, is_enabled), and explains how `input_type` provides a locally-validated structured-input schema and how `input_filter` shapes the conversation history the receiving agent sees.

## Key evidence
- (ev_001) A handoff lets one agent delegate a task to another agent, useful when agents specialize in distinct areas.
- (ev_002) Handoffs are exposed to the LLM as tools named transfer_to_<agent_name>.
- (ev_003) The handoffs param accepts an Agent or a Handoff object; handoff() exposes agent, tool_name_override, tool_description_override, on_handoff, input_type, input_filter, and is_enabled.
- (ev_004) input_type schema is exposed to the model as the tool's parameters, validated locally, then passed to on_handoff.
- (ev_005) on_handoff receives agent context plus optional LLM-generated input governed by input_type.
- (ev_006) input_filter controls received conversation history, with prebuilt patterns in agents.extensions.handoff_filters.
