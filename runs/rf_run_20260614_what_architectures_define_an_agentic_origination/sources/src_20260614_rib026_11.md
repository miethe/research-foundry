# Source Card: Semantic Router — Introduction (Aurelio AI official docs)

## Summary
Aurelio AI's official Semantic Router documentation introduces the library as a "superfast decision-making layer for LLMs and agents" that routes requests by semantic meaning in vector space rather than waiting on LLM generation for tool-use decisions. It frames the value proposition around faster, cheaper routing (milliseconds vs seconds; avoiding LLM inference for simple routing) and enumerates supported embedding providers (OpenAI, Cohere, Hugging Face, FastEmbed), vector-store backends (Pinecone, Qdrant), and three execution modes (cloud, hybrid, fully local with Llama/Mistral). This establishes Semantic Router as a candidate routing/decision component in an agentic origination architecture.

## Key evidence
- (ev_001) Semantic Router routes requests by meaning in semantic vector space instead of waiting on LLM generations for tool-use decisions.
- (ev_002) Vendor claims millisecond-level routing decisions (vs seconds) and avoidance of expensive LLM inference for simple routing.
- (ev_003) Three execution modes: cloud (API embeddings), hybrid (local embeddings + API LLMs), and fully local (Llama, Mistral).
- (ev_004) Integrates with Pinecone and Qdrant vector stores for persistence.
- (ev_005) Supports Cohere, OpenAI, Hugging Face, and FastEmbed embedding integrations.
- (ev_006) v0.1 is the current released version, with a migration guide for earlier-version users.
- (ev_007) Supports multi-modal (image-content) routing and fully local execution with no API dependencies.
