# Source Card: CrochetPARADE Remesher: Turn a 3D Model Into Crochet Instructions

## Summary
A first-party CrochetPARADE forum thread announces a "Remesher" that converts a closed/watertight STL into round-by-round crochet instructions (no turning chains), seeded by a Magic Ring or Starting Chain. The backend is a surface remesher that maintains a growing "frontier," tries stitch/decrease/increase moves across stitch heights (ss/sc/hdc/dc), enforces geometric/topological guards, and backtracks with an adaptive relaxation ladder, emitting zip sewing edges and supporting multiple yarns. It outputs a 3D preview plus copy-pasteable instructions, permits manual injection, and — per an admin follow-up — can also handle non-watertight STLs with holes.

## Key evidence
- (ev_001) Rounds-only growth (no turning chains) seeded by Magic Ring or Starting Chain.
- (ev_002) Surface remesher maintains a "frontier" — the current boundary of the crocheted surface.
- (ev_003) Tries stitch/decrease/increase moves across ss/sc/hdc/dc with overlap/surface-proximity guards.
- (ev_004) Backtracks to snapshots with an adaptive relaxation ladder; emits zips and supports multiple yarns.
- (ev_005) Fine stitch size (small L) explodes stitch count and can hit WebAssembly memory limits.
- (ev_006) Outputs a 3D preview + copy-pasteable instructions; users can pause and inject manual moves.
- (ev_007) Admin (2026-02-16) reports usable round-based instructions for non-watertight STLs with holes.

## Limitations
- Forum announcement authored by the tool's own maker; claims describe the tool's intended/observed behavior and are not independently verified. The non-watertight-STL capability (ev_007) is an admin observation, explicitly outside the tool's watertight-input design assumption.
