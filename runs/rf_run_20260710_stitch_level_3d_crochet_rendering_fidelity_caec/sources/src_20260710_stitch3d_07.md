# Source Card: Amigurumi Crochet Patterns from Geodesic Distances

## Summary
A Bridges 2024 overview of the AmiGo system, which turns an input 3D model plus a seed point into a human-readable amigurumi crochet pattern. The core idea: amigurumi are worked in the round in single crochet with near-constant row height, so rows coincide with geodesic equidistant curves from the seed; the model is cut and flattened (vertical = geodesic distance/row, horizontal = arc length/stitch), evenly sampled, and mapped back to a crochet graph whose column edges come from a length-minimizing optimization. Curvature guides non-uniform sampling and automatic BLO/FLO crease stitches, and an optimization-based 3D embedding previews the result without physical simulation.

## Key evidence
- (ev_001) Constant SC row height makes each stitch in a row equidistant from the seed, so rows follow geodesic equidistant curves.
- (ev_002) Stitch locations come from flattening the model with vertical = geodesic distance/row and horizontal = arc length/stitch, then even sampling.
- (ev_003) Vertical/column edges are selected by an optimization minimizing total vertical-edge length.
- (ev_004) Craters (neg. mean + pos. Gaussian curvature) are unrealizable and removed by pre-inflation; saddle regions use fewer stitches.
- (ev_005) BLO/FLO stitches are switched in automatically where max absolute curvature is high and orthogonal to the crocheting direction.
- (ev_006) A 3D preview via constrained graph embedding matches the real crocheted output without physical simulation.
- (ev_007) The method join-and-turns per round (not spiral) to avoid slanting; spiral support is future work.

## Limitations
- Overview/short paper; the full method is in the separate "AmiGo" SCF paper (ref [2]). Method assumes maximal stuffing and square SC stitches (rectangular also supported); join-and-turn construction departs from classic spiral amigurumi, and color/yarn-amount/higher-stitch handling is future work.
