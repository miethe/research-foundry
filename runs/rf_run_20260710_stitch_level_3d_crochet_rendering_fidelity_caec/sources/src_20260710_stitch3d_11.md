# Source Card: AmiGo: Computational Design of Amigurumi Crochet Patterns

## Summary
AmiGo (Edelstein et al., SCF 2022) is a computational pipeline that turns a closed 3D mesh plus a single user seed point into a crochetable amigurumi pattern that needs no sewing and uses only simple stitches. It abstracts crochet as a "crochet graph" (stitch tops/bases/stems) — coarser than yarn-level Stitch Meshes — enabling instruction generation in a few minutes versus tens of minutes for AutoKnit. A 3D preview is obtained by geometrically embedding the crochet graph so edge lengths match stitch size, rather than by simulating yarn physics.

## Key evidence
- (ev_001) Input is a closed triangle mesh plus one user seed point; output approximates the stuffed geometry.
- (ev_002) Join-as-you-go attachment removes the need for sewing and uses only simple stitches.
- (ev_003) Stitch model = top/base/stem; inc/dec stitches have multiple stems.
- (ev_004) The crochet graph is more abstract than yarn-level Stitch Meshes (heads/bases/stems only).
- (ev_005) Generation takes a few minutes vs tens of minutes for AutoKnit.
- (ev_006) 3D preview via curvature/edge-length embedding, purely geometric — not yarn physics.
- (ev_007) "Craters" (positive Gaussian + negative mean curvature) obstruct crochetability.
- (ev_008) Assumes stuffing to maximal volume without overstretching the yarn.

## Limitations
- Crochetability obstructions (negative mean curvature "craters", branching/multiply-connected topology) mean not all input meshes are reproducible; results depend on the stuffing-to-maximal-volume assumption. Timing figures ("a few minutes") are author-reported and hardware-dependent.
