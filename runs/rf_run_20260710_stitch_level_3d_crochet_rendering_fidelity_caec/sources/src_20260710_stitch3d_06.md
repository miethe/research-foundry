# Source Card: KnitWit Design Spec (project seed)

## Summary
This is the seed design spec for KnitWit, an "one-stop crochet OS" combining a pattern library/marketplace, row-by-row project execution, inventory/project management, and 3D visualization. It argues the core novelty and risk lie in the 3D visualizer plus reverse 3D-to-pattern generation, and proposes decomposing the 3D ambition into a four-level ladder (L0-L3) with an MVP on pattern playback + tracker + annotated stitches before approximate structural 3D previews and, much later, 3D-to-pattern synthesis. It also sketches a Crochet intermediate representation (IR) as the enabling data model and flags pattern IP and parsing ambiguity as the top business and technical risks.

## Key evidence
- (ev_001) 4-level 3D ambition ladder L0-L3 (2D+counters -> structural preview -> editable semantics -> 3D-to-pattern synthesis).
- (ev_002) Recommends structural-geometry 3D (graph/surface with thickness, textured to suggest yarn) over yarn physics to avoid overloading phones.
- (ev_003) Crochet IR spec: units, pieces[], rows[] stitch ops with counts, constraints/checkpoints, assembly/placement markers.
- (ev_004) Names incumbents Ribblr and My Row Counter; notes no official native Ravelry app.
- (ev_005) IP/copyright is the #1 business landmine; parsing ambiguity is a core technical risk.
- (ev_006) Reverse 3D-to-pattern is Phase 3: computationally grounded but "looks like the model" != pleasant/stable to crochet.

## Limitations
- AI-authored (ChatGPT 5.2) project seed; its external market and academic citations (Ribblr, My Row Counter, CrochetPARADE, amigurumi papers) are secondhand and should be independently verified before load-bearing use. Represents product intent/aspiration, not a validated technical result.
