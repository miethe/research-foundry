# Source Card: Threshold Selection Tool — Splink documentation

## Summary
Splink's threshold selection tool documents how a single match-weight threshold partitions scored record pairs into matches and non-matches, and how moving that threshold trades precision against recall by reshuffling confusion-matrix counts (TP/FP/TN/FN). The chart is generated via `accuracy_analysis_from_labels_table(output_type="threshold_selection")` (with optional metrics like F1), and the chosen operating point is then applied with `cluster_pairwise_predictions_at_threshold()`. No fixed default numeric cutoff is prescribed; the threshold is selected per use-case precision/recall needs from the curve.

## Key evidence
- (ev_001) A match-weight threshold partitions record pairs into matches (scores above) and non-matches (scores below).
- (ev_002) Precision is maximised by raising the threshold (fewer false positives) and recall by lowering it (fewer false negatives).
- (ev_003) Raising the threshold converts True Positives to False Negatives and False Positives to True Negatives.
- (ev_004) Lowering the threshold to the extreme maximises recall at the cost of precision.
- (ev_005) The tool visualizes the relationship between match weight and match probability.
- (ev_006) The chart is produced by `accuracy_analysis_from_labels_table(output_type="threshold_selection", add_metrics=["f1"])`.
