# Source Card: Fuzzy Matching and Deduplicating Hundreds of Millions of Records with Splink

## Summary
Robin Linacre's first-party blog post introduces Splink, an open-source record-linkage library implementing the Fellegi-Sunter model with Expectation-Maximisation as unsupervised learning, so no labeled training data is needed to estimate match weights. The method computes per-attribute partial match weights for arbitrary user-defined comparison levels (e.g. postcode exact vs within 10 miles), which sum into an overall match score representing the total weight of evidence that two records are the same. Pairwise predictions are then resolved into clusters at a configurable match-probability threshold (0.95 in the sample), with the library claimed to scale to 100M+ records on Spark or AWS Athena.

## Key evidence
- (ev_001) Splink implements the Fellegi-Sunter / EM model as unsupervised learning yielding a per-pair match score.
- (ev_002) No labeled training data is required; models train via an unsupervised approach.
- (ev_003) Partial match weights support arbitrary user-defined scenarios (e.g. postcodes within 10 miles), not just binary match/non-match.
- (ev_004) A date-of-birth match carries more evidential weight than a gender match.
- (ev_005) Partial match weights combine into an overall match score = total weight of evidence two records are the same.
- (ev_006) Sample clustering call uses a 0.95 match-probability threshold.
- (ev_007) Claimed to link very large datasets (100M+ records) on Spark or AWS Athena.
- (ev_008) Originally posted 2020-04-16; last updated 2022-08-04.
