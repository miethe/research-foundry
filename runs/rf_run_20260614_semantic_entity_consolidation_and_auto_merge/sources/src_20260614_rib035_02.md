# Source Card: dedupe 3.0.2 Library Documentation

## Summary
The official API reference for the open-source `dedupe` Python library (documented version 3.0.2) specifies an active-learning entity-resolution workflow: the `Dedupe` class iteratively surfaces uncertain record pairs for human labeling (`uncertain_pairs()` / `mark_pairs()`), then `train()` learns a final pairwise classifier and blocking ("fingerprinting") rules. At scoring time, `score()` assigns each candidate pair a match probability, and `cluster()` / `partition()` group records into entity clusters, admitting records only when a cluster's cophenetic similarity exceeds a tunable threshold (default 0.5) that trades recall against precision. This page is the authoritative class/method contract; the higher-level Fellegi-Sunter and agglomerative-clustering framing lives on a separate "How it works" page.

## Key evidence
- (ev_001) The Dedupe class performs active learning deduplication for datasets with multiple records referring to one entity.
- (ev_002) uncertain_pairs() returns the pairs the model is most uncertain about for human labeling.
- (ev_003) mark_pairs() adds labeled match/distinct pairs and updates the matching model.
- (ev_004) train() learns the final pairwise classifier and fingerprinting (blocking) rules.
- (ev_005) score() returns each pair's probability of being a match (the probabilistic linkage step).
- (ev_006) cluster() groups scored pairs into entity clusters with per-record confidence scores.
- (ev_007) partition() admits records to a cluster only above a cophenetic-similarity threshold (default 0.5); threshold trades recall vs precision.
- (ev_008) Documented release is dedupe 3.0.2 by Forest Gregg, Derek Eder, and contributors (copyright 2024).
