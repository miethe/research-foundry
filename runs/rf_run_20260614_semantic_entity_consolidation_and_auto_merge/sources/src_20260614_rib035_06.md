# Source Card: Entity Resolution and Deduplication — Neo4j Agent Memory

## Summary
Neo4j Labs documentation for the Agent Memory project describes a confidence-driven entity resolution and deduplication pipeline. It establishes a three-tier decision model (auto-merge, flag-for-review via a SAME_AS relationship, or create-new), domain-tuned threshold profiles, a blended embedding-plus-fuzzy confidence score with same-type constraints, and operational health metrics for monitoring resolution quality.

## Key evidence
- (ev_001) Three decision tiers: above auto-merge -> merge; between flag and merge -> SAME_AS review link; below flag -> new entity.
- (ev_002) Domain-tuned thresholds: Financial Services 0.98/0.90, Ecommerce 0.95/0.85, Content/Media 0.92/0.75.
- (ev_003) Blended confidence score = embedding_score * 0.7 + fuzzy_score * 0.3, matching only same-type entities.
- (ev_004) SAME_AS review relationship metadata: confidence, status (pending/confirmed/rejected), created_at/reviewed_at, reviewed_by, method.
- (ev_005) Health thresholds: auto-merge rejection rate < 2%, flag-to-confirm rate 60-90%, orphan entity rate < 10%.
