# Source Card: Python Record Linkage Toolkit 0.15 documentation - About

## Summary
The "About" page of the Python Record Linkage Toolkit documents an open-source Python library for linking records across or within data sources (record linkage / deduplication). It is built on pandas and numpy, was inspired by the FEBRL project, and structures linkage as a five-step workflow: cleaning, indexing, comparing, classifying, and evaluation. It ships indexing strategies (blocking, sorted neighbourhood), configurable string-similarity comparisons (e.g., Jaro-Winkler, Damerau-Levenshtein), and both supervised (Logistic Regression) and unsupervised (Bernoulli EM / ECM) classifiers, grounding its design in the canonical record-linkage literature.

## Key evidence
- (ev_001) Implements the standard five-step linkage workflow (cleaning, indexing, comparing, classifying, evaluation) with feedback to earlier steps.
- (ev_002) Inspired by FEBRL but built extensively on pandas and numpy for speed and integration with existing data projects.
- (ev_003) Generates candidate pairs via blocking and sorted neighbourhood indexing; compares strings, numbers, and dates.
- (ev_004) Compare class supports configurable string methods including Jaro-Winkler and Damerau-Levenshtein with thresholds, plus exact matching.
- (ev_005) Offers supervised LogisticRegressionClassifier and unsupervised BernoulliEMClassifier (the ECM-algorithm).
- (ev_006) Cites Christen (2012), Fellegi & Sunter (1969), Dunn (1946), and Herzog/Scheuren/Winkler (2007).

## Limitations
- Self-described capabilities from the project's own documentation, not independent benchmarks; the page explicitly frames the toolkit as "developed for research and the linking of small or medium sized files."
