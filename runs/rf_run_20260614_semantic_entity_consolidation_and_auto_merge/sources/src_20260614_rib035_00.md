# Source Card: Splink: Fast, accurate and scalable probabilistic data linkage with support for multiple SQL backends (MoJ Analytical Services)

## Summary
Splink is an open-source probabilistic record-linkage library maintained by Ministry of Justice Analytical Services. Its matching algorithm is based on the Fellegi-Sunter model with accuracy customisations, and it estimates model parameters via unsupervised learning requiring no labeled training data. It runs on multiple SQL backends (DuckDB by default, plus Spark, AWS Athena, and PostgreSQL), claiming to link a million records on a laptop in roughly a minute and to scale to 100+ million records on big-data backends. Splink 4 was announced on 2024-07-24.

## Key evidence
- (ev_001) Splink's linkage algorithm is built on the Fellegi-Sunter model of record linkage with customisations to improve accuracy.
- (ev_002) Splink performs unsupervised model training requiring no labeled training data.
- (ev_003) Splink can link approximately one million records on a laptop in around a minute.
- (ev_004) Splink runs in Python on DuckDB or on big-data backends such as AWS Athena or Spark to scale to 100+ million records.
- (ev_005) Splink supports term frequency adjustments and user-defined fuzzy matching logic to improve match accuracy.
- (ev_006) Splink provides interactive visualisations to help users understand their model and diagnose problems.

## Limitations
- Performance figures (million records/minute; 100M+ records) are vendor/maintainer claims from the README, not independently benchmarked here; actual throughput depends on hardware, backend, and blocking configuration. PostgreSQL backend support is listed via installation options in the README rather than restated in the Key Features list.
