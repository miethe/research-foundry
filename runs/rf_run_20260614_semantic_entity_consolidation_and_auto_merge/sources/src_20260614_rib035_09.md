# Source Card: OpenSanctions Pairs: Large-Scale Entity Matching with LLMs

## Summary
This paper introduces a large-scale entity-matching benchmark (OpenSanctions Pairs) of 755,540 labeled pairs from 293 international sanctions sources across 31 countries, covering 1,002,093 unique entities with up to 132 property fields. It evaluates rule-based and LLM matchers, finding LLMs (best ~98.23-98.95% F1) substantially outperform the Nomenklatura rule-based baseline (91.33% F1), with the strongest prompt using a conflict-detection framing. The authors argue pairwise matching is approaching a practical ceiling and recommend reallocating effort to blocking, clustering, and uncertainty-aware review.

## Key evidence
- (ev_001) The benchmark comprises 755,540 labeled entity pairs from 293 sources across 31 countries.
- (ev_002) The corpus covers 1,002,093 unique entities, each with up to 132 property fields.
- (ev_003) Nomenklatura reaches 91.33% F1; best open-source LLM 98.23% F1; GPT-4o 98.95% F1.
- (ev_004) The best prompt uses conflict-detection framing: default POSITIVE unless contradictory evidence.
- (ev_005) Rule-based and LLM methods have complementary errors (over-prediction vs. transliteration failures).
- (ev_006) Authors recommend ensembling Nomenklatura high-recall filtering with LLM re-ranking.
- (ev_007) Pairwise matching is near a practical ceiling; effort should shift to blocking, clustering, uncertainty-aware review.

## Limitations
- Reported F1 numbers and claims are from the authors' own preprint and have not been independently reproduced here; the benchmark is single-domain (international sanctions), which may limit generalization to other entity-consolidation settings.
