# Source Card: SmolVLM - small yet mighty Vision Language Model

## Summary
The official Hugging Face announcement of SmolVLM, a 2B-parameter vision language model built on the SmolLM2 1.7B backbone. It establishes first-party figures for SmolVLM's memory footprint (min 5.02 GB GPU RAM), aggressive visual-token compression (81 tokens per 384x384 patch; ~1.2k vs Qwen2-VL's ~16k tokens for a single image + prompt), throughput gains over Qwen2-VL, benchmark accuracy across DocVQA/TextVQA/MMMU/MathVista/MMStar, and an Apache 2.0 open-source release.

## Key evidence
- (ev_001) SmolVLM reports best-in-class memory usage among VLMs in transformers (min 5.02 GB GPU RAM).
- (ev_002) SmolVLM uses SmolLM2 1.7B as its language backbone, replacing Llama 3.1 8B.
- (ev_003) Each 384x384 image patch is encoded to 81 visual tokens.
- (ev_004) A test prompt + single image is ~1.2k tokens vs Qwen2-VL's ~16k tokens.
- (ev_005) Prefill throughput 3.3-4.5x and generation throughput 7.5-16x faster than Qwen2-VL.
- (ev_006) Benchmarks: DocVQA 81.6, TextVQA 72.7, MMMU(val) 38.8, MathVista 44.6, MMStar 42.1.
- (ev_007) Released under the Apache 2.0 license (checkpoints, datasets, recipes, tools).

## Limitations
- Benchmark and throughput numbers are self-reported by the model authors/vendor and have not been independently reproduced; treat as first-party claims.
