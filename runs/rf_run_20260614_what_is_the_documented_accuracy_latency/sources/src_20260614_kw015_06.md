# Source Card: AmiGo: Computational Design of Amigurumi Crochet Patterns

## Summary
AmiGo (ACM SCF '22) is a peer-reviewed method that inverts a 3D geometry into human-executable amigurumi crochet instructions. Given a closed (watertight) triangle mesh, a single user-selected seed point, and a target stitch width, it builds a Crochet Graph and translates it into a structured pattern of rounds restricted to single-crochet, increase, and decrease stitches. The work establishes that inverse mesh-to-pattern design is computationally grounded and bounded by strong preconditions, not general free-form generation.

## Key evidence
- (ev_001) Generates human-executable crochet instructions from a closed mesh plus a single user point, yielding a toy similar to the input geometry.
- (ev_002) Inputs are precisely a 3D mesh, a seed point, and a stitch width (gauge).
- (ev_003) The mesh is formally required to be a closed triangle mesh, encoding a watertight-topology precondition.
- (ev_004) Shape is automatically segmented into crochetable components joined via join-as-you-go, with no extra sewing.
- (ev_005) Output is restricted to a fixed instruction set (sc, inc, dec), i.e. deterministic structured output.
- (ev_006) Frames inverse mesh-to-pattern design as a computationally grounded approach producing patterns, not free-form generation.

## Limitations
- The ACM canonical PDF is paywalled (HTTP 403); body-section quotes were verified against the authors' open-access arXiv preprint (arXiv:2211.01178) of the same work, while the Abstract quote is confirmed verbatim against arXiv. Strong preconditions (closed/watertight mesh, seed point, fixed gauge) bound the problem and limit generality to inputs satisfying those constraints.
