# Source Card: dots.ocr / dots.mocr - Multilingual Document Layout Parsing in a Single Vision-Language Model

## Summary
The rednote-hilab dots.ocr project README documents dots.mocr, a single multilingual document-layout-parsing vision-language model (built on a 1.7b-parameter LLM, described as a 3B-parameter VLM) that was rebranded from dots.ocr-1.5 on 2026-03-19. It self-reports strong accuracy benchmarks (olmOCR-Bench 83.9 plus/minus 0.9; OmniDocBench text-edit distance 0.031, ahead of Gemini-2.5 Pro and Qwen3-VL-235B) and parses eleven layout categories in human reading order. The README publishes no latency or throughput figures.

## Key evidence
- (ev_001) dots.ocr-1.5 was rebranded as dots.mocr on 2026-03-19, with details in an arXiv paper.
- (ev_002) Single multilingual document-parsing VLM based on a 1.7b LLM, described as a 3B-parameter VLM.
- (ev_003) olmOCR-Bench overall score of 83.9 plus/minus 0.9.
- (ev_004) OmniDocBench text-edit distance 0.031 vs Gemini-2.5 Pro 0.075 and Qwen3-VL-235B 0.069 (lower is better).
- (ev_005) Parses eleven layout categories (Caption, Footnote, Formula, List-item, Page-footer, Page-header, Picture, Section-header, Table, Text, Title), sorted in human reading order.
- (ev_006) Claims SOTA multilingual parsing among comparable-size models and recognition of virtually any human script.
- (ev_007) No explicit latency/throughput metrics published; on-device inference cost is unquantified.

## Limitations
- Benchmarks are self-reported by the project maintainers (primary source, no independent verification).
- Parameter count is stated inconsistently in the README (1.7b LLM base vs 3B VLM); both phrasings are captured.
- No latency/throughput data is published, so runtime cost remains unquantified. The OmniDocBench Gemini-2.5 Pro text-edit figure in the README (0.075) differs from the 0.066 value circulating in some secondary summaries; the README value is used here.
