# Source Card: Apple Intelligence Foundation Language Models Tech Report 2025

## Summary
Apple's 2025 technical report describes the two foundation models behind Apple Intelligence: a ~3B-parameter on-device model optimized for Apple silicon and a scalable Parallel-Track Mixture-of-Experts (PT-MoE) server model running on Private Cloud Compute. The on-device model uses a two-block KV-cache-sharing architecture and 2-bit quantization-aware training (4-bit embeddings, 8-bit KV cache) to cut memory and latency, and both models are multimodal (image understanding) and agentic (tool calls). Apple reports the on-device model outperforming several comparably sized open baselines (Qwen-2.5-3B, Gemma-3-4B) on MMLU/MMMLU while lagging the larger Qwen-3-4B.

## Key evidence
- (ev_001) ~3B on-device model optimized for Apple silicon via KV-cache sharing and 2-bit QAT.
- (ev_002) Two-block split (62.5%/37.5%); Block 2 shares Block 1's KV cache, cutting KV-cache memory by 37.5%.
- (ev_003) Block 2 produces no KV, so prefill is bypassed, reducing TTFT by ~37.5%.
- (ev_004) On-device model compressed to 2 bits-per-weight (QAT); embedding table to 4 bits, KV-cache to 8 bits, with low-rank recovery adapters.
- (ev_005) Multimodal + agentic capabilities; server model is a PT-MoE transformer on Private Cloud Compute.
- (ev_006) On-device model beats Qwen-2.5-3B and Gemma-3-4B on MMLU/MMMLU but lags the larger Qwen-3-4B.

## Limitations
- Benchmark figures are vendor-reported (first-party) and not independently verified; evaluation was limited to AFM-supported languages and instruct/non-thinking model variants. The summary landing page only carries abstract-level claims; the detailed quantization and benchmark quotes are drawn from the full report (arXiv:2507.13575v3).
