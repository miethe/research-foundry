# Source Card: nomic-ai/nomic-embed-text-v1.5 - Hugging Face model card

## Summary
The official Nomic AI model card for nomic-embed-text-v1.5, an open-weight text embedding model (~0.1B / 137M params) built on a long-context BERT base (nomic-bert-2048) that supports sequence lengths past 2048 tokens up to 8192. Its defining feature is Matryoshka Representation Learning, which lets a single model emit embeddings truncatable to 768/512/256/128/64 dimensions with only minor MTEB degradation, enabling storage/cost tradeoffs in vector stores without re-embedding.

## Key evidence
- (ev_001) Matryoshka Representation Learning lets developers trade embedding size for negligible performance loss.
- (ev_002) Supported dimensions 768/512/256/128/64 map to MTEB averages 62.28/61.96/61.04/59.34/56.10.
- (ev_003) 768->512 costs ~0.32 MTEB points; 768->256 costs ~1.24 points.
- (ev_004) Native sequence-length scaling past 2048 tokens (up to 8192).
- (ev_005) Built starting from a long-context BERT base (nomic-bert-2048) with contrastive pretraining.
- (ev_006) Dimension reduction is done client-side by layer-norm then slicing to matryoshka_dim.

## Limitations
- MTEB scores are author-reported on the official model card, not independently audited; per-task results vary and the 64d setting (56.10) shows materially larger degradation than the 256/512 settings.
