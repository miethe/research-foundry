# Source Card: Releases · asg017/sqlite-vec

## Summary
The asg017/sqlite-vec GitHub releases page documents the version timeline and feature evolution of sqlite-vec, a small loadable SQLite extension for vector search. As of mid-2026, the latest releases are the v0.1.10-alpha series (alpha.4, 2026-05-18) and the last stable line is v0.1.9 (2026-03-31). The alpha series adds approximate-nearest-neighbor (ANN) indexing — DiskANN, IVF (experimental, disabled), and rescore — building on the brute-force KNN that defined the stable v0.1.x line, with all vector data stored inside the SQLite database via vec0 virtual tables.

## Key evidence
- (ev_001) Latest releases are the v0.1.10-alpha series (alpha.4 dated 2026-05-18); last stable line is v0.1.9 (2026-03-31).
- (ev_002) The v0.1.10 alpha introduces three ANN approaches: rescore, IVF (experimental, not enabled), and DiskANN.
- (ev_003) ANN/DiskANN was announced as the "biggest update coming soon" in v0.1.7, implying prior stable releases were brute-force KNN only.
- (ev_004) sqlite-vec is a pure-C loadable SQLite extension with no dependencies that runs anywhere SQLite runs (Linux/macOS/Windows, WASM, Raspberry Pi).
- (ev_005) Vectors are stored and queried in vec0 virtual tables (float, int8, binary), so the store lives inside the SQLite database file.
- (ev_006) DiskANN/IVF ANN features remain under active alpha development with bug fixes and integration tests across flat/ANN indexes.

## Limitations
- Release-note descriptions are terse; the "data lives inside the SQLite database file" property follows from sqlite-vec being a SQLite extension using virtual tables rather than being restated verbatim per release. IVF is explicitly experimental and disabled; DiskANN is still alpha (pre-stable) as of v0.1.10-alpha.4.
