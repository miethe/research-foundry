# Source Card: pgvector 0.7.0 Released!

## Summary
This is the official PostgreSQL.org news post announcing the 0.7.0 release of pgvector, the open-source PostgreSQL extension for vector similarity search. The release introduces new vector storage types (halfvec, sparsevec), binary vector indexing via the bit type, vector quantization (including binary_quantize), new bit-vector distance functions, HNSW support for L1 distance, and SIMD with CPU dispatching on Linux x86-64. It establishes the concrete dimensional limits and capability set available in pgvector 0.7.0 for an embedding/vector-store strategy.

## Key evidence
- (ev_001) pgvector 0.7.0 announced on PostgreSQL.org 2024-04-30 (CHANGELOG dated 2024-04-29).
- (ev_002) Adds halfvec (2-byte floats, indexable to 4,000 dims) and sparsevec (indexable to 1,000 nonzero dims).
- (ev_003) Adds binary vector indexing via the bit type, indexable up to 64,000 dimensions.
- (ev_004) Adds expression-index quantization: 4-byte to 2-byte floats and binary_quantize.
- (ev_005) Adds hamming_distance and jaccard_distance for bit vectors plus HNSW support for L1 distance.
- (ev_006) Adds SIMD with CPU dispatching for Linux x86-64.

## Limitations
- This announcement covers pgvector 0.7.0 only; it does not mention 0.8.0 features such as iterative scans pairing HNSW with binary quantization, so any 0.8.0 claim requires a separate source.
- The post is a high-level summary and points to the GitHub CHANGELOG (070-2024-04-29) for full per-feature detail.
