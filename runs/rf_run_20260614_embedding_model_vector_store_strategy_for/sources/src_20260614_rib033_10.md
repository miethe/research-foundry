# Source Card: pgvector/pgvector: Open-source vector similarity search for Postgres

## Summary
pgvector is the official open-source vector similarity search extension for PostgreSQL, distributed as the `pgvector/pgvector` repository. The README is authoritative for index defaults and tuning guidance: HNSW (m=16, ef_construction=64, ef_search=40) trades faster, higher-recall queries for slower builds and more memory than IVFFlat (lists sized rows/1000 up to 1M rows, sqrt(rows) above; probes starting at sqrt(lists)). It supports halfvec indexing up to 4,000 dimensions and bit/binary indexing up to 64,000 dimensions, and as a Postgres extension (current release v0.8.2) it requires a running Postgres server rather than an embeddable single-file store.

## Key evidence
- (ev_001) HNSW defaults: m=16 (max connections per layer) and ef_construction=64 (build candidate list).
- (ev_002) HNSW query-time ef_search defaults to 40.
- (ev_003) HNSW gives better speed-recall query performance than IVFFlat but slower builds and higher memory.
- (ev_004) IVFFlat list sizing: rows/1000 up to 1M rows, sqrt(rows) above 1M rows.
- (ev_005) IVFFlat probes recommended to start at sqrt(lists).
- (ev_006) Indexing supports halfvec up to 4,000 dimensions and bit/binary up to 64,000 dimensions.
- (ev_007) Current release v0.8.2; runs as a Postgres extension requiring a running Postgres server (Postgres 13+).

## Limitations
- README-documented defaults and tuning recommendations are version-specific (captured at v0.8.2) and may change in later releases; verify against the installed extension version before relying on exact numbers.
