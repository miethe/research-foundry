# Source Card: Speeding up Inference - Sentence Transformers documentation

## Summary
The official Sentence Transformers efficiency guide documents first-party benchmarks comparing inference backends (PyTorch fp32/fp16/bf16, ONNX with O1-O4 optimization and int8 quantization, and OpenVINO with int8 quantization) on CPU and GPU. It establishes concrete speedup factors over the PyTorch baseline and provides a hardware-driven flowchart for backend selection. Throughout, it stresses that the numbers are aggregate averages and that real workloads must be benchmarked per model, text length, and batch size.

## Key evidence
- (ev_001) On CPU, ONNX int8 quantization reaches 3.08x speedup for short texts vs PyTorch (plain ONNX 1.39x, OpenVINO 1.29x).
- (ev_002) For Intel CPUs with a tolerable 0.4% accuracy loss the docs recommend openvino-qint8; non-Intel CPUs use plain onnx.
- (ev_003) On GPU, recommend onnx-O4 for texts under ~500 characters and PyTorch float16 for longer texts.
- (ev_004) On GPU short texts, ONNX-O4 reaches 1.83x vs fp16/bf16 at 1.54x/1.53x, hence ONNX recommended for shorter GPU texts.
- (ev_005) Speedups are averaged across 4 models, 3 datasets, and many batch sizes; aggressive averaging hides finer patterns.
- (ev_006) For longer texts ONNX/OpenVINO can be slightly worse than PyTorch; always test backends on your own model and data.

## Limitations
- Benchmark speedups are aggregate averages across 4 models, 3 datasets, and numerous batch sizes and are not guaranteed for any single model, text length, or batch size; the page itself repeatedly directs readers to measure per their own workload.
