# Source Card: Vector embeddings | OpenAI API (text-embedding-3 model specs)

## Summary
This official OpenAI API guide documents the specifications of the text-embedding-3 model family relative to the legacy text-embedding-ada-002 model. It establishes vendor-reported MTEB benchmark scores, default output dimensions, the shared 8192-token input limit, and the behavior of the `dimensions` parameter that enables Matryoshka-style embedding truncation without proportional loss of representational quality.

## Key evidence
- (ev_001) MTEB scores: 3-large 64.6%, 3-small 62.3%, ada-002 61.0%.
- (ev_002) All three models accept a maximum of 8192 input tokens.
- (ev_003) Default output dimensions: 3-small 1536, 3-large 3072.
- (ev_004) The `dimensions` parameter shortens embeddings without losing concept-representing properties.
- (ev_005) 3-large shortened to 256 dims still outperforms unshortened ada-002 at 1536 dims.
- (ev_006) OpenAI recommends setting `dimensions` at creation time as the preferred reduction approach.

## Limitations
- MTEB scores are vendor self-reported and reflect a single aggregate benchmark; downstream task performance may differ.
- API specifications and pricing are subject to change without notice; figures are accurate as of the documented version.
