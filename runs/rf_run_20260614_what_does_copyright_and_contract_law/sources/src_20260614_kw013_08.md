# Source Card: Rate Limits | Etsy Open API v3

## Summary
Etsy's first-party Open API v3 documentation specifies application-based rate limiting built from two dimensions — Queries Per Second (QPS) and Queries Per Day (QPD) — enforced per API key for both public and private auth. The QPD limit uses a progressive sliding-window algorithm (rolling 24-hour window divided into buckets) rather than a fixed midnight reset, and successful responses surface live usage via x-limit/x-remaining headers. When QPS (checked first) or QPD is exceeded, the API returns HTTP 429 with a retry-after header; developers raise limits by emailing developer@etsy.com with an app description and usage estimate, and are advised to cache aggressively and back off exponentially.

## Key evidence
- (ev_001) Application-based rate limits = QPD + QPS, applied per API key for both public and private auth.
- (ev_002) QPD uses a progressive Sliding Window Rate Limiting Algorithm, not a fixed midnight-to-midnight reset, computed over the last rolling 24-hour period.
- (ev_003) As the oldest bucket exits the 24-hour window its consumed quota is immediately freed for reuse.
- (ev_004) Successful responses include usage headers (x-limit-per-second, x-limit-per-day, etc.) with documented example values of 150 QPS / 100000 QPD.
- (ev_005) Limits are evaluated QPS-first then QPD; exceeding either returns HTTP 429 with a retry-after header.
- (ev_006) Higher limits are requested via developer@etsy.com with an app description and QPD/QPS usage estimate.
- (ev_007) Etsy recommends caching to cut redundant calls and exponential backoff on 429 responses.

## Limitations
- The live page (accessed 2026-06-14) does NOT state explicit default public-auth limits (e.g., 10 QPS / 10,000 QPD), an Enterprise tier of ~3 million calls/day, the API Terms-of-Use 6-hour display / minimal-retention storage limits, or a "creating additional API keys is strictly prohibited" clause; those claims from upstream notes could not be verified against the current page text and are omitted. The 150 QPS / 100000 QPD figures are documented only as header *example values*, not as guaranteed tier limits. The page carries no explicit publication/last-updated date (footer shows © 2021).
