# Source Card: VeriFastScore: Speeding up long-form factuality evaluation

## Summary
VeriFastScore (Rajendhran et al., arXiv:2505.16973, May 2025) fine-tunes Llama-3.1-8B to extract and verify all verifiable claims in a long-form response in a single forward pass, distilling a synthetic dataset generated by the slower VeriScore decompose-then-verify pipeline. It reports a 6.6x overall speedup (9.9x excluding evidence retrieval) with strong correlation to VeriScore (r=0.80 example-level, r=0.94 system-level). A coarser-granularity evidence-retrieval variant further cuts API costs by 39% and nearly halves retrieval time, making it relevant as a cheaper alternative architecture for a claim-verification gate.

## Key evidence
- (ev_001) Fine-tunes Llama-3.1-8B to extract and verify all verifiable claims at once, using Google Search evidence.
- (ev_002) Strong correlation with VeriScore (r=0.80 example, r=0.94 system) at a 6.6x overall speedup (9.9x ex-retrieval).
- (ev_003) Coarser-granularity retrieval cuts API costs 39% and nearly halves retrieval time.
- (ev_004) Decompose-then-verify baselines (FactScore, VeriScore) can take 100+ seconds per response.
- (ev_005) The combined task is too complex for few-shot prompting: ~4K tokens of evidence, concurrent decompose/judge/verify.
- (ev_006) Training data is a synthetic dataset generated via the VeriScore pipeline (distillation).

## Limitations
- Results are self-reported in a preprint and not independently replicated; human evaluation was limited to 401 tuples from 21 test instances by the first author. Within the 180-day freshness window (published 2025-05, accessed 2026-06-14); claims about specific numbers (speedup, 39%, r-values) reflect the v3 revision.
