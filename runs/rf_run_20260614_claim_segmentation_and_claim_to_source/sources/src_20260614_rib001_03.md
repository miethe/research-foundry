# Source Card: A Closer Look at Claim Decomposition

## Summary
This *SEM 2024 paper (Wanner et al., Johns Hopkins) argues that claim-decomposition methods are not interchangeable: the same generated text yields different numbers of subclaims and different FActScore values depending on how it is split, so apparent score changes can be artifacts of the decomposition step rather than the model. It defines decomposition quality via coherence (faithfulness to the original claim), coverage (all parts of the claim represented), and atomicity (maximal granularity without redundancy), and introduces DecompScore — the average number of supported subclaims per passage, validated against the original passage rather than external truth. Empirically, atomic-fact yield per biography ranges from ~20 (WiCE) to ~42 (Russellian/Neo-Davidsonian), demonstrating the splitter materially shapes downstream metrics.

## Key evidence
- (ev_001) Factuality metrics attribute textual support to the model even though error can originate in the metric's decomposition step.
- (ev_002) FActScore can change for the same text depending on the atomicity/characteristics of the decomposition method.
- (ev_003) Decomposition quality requires subclaims to be coherent/faithful to (supported or entailed by) the original claim.
- (ev_004) Subclaims should cover all parts of the claim and be as atomic as possible (coverage + atomicity).
- (ev_005) DecompScore = average number of supported subclaims per passage for a decomposition method, judged against the original passage.
- (ev_006) Atomic-fact yield per biography: Russellian/Neo-Davidsonian 42.3, FActScore/Chen ~32, PredPatt 29.2, CoNLL-U 27.1, WiCE 20.0.

## Limitations
- DecompScore measures subclaim coherence/yield against the original passage, not external factual correctness; high yield does not imply higher factual accuracy.
- Atomic-fact yields are measured on a static set of biographies; generalization to other domains is not established.
- The ev_006 numeric values are reported from Figure 3 / Results; no short verbatim sentence captures them, so the quote is null while the figures are cited from the source's results table.

## Relevance to RF
For Research Foundry this implies the claim splitter must be fixed and versioned so claim-yield and any verification score are reproducible across runs; swapping decomposition methods silently changes claim counts and metrics on identical evidence text.
