# Source Card: watchdog — Python API and shell utilities to monitor file system events (GitHub repo)

## Summary
watchdog is the canonical Python library for monitoring file system events, abstracting over platform-native backends (inotify, FSEvents, kqueue, ReadDirectoryChangesW) with a cross-platform PollingObserver fallback. Its README documents backend selection, known scalability limits of kqueue, editor-specific event quirks (Vim), and network-filesystem caveats (CIFS). The current release is 6.0.0 (2024-11-01), requiring Python 3.9+.

## Key evidence
- (ev_001) watchdog selects a platform-native backend per OS — inotify (Linux), FSEvents/kqueue (macOS), kqueue (BSD), ReadDirectoryChangesW (Windows) — with an OS-independent polling fallback.
- (ev_002) kqueue uses one file descriptor per watched file, so it does not scale to deeply nested directories with many files.
- (ev_003) Vim edits via backup+swap, so watchdog's on-modified events do not fire for Vim-edited files.
- (ev_004) For CIFS mounts, watchdog must be told explicitly to use PollingObserver rather than auto-selecting a backend.
- (ev_005) watchdog 6.0.0 (2024-11-01) requires Python 3.9+.
