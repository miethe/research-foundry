# Source Card: skip — Lefthook Documentation

## Summary
The official Lefthook `skip` configuration reference documents how to suppress hooks, commands, and scripts. Skipping can be unconditional (`skip: true`) or conditional on git state (`merge`, `rebase`, `merge-commit`), on a branch/ref (`ref: main`, glob `ref: dev/*`), or on a shell test (`run: ...`) that returns 0. The `skip` key can be placed at the hook level or per individual command/script.

## Key evidence
- (ev_001) Lefthook's `skip` option suppresses all or specific commands/scripts, including on merge, rebase, or a given branch, with glob support for branches.
- (ev_002) `merge` and `rebase` skip values suppress hooks while in those git states; `merge-commit` skips when HEAD is the merge commit.
- (ev_003) `ref:` targets a branch/ref (e.g. `ref: main`), and glob patterns like `ref: dev/*` are supported.
- (ev_004) `run: test ${SKIP_ME} -eq 1` skips when the shell command succeeds (return code 0).
- (ev_005) The existence guard `run: "! which <tool>"` runs a command only if the tool is installed.
- (ev_006) Skip can be applied at the hook level or per-command/per-script for fine-grained control.
