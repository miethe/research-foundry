# Source Card: pre-commit — A framework for managing git pre-commit hooks

## Summary
The official pre-commit documentation defines the framework's supported git hook stages and the configuration semantics that govern when hooks run. It enumerates ten hook types (from commit-msg through prepare-commit-msg), explains that file-less hooks (post-checkout, post-commit, post-merge, post-rewrite, pre-rebase) must use always_run or be skipped, and documents how hook stages are selected and installed via --hook-type, default_install_hook_types, the stages property, and default_stages. By default only the pre-commit hook is installed unless additional types are configured.

## Key evidence
- (ev_001) pre-commit supports ten git hook stages including post-checkout, post-commit, post-merge, post-rewrite, pre-push, and pre-rebase.
- (ev_002) File-less hooks (post-checkout, post-commit, post-merge, post-rewrite, pre-rebase) must be set always_run: true or they are always skipped.
- (ev_003) Hooks are installed for particular stages via pre-commit install --hook-type, repeatable across multiple types.
- (ev_004) The stages property (overridable from .pre-commit-hooks.yaml to .pre-commit-config.yaml) restricts hook stages, falling back to default_stages.
- (ev_005) default_install_hook_types defaults to [pre-commit], so only the pre-commit hook is installed unless other types are specified.
- (ev_006) default_stages is a configuration-wide default for the stages property and defaults to all stages.
