# Source Card: chokidar — Minimal and efficient cross-platform file watching library (README)

## Summary
chokidar is the canonical Node.js cross-platform file-watching library (maintained by Paul Millr). Its README documents the option defaults and behaviors most relevant to file-watch-driven hooks: atomic-write coalescing within a 100 ms window, awaitWriteFinish debouncing of in-progress writes, polling-vs-native-watch trade-offs, and OS-level watch-limit (ENOSPC/EMFILE) troubleshooting. As of November 2025 it is at v5 (ESM-only, Node.js v20+).

## Key evidence
- (ev_001) atomic option (default true when not using fsevents/polling) merges a delete + re-add within 100 ms into one 'change' event.
- (ev_002) awaitWriteFinish debounces large/chunked writes: stabilityThreshold default 2000 ms, pollInterval default 100 ms.
- (ev_003) Polling off by default; polling interval default 100 ms, binaryInterval default 300 ms.
- (ev_004) File-handle exhaustion surfaces as EMFILE/ENOSPC; ENOSPC (inotify limit) is fixable by raising fs.inotify.max_user_watches.
- (ev_005) v5 (Nov 2025) is ESM-only and needs Node v20+; v4 (Sep 2024) dropped globs and cut deps 13→1.

## Limitations
- Defaults and exact millisecond values are version-specific (current README reflects v5, Nov 2025) and may differ in older installed versions; verify against the version in use.
- The README's troubleshooting text shows a typo ("ENOSP") in the EMFILE/ENOSPC sentence; quoted verbatim.
