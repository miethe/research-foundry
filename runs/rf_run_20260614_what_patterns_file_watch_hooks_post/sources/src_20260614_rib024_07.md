# Source Card: 9.2 Server-sent events — HTML Living Standard (WHATWG)

## Summary
This is the normative WHATWG HTML Living Standard section defining server-sent events (SSE) and the EventSource interface. It establishes the wire format (text/event-stream over UTF-8), the resume protocol via the `id` field, the last event ID buffer/string, and the `Last-Event-ID` request header, plus automatic reconnection semantics (reconnection time in ms, HTTP 204 to stop reconnecting). It is the authoritative reference for at-least-once resume behavior in file-watch/hook post patterns built on SSE.

## Key evidence
- (ev_001) The Last-Event-ID HTTP request header reports the EventSource's last event ID string to the server on reconnection.
- (ev_002) An 'id' field (no NULL) sets the last event ID buffer; on dispatch the last event ID string persists until the server next sets it, enabling resume.
- (ev_003) Reconnection time is in milliseconds; the initial value is implementation-defined, "probably in the region of a few seconds".
- (ev_004) The event stream format's MIME type is text/event-stream.
- (ev_005) Clients reconnect automatically on close; HTTP 204 No Content tells a client to stop reconnecting.
- (ev_006) A 'retry' field of only ASCII digits sets the event stream's reconnection time as a base-ten integer.

## Limitations
- Living Standard: section numbering and exact wording can change over time; quotes captured as of the 2026-06 access. The "few seconds" reconnection default is implementation-defined, so real values vary across user agents.
