# Source Card: Pricing — Anthropic Claude API (Official Docs)

## Summary
This is Anthropic's first-party pricing documentation for the Claude API as of mid-2026. It establishes per-MTok list prices across the model tiers (Haiku at $1/$5, Sonnet at $3/$15, Opus at $5/$25 input/output), the prompt-caching multipliers (1.25x / 2x writes, 0.1x reads), the 50% Batch API discount, and a worked support-ticket cost example. It is authoritative for list pricing but excludes negotiated discounts and partner-cloud (Bedrock/Vertex) rates.

## Key evidence
- (ev_001) Claude Haiku 4.5 lists at $1/MTok input, $5/MTok output, $0.10/MTok cache read.
- (ev_002) Claude Sonnet 4.5 and 4.6 list at $3/MTok input, $15/MTok output.
- (ev_003) Claude Opus 4.5–4.8 all list at $5/MTok input, $25/MTok output.
- (ev_004) Prompt-caching multipliers: 5-min write 1.25x, 1-hour write 2x, cache read 0.1x base input.
- (ev_005) Batch API gives a 50% discount on both input and output tokens.
- (ev_006) 10,000 support tickets (~3,700 tokens each) on Haiku 4.5 cost ~$37.00 total.
- (ev_007) Opus 4.7+ use a new tokenizer that can use up to 35% more tokens for the same text.

## Limitations
- List prices only; excludes negotiated volume/enterprise discounts and academic/research pricing.
- Partner-operated clouds (Amazon Bedrock, Vertex AI) have independent pricing not covered by these figures.
- Prices are time-sensitive; the page itself directs readers to claude.com/pricing for the most current rates.
- The new-tokenizer note (ev_007) means headline per-MTok parity does not always equal per-text cost parity across Opus versions.
