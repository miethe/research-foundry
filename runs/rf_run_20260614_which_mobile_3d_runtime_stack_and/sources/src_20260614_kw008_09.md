# Source Card: Filament VertexBuffer.h (libs/filament/include/filament/VertexBuffer.h)

## Summary
This is the canonical public C++ header for Filament's VertexBuffer API. It documents how vertex data is uploaded to the GPU: full or partial slot updates via setBufferAt() at a 4-byte-aligned byte offset, a hard cap of 8 buffer slots per vertex buffer set, a "buffer objects" mode (enableBufferObjects() + setBufferObjectAt()) that hot-swaps whole GPU BufferObjects instead of re-uploading, and async variants (setBufferAtAsync / setBufferObjectAtAsync) for non-blocking, callback-completed streaming uploads.

## Key evidence
- (ev_001) setBufferAt does a partial copy-init into one buffer slot at a byte offset that must be a multiple of 4.
- (ev_002) A vertex buffer set supports a maximum of 8 buffers, so attributes can live in separate updatable slots.
- (ev_003) enableBufferObjects mode requires setBufferObjectAt instead of setBufferAt and allows data sharing between VertexBuffers.
- (ev_004) setBufferObjectAt swaps in a whole GPU BufferObject (hot-swap) rather than re-uploading vertex data.
- (ev_005) setBufferAtAsync is a non-blocking async version of setBufferAt with a completion callback.

## Limitations
- Tracks the `main` branch, which is a moving target; exact signatures and defaults may drift between Filament releases.
- Header doc comments describe API intent; full runtime semantics (alignment enforcement, backend behavior) live in the .cpp and backend layers, not this header.
