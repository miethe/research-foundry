# Source Card: three.js Docs — BufferAttribute

## Summary
The official three.js API reference for the BufferAttribute class documents the partial-update path for GPU buffers. An attribute holds typed geometry data (positions, indices, normals, colors, UVs, custom attributes); modifying its array and setting needsUpdate = true re-uploads it. The updateRanges array (populated by addUpdateRange(start, count) and reset by clearUpdateRanges()) restricts the re-upload to specific sub-ranges, and the usage property / setUsage(value) signal whether the buffer is static or dynamic for driver optimization.

## Key evidence
- (ev_001) addUpdateRange(start, count) queues a sub-range of the data array for GPU update.
- (ev_002) updateRanges enables re-uploading only some vector components (e.g., color) and is filled via addUpdateRange().
- (ev_003) clearUpdateRanges() resets the queued update ranges.
- (ev_004) needsUpdate = true flags the attribute as changed so it is re-sent to the GPU.
- (ev_005) usage (default StaticDrawUsage; DynamicDrawUsage available) is set via setUsage(value) but is immutable after a buffer's first use.
- (ev_006) BufferAttribute stores per-attribute geometry data for efficient GPU passing.
