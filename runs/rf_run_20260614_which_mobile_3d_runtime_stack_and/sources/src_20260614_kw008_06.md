# Source Card: Thermion — open-source 3D rendering toolkit for Flutter and Dart (Filament-backed)

## Summary
Thermion is a first-party, open-source framework for building cross-platform 3D applications in Dart and/or Flutter, rendered by Google's Filament physically-based-rendering engine (v1.56.4). Its ThermionViewer API loads glTF models and KTX/PNG/JPEG textures, adds lights and skyboxes, manipulates the camera, and plays skeletal/morph-target animations across iOS, macOS, Android, Windows, and an experimental Web/WASM build. The toolkit is split into a Flutter rendering-surface plugin (thermion_flutter) and a platform-agnostic viewer package (thermion_dart).

## Key evidence
- (ev_001) Thermion is a framework for cross-platform 3D apps in Dart and/or Flutter.
- (ev_002) ThermionViewer exposes an API for 3D scenes powered by the Filament rendering engine.
- (ev_003) The viewer loads/manipulates entities, lights, skyboxes, and plays skeletal/morph animations.
- (ev_004) Supported platforms: iOS, macOS, Android, Windows, and Web/WASM.
- (ev_005) Ingests glTF models plus KTX/PNG/JPEG textures; supports skinning and morph animations.
- (ev_006) Web support is experimental and requires manual compilation to WASM (Emscripten).
- (ev_007) Split into thermion_flutter (Flutter surface plugin) and thermion_dart (viewer logic).

## Limitations
- No publication/last-updated date is visible on the docs site; version pinned to Filament v1.56.4 at access time.
- Web support is self-described as experimental and not turnkey (manual WASM compile required).
- The source-brief claims that Thermion "supersedes flutter_filament" and the related "interactive_3d" plugin (Filament/Android, SceneKit/iOS) could NOT be confirmed in the actual source text and are therefore omitted from this card.
