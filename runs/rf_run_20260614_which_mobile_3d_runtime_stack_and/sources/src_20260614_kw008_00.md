# Source Card: google/filament — Real-time physically based rendering engine (official repo)

## Summary
The official google/filament GitHub repository establishes Filament as a real-time physically based rendering (PBR) engine built from a single C++ codebase that targets Android, iOS, Linux, macOS, Windows, and WASM/WebGL. It exposes multiple graphics backends (OpenGL/ES, Metal, Vulkan 1.0, WebGPU, WebGL 2.0) and ships gltfio, a native glTF 2.0 loader supporting Draco mesh compression and PBR material extensions. The engine is explicitly designed for a small, efficient mobile footprint and is actively maintained, with release v1.71.6 published 2026-06-10.

## Key evidence
- (ev_001) Filament is a single-codebase real-time PBR engine spanning Android, iOS, Linux, macOS, Windows, and WASM.
- (ev_002) It is explicitly designed to be as small and efficient as possible on Android (lightweight mobile posture).
- (ev_003) Backends cover OpenGL/ES, Metal, Vulkan 1.0, WebGPU, and WebGL 2.0 across desktop, mobile, and web.
- (ev_004) gltfio is a native glTF 2.0 loader supporting KHR_draco_mesh_compression, KHR_materials_clearcoat, and KHR_materials_transmission.
- (ev_005) Latest release v1.71.6 on 2026-06-10 is a freshness/active-maintenance signal.

## Limitations
- README documents stated design goals and capabilities, not independently measured performance; "small and efficient" is a vendor posture claim, not a benchmark. Backend/platform support matrices may shift between releases.
