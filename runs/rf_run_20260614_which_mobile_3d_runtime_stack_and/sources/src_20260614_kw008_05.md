# Source Card: LowLevelMesh | Apple Developer Documentation (RealityKit)

## Summary
This is Apple's first-party RealityKit reference for `LowLevelMesh`, a container for vertex data that lets an app supply its own custom vertex format and create or update meshes frequently. The API is available from iOS 18 / iPadOS 18 / macOS 15 / visionOS 2 onward, supports both CPU (Swift) and GPU (Metal compute shader) mesh updates, and exposes nested types (Attribute, Layout, Part) for controlling buffer layout and per-part material assignment. A `MeshResource` built from a `LowLevelMesh` retains the mesh and reflects changes at render time, enabling interactive, rebuild-free dynamic playback.

## Key evidence
- (ev_001) LowLevelMesh is a container for vertex data to create and update meshes using a custom format.
- (ev_002) Requires iOS 18 / iPadOS 18 / macOS 15 / visionOS 2 (and tvOS 26) -- the deployment-target floor for KnitWit features depending on it.
- (ev_003) Mesh data can be updated on the CPU (Swift) or GPU (Metal compute shaders) for frequent/dynamic updates.
- (ev_004) MeshResource retains the LowLevelMesh and reflects changes when the renderer updates -- no entity rebuild needed.
- (ev_005) The Part sub-type maps a primitive range to a material index, enabling per-region highlight/recolor.
- (ev_006) Attribute and Layout sub-types control vertex buffer memory layout and shader-attribute mapping.
