# Source Card: Bring your SceneKit project to RealityKit — WWDC25 Session 288

## Summary
In WWDC25 Session 288, Apple software engineer Max Cobb announces that Apple is officially (but softly) deprecating SceneKit across all platforms, with SceneKit entering maintenance mode (critical bug fixes only) while Apple steers new native 3D development to RealityKit. The session contrasts the two engines: RealityKit is built on the open USD/USDZ standard (Pixar, 2012) and uses an Entity Component System, whereas SceneKit serializes proprietary SCN files and uses a node-based scene graph. It also documents how to convert existing assets to USD/USDZ via Xcode export or the xcrun scntool CLI. This is the canonical first-party source for the SceneKit deprecation and the recommended migration path for iOS-native 3D runtimes.

## Key evidence
- (ev_001) Apple officially deprecated SceneKit across all platforms at WWDC25.
- (ev_002) The deprecation is soft: existing apps keep working, no rewrite required, but SceneKit is not recommended for new work.
- (ev_003) SceneKit is in maintenance mode — critical bug fixes only, no new features or optimizations.
- (ev_004) RealityKit supports visionOS, iOS, macOS, iPadOS, and (new at WWDC25) tvOS.
- (ev_005) RealityKit is built on the open USD standard (Pixar, 2012); SceneKit uses proprietary SCN files.
- (ev_006) RealityKit uses an Entity Component System vs SceneKit's node-based scene graph.
- (ev_007) Existing SceneKit/DCC assets must be converted to USD/USDZ (Xcode export or xcrun scntool CLI).

## Limitations
- glTF is NOT mentioned anywhere in this session; the source key-point claim about "no glTF support" and the need to convert glTF to USD is not supported by the transcript and was omitted.
- The source key-point about "per-entity components for highlight/recolor" is not stated in the transcript (the session lists components such as AnimationLibraryComponent, DirectionalLightComponent, AudioLibraryComponent, and ParticleEmitterComponent, but does not discuss highlight/recolor components); that claim was omitted.
- This is a vendor (Apple) developer session; positioning of RealityKit as the recommended successor reflects Apple's first-party perspective.
