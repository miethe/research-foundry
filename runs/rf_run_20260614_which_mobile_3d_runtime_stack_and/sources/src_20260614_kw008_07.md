# Source Card: react-native-filament — React Native wrapper for Google Filament (Margelo)

## Summary
react-native-filament (by Margelo GmbH) is a first-party React Native wrapper around Google's Filament real-time physically based rendering (PBR) engine, exposing 3D rendering through React components. It renders natively on the GPU (Metal on iOS, OpenGL/Vulkan on Android) on separate threads, adds only ~4MB to the downloadable app, and supports direct loading of .glb (binary glTF) assets, with conversion required for other formats. The maintainer reports it is used in production apps with millions of users; the latest release at access time is v1.11.0 (2026-05-27).

## Key evidence
- (ev_001) react-native-filament renders 3D graphics in a React Native app via React components, wrapping Filament.
- (ev_002) The native dependency adds only ~4MB to the downloadable app size.
- (ev_003) Native GPU rendering uses Metal on iOS and OpenGL/Vulkan on Android.
- (ev_004) Filament is a native C++ PBR engine; the library is GPU-accelerated by Metal and OpenGL/Vulkan.
- (ev_005) Rendering runs on separate threads (off the JS/UI thread).
- (ev_006) Maintainer claims production use in apps with millions of users and high stability.
- (ev_007) Only .glb is directly loadable; gltf/obj/FBX must be converted to .glb.

## Limitations
- Production-usage, stability, and threading statements are vendor self-reported (primary but promotional).
- Version note: ecosystem summaries cite v1.9.0, but the live GitHub repo shows v1.11.0 (2026-05-27) at access time; the ~4MB figure is the maintainer's stated number, not an independent measurement.
- Format support is narrow: direct loading is limited to .glb; the "compatible with glTF and other standard formats" framing requires conversion in practice.
