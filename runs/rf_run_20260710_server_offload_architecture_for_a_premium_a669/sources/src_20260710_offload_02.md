# Source Card: OpenAI API Pricing

## Summary
OpenAI's first-party API pricing page publishes per-1M-token list prices for the gpt-5.6 line and prior models across standard, batch, flex, and priority tiers. The gpt-5.6 tier spans luna (cheap/fast, $1/$6 in/out), terra (workhorse, $2.50/$15), and sol (frontier, $5/$30), with cached input billed at roughly 10% of standard input. Built-in tools (web search, file search, containers) and a 10% data-residency uplift are priced separately.

## Key evidence
- (ev_001) gpt-5.6-terra: $2.50/1M input, $0.25 cached, $15.00/1M output (standard tier).
- (ev_002) gpt-5.6-luna: $1.00/1M input, $0.10 cached, $6.00/1M output (standard tier).
- (ev_003) gpt-5.6-sol: $5.00/1M input, $0.50 cached, $30.00/1M output (standard tier).
- (ev_004) Non-preview web search on gpt-4o-mini / gpt-4.1-mini bills search content as a fixed 8,000-input-token block per call.
- (ev_005) Batch tier halves standard prices (terra batch = $1.25/$7.50), i.e. a 50% batch discount.
- (ev_006) Data-residency (regional processing) endpoints add a 10% price uplift for eligible post-2026-03-05 models.

## Limitations
- List prices only; excludes negotiated/enterprise and volume rates. No visible publication date or author byline on the page; the source key-point's "mini models" web-search framing is narrower in the doc, applying specifically to gpt-4o-mini and gpt-4.1-mini with the non-preview tool. Cached-input value is inferred as the second numeric column of each pricing row.
