# Source Card: Generating content and performing tasks with Foundation Models

## Summary
This first-party Apple Developer document describes how to prompt the on-device Foundation Models system LLM and states its hard context limit: up to 4,096 tokens per session, shared across instructions, prompts, and outputs. It defines the token-to-character ratio, specifies the `LanguageModelError.contextSizeExceeded` failure mode, and prescribes chunking long data across separate sessions. It also documents guided generation, which constrains output to a developer-defined Swift data structure.

## Key evidence
- (ev_001) The on-device system model supports a context window of up to 4,096 tokens per session.
- (ev_002) A token maps to ~3-4 characters in English/Spanish/German, and one token per character in Japanese/Chinese/Korean.
- (ev_003) Instructions + all prompts + all outputs jointly count toward the 4,096-token budget.
- (ev_004) Data exceeding the window must be chunked, processed per-session, and recombined.
- (ev_005) Guided generation yields a developer-defined custom Swift data structure rather than raw strings.

## Limitations
- The Foundation Models framework content is Beta-labeled and Apple-version-mutable; token limits and API surface may change across OS releases. The document is undated beyond a 2025 copyright/publication year.
