# Source Card: Prompt caching - Claude Platform Docs

## Summary
Anthropic's official Claude Platform documentation on prompt caching establishes the pricing model and prefix-referencing behavior of the feature. Cache reads are cheap (0.1x base input), while cache writes carry a premium (1.25x for the default 5-minute TTL, 2x for the optional 1-hour TTL). The cache references the entire request prefix (tools, system, messages) up to a `cache_control` breakpoint, so keeping that prefix stable maximizes hit rate and cost savings.

## Key evidence
- (ev_001) Cache read (hit) tokens are priced at 0.1x the base input token price.
- (ev_002) 5-minute cache write tokens cost 1.25x base input; 1-hour cache write tokens cost 2x base input.
- (ev_003) Default cache TTL is 5 minutes, refreshed for free on each hit; a 1-hour TTL is available at extra cost.
- (ev_004) Claude Haiku 4.5 example: $1/MTok base input, $1.25 5m-write, $2 1h-write, $0.10 cache-hit, $5 output.
- (ev_005) Caching references the entire prefix (tools + system + messages) up to the `cache_control` breakpoint; a stable prefix maximizes hit rate.

## Limitations
- Pricing figures are model- and date-specific (the example is Claude Haiku 4.5); verify current rates against the live pricing page before relying on exact numbers.
