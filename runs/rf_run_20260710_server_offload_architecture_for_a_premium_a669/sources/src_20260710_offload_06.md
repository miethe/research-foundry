# Source Card: Updates to Apple's On-Device and Server Foundation Language Models

## Summary
This first-party Apple Machine Learning Research post describes Apple's 2025 foundation-model stack: a compact ~3B-parameter on-device model paired with a larger mixture-of-experts server model whose architecture is tailored for Private Cloud Compute. It documents the efficiency techniques behind each tier (2-bit quantization and KV-cache reduction on-device, a parallel-track MoE design on the server) and frames the on-device model as a tool for summarization, extraction, and structured text tasks rather than a general-knowledge chatbot.

## Key evidence
- (ev_001) Compact ~3B on-device model plus a MoE server model tailored for Private Cloud Compute.
- (ev_002) Server model uses a parallel-track MoE (PT-MoE) design reducing synchronization overhead.
- (ev_003) On-device architecture cuts KV cache memory by 37.5% and improves time-to-first-token.
- (ev_004) On-device model compressed to 2 bits per weight via Quantization-Aware Training.
- (ev_005) On-device model optimized for summarization, entity extraction, understanding, refinement.
- (ev_006) On-device model is explicitly not designed as a general-knowledge chatbot.
- (ev_007) Apple does not use users' private personal data or interactions to train the models.

## Limitations
- The page's Private Cloud Compute claims are stated by association (architecture "tailored for" PCC) and link out to a separate security blog; specific stateless/verifiable/"never stored" privacy guarantees are not detailed verbatim in this post and should be cited from the PCC security page instead. All performance and privacy figures are vendor self-reported.
