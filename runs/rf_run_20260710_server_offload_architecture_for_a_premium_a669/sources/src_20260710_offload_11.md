# Source Card: GPT-5 mini model — OpenAI API

## Summary
The official OpenAI API reference for GPT-5 mini describes a reasoning model tuned for cost-sensitive, low-latency, high-volume workloads, with a 400K-token context window and support for the batch endpoint plus structured outputs, function calling, and web search. For new high-volume/low-latency work OpenAI now recommends its GPT-5.6 Terra tier instead. The live page carries no per-token pricing, so the specific price figures asserted in the swarm's source metadata are unverified here.

## Key evidence
- (ev_001) Positioned for cost-sensitive, low-latency, high-volume workloads.
- (ev_002) OpenAI recommends GPT-5.6 Terra for most new high-volume/low-latency workloads.
- (ev_003) Supports the batch endpoint for asynchronous/offloaded workloads.
- (ev_004) 400K-token context window; 272K max input / 128K max output.
- (ev_005) Supports structured outputs, function calling, and web search — good for offloaded structuring tasks.
- (ev_006) Reasoning model; snapshot gpt-5-mini-2025-08-07; knowledge cutoff 2024-05-31.

## Limitations
- The pricing figures ($/M tokens, cached input, batch rates) and the "GPT-5.4 mini/nano released March 2026" claim from the ingest metadata are NOT present on this page and are unconfirmed; pricing lives on a separate OpenAI pricing page.
- Current snapshot dates to 2025-08-07; OpenAI steers new workloads to the newer GPT-5.6 line.
