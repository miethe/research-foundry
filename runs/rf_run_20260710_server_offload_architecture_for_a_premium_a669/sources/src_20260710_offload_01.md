# Source Card: Modeling Crochet Patterns with a Force-directed Graph Layout

## Summary
This peer-reviewed Eurographics Expressive/WICED 2025 paper (Greer & Mould, Carleton University) presents a modeling tool that converts a written amigurumi crochet pattern in row-by-stitch notation into a pattern graph — stitches as nodes, physical connections as edges — and then computes a force-directed graph layout to produce a 3D model accurate in shape and size to the real object. The layout is framed as a non-linear least-squares optimization minimizing edge-length and curvature energy functions, solved iteratively with the Ceres Solver; the authors note the iterative solver is by far the most time-expensive part of the pipeline.

## Key evidence
- (ev_001) Written pattern is translated into a graph and laid out to yield a 3D model matching the crocheted shape/size.
- (ev_002) Stitches are nodes and physical connections between stitches are edges of the pattern graph.
- (ev_003) The force-directed layout is a non-linear least-squares optimization minimizing two energy functions.
- (ev_004) The iterative linear solver is the dominant compute cost, flagged as a general-audience usability concern.
- (ev_005) Builds on Isenburg et al.'s planar-graph inflation (assign edge lengths + minimize curvature to maximize volume).
- (ev_006) Solved with the Ceres Solver library at a static blend weight lambda = 0.65 for most pieces.

## Limitations
- Scope is restricted to amigurumi with single-crochet-like stitches and stuffed (not flattened) pieces; the authors state these restrictions are "too narrow for modeling intermediate or advanced patterns." Compute-vs-phone-budget framing is an inference from the paper's solver-cost note, not a stated benchmark.
