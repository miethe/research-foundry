# Source Card: Engineering Blog: Journey to superhuman performance on scientific tasks

## Summary
This first-party FutureHouse engineering blog documents the design rationale and benchmark behavior of PaperQA2, an agentic RAG system for scientific literature QA. It establishes that PaperQA2 deliberately trades cost and latency for accuracy, evaluates on the 200-question LitQA2 set, and shows that an agent loop (averaging >4 tool calls and 1.26 searches per question, with selective citation traversal on ~46% of questions) yields significant gains in accuracy and answer recall over a non-agentic baseline.

## Key evidence
- (ev_001) PaperQA2's design philosophy explicitly de-prioritizes cost and latency to maximize accuracy.
- (ev_002) PaperQA2 was evaluated on LitQA2, a set of 200 expert-crafted multiple-choice questions.
- (ev_003) The agent-based system improved both accuracy and answer recall over a non-agentic baseline.
- (ev_004) The citation traversal tool is used on only ~46% of questions (selective, query-dependent tool use).
- (ev_005) The agent averages more than 4 tool calls per question.
- (ev_006) The agent averages 1.26 searches per question.

## Limitations
- Vendor/first-party engineering blog (FutureHouse develops PaperQA2); statistics are self-reported and not independently reproduced here. Figures reflect the 2024-09 benchmark configuration and may not generalize to later versions or other evaluation sets.
