# Source Card: Tailored Research - GPT Researcher Documentation

## Summary
This official GPT Researcher documentation page explains how to scope and steer a research run through its Python API. Research can be confined to a caller-supplied list of URLs (`source_urls` with `complement_source_urls=False`), redirected to local documents (`DOC_PATH` + `report_source="local"`), combined in a `"hybrid"` mode, or run over LangChain `Document` instances — all through the same async pattern of `GPTResearcher(...)`, `conduct_research()`, then `write_report()`. Agent behavior and report layout can be customized via the `query` prompt paired with the `custom_report` report type.

## Key evidence
- (ev_001) source_urls constrains research to a caller-supplied list of URLs.
- (ev_002) complement_source_urls defaults to False, restricting research to only the provided URLs.
- (ev_003) A custom prompt as `query` plus `report_type="custom_report"` steers research direction and report layout.
- (ev_004) Local-document research via DOC_PATH and report_source="local"; supported input formats include PDF, CSV, Excel, Markdown, PowerPoint, Word, and plain text.
- (ev_005) report_source="hybrid" combines web sources and local documents in one run.
- (ev_006) Accepts a list of LangChain Document instances (report_source="langchain_documents").
- (ev_007) Uniform async API: instantiate, conduct_research(), write_report().

## Limitations
- The page carries no explicit publication or last-updated date.
- The page does NOT document report output/export formats (e.g., PDF, Word, JSON, CSV). The listed file formats refer to supported local *input* documents, not exportable report outputs — do not cite this page for output-export claims.
