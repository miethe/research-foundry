# Source Card: Future-House/paper-qa (PaperQA2): high-accuracy RAG over scientific literature

## Summary
PaperQA2 is an open-source (Apache-2.0) Python package from Future House for high-accuracy retrieval-augmented generation over scientific documents, including PDFs, text files, Microsoft Office documents, and source code. It exposes a simple Python API (`from paperqa import Settings, ask`, plus an async `agent_query()`) that returns an answer object with `formatted_answer`, `answer`, `question`, and `context` fields, and emits in-text citation keys like `(pqac-abcd1234)` referencing specific source passages. It supports all LiteLLM-compatible models (defaulting to `gpt-4o-2024-11-20`) and uses Semantic Scholar, Crossref, and Unpaywall for paper search and metadata.

## Key evidence
- (ev_001) PaperQA2 does high-accuracy RAG over PDFs, text, Office docs, and source code, focused on scientific literature.
- (ev_002) The answer object exposes formatted_answer, answer, question, and context (passage summaries).
- (ev_003) In-text citations use keys like (pqac-abcd1234) referencing specific source passages.
- (ev_004) Customization interface defaults to support for all LiteLLM-compatible models.
- (ev_005) Default LLM across summary_llm, llm, and agent_llm is OpenAI's gpt-4o-2024-11-20.
- (ev_006) Paper search/metadata is provided via Semantic Scholar, Crossref, and Unpaywall.
- (ev_007) Python API entry point is `from paperqa import Settings, ask`, with an async `agent_query()` alternative.

## Limitations
- The README documents stated behavior, defaults, and feature claims for the package; it is not an independent benchmark. Reported "superhuman" accuracy claims trace to a separate 2024 paper rather than the README text itself, so accuracy assertions should be cited to that paper, not this card.
