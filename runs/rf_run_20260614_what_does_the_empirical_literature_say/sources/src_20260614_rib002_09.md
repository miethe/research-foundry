# Source Card: Future-House/paper-qa (PaperQA2): high-accuracy RAG over scientific literature

## Summary
PaperQA2 is Future House's open-source (Apache-2.0) retrieval-augmented generation system for high-accuracy question answering over scientific literature. Its README documents a three-phase pipeline (Paper Search, Gather Evidence, Generate Answer) that produces answers with passage-level in-text citation keys like `(pqac-abcd1234)`, exposed via the answer object's `formatted_answer` and `context` attributes. The maintainers claim superhuman performance on scientific tasks such as question answering, summarization, and contradiction detection, with default models gpt-4o-2024-11-20 and text-embedding-3-small.

## Key evidence
- (ev_001) PaperQA2 emits passage-level in-text citation keys formatted like (pqac-abcd1234), each tied to a specific retrieved passage.
- (ev_002) The answer object exposes a formatted_answer with integrated citations alongside a context attribute holding passage summaries.
- (ev_003) Phase 1 (Paper Search) retrieves candidate papers via an LLM-generated keyword query, then chunks, embeds, and adds them to state.
- (ev_004) Phase 2 (Gather Evidence) embedding-ranks chunks, builds scored contextual summaries, then re-scores and selects the most relevant via an LLM.
- (ev_005) PaperQA2 is documented as achieving superhuman performance on scientific-literature tasks including question answering, summarization, and contradiction detection.
- (ev_006) PaperQA2 is Apache-2.0 licensed and self-hostable, with local model backends such as ollama and llamafile supported.

## Limitations
- Performance claims (e.g., "superhuman", "exceeded human performance") are maintainer self-reported in the project README and link to project-affiliated benchmark pages (e.g., LitQA2 / paper.wikicrow.ai); they are not independent third-party evaluations. Default model identifiers and the latest release version (v2026.03.18) are point-in-time and may change with new releases.
