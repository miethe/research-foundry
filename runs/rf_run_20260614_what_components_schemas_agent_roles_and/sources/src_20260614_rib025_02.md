# Source Card: Future-House/paper-qa (PaperQA2) GitHub README

## Summary
The PaperQA2 README is the official project documentation for Future House's high-accuracy retrieval-augmented generation (RAG) library for answering questions from scientific documents with citations. It establishes PaperQA2's agentic architecture (Paper Search, Gather Evidence, Generate Answer tools invokable in any order), its core RCS (reranking and contextual summarization) retrieval mechanism, its citation key format, the structure of its answer object, its default model configuration, and its integration with external scholarly metadata providers.

## Key evidence
- (ev_001) PaperQA2's agentic workflow is organized as three tools — Paper Search, Gather Evidence, and Generate Answer — invokable in any order by a language agent.
- (ev_002) The core retrieval mechanism is RCS: document metadata-aware embeddings plus LLM-based re-ranking and contextual summarization.
- (ev_003) Gather Evidence creates a query-contextual scored summary of each chunk, then uses an LLM to re-score and select the most relevant summaries.
- (ev_004) In-text citations use a key pattern of the form (pqac-abcd1234).
- (ev_005) The answer object exposes formatted_answer, answer, question, and context attributes.
- (ev_006) Defaults to OpenAI's gpt-4o-2024-11-20 across summary_llm, llm, and agent_llm roles.
- (ev_007) The LLM interface uses the lmi package, which in turn uses litellm to support many LLM providers.
- (ev_008) Integrates external metadata providers Semantic Scholar, Crossref, and Unpaywall.
