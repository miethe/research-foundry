# Source Card: stanford-oval/storm GitHub README

## Summary
STORM (Synthesis of Topic Outlines through Retrieval and Multi-perspective Question Asking) is a Stanford OVAL LLM system that writes Wikipedia-like, fully cited reports from scratch by running an Internet-research pre-writing stage followed by an article-writing stage. The engine has four modules (Knowledge Curation, Outline Generation, Article Generation, Article Polishing); pre-writing discovers perspectives and simulates a Wikipedia-writer/topic-expert conversation grounded in web sources. Co-STORM extends this with a collaborative discourse protocol (LLM experts, a Moderator, and a Human user) plus a dynamic mind map, all built modularly on DSPy with LiteLLM integration and pluggable retrievers; the project is MIT-licensed (v1.1.0, Jan 23, 2025; Co-STORM in v1.0.0, Sept 2024).

## Key evidence
- (ev_001) STORM is an LLM system writing cited Wikipedia-like articles via a two-stage pre-writing then writing pipeline.
- (ev_002) The STORM engine has four modules: Knowledge Curation, Outline Generation, Article Generation, Article Polishing.
- (ev_003) Pre-writing uses perspective-guided question asking and a simulated Wikipedia-writer/topic-expert conversation grounded in Internet sources.
- (ev_004) Co-STORM adds a collaborative discourse protocol with three roles: LLM experts, a Moderator, and a Human user.
- (ev_005) Co-STORM maintains a dynamic mind map organizing information into a hierarchical concept structure for a shared conceptual space.
- (ev_006) STORM and Co-STORM are built modularly on DSPy with support for many retrievers (You, Bing, Vector, Serper, Brave, SearXNG, DuckDuckGo, Tavily, Google, Azure AI Search).
- (ev_007) LiteLLM integration arrived in knowledge-storm v1.1.0 (Jan 23, 2025); Co-STORM was integrated in v1.0.0 (Sept 2024).
