# Source Card: Nick's Research Foundry MVP Spec

## Summary
The Research Foundry MVP spec (authored 2026-06-12 by Nick Miethe) defines a Markdown/YAML-first, evidence-first research control plane and codifies its design principles. It establishes four-way claim labeling, a disposable-swarm/durable-bundle model, cost-tiered model routing, and runtime governance gating, and maps external tools (GPT Researcher, PaperQA2, LiteLLM, Agent Council) to MVP roles while prioritizing the first CLI commands to implement.

## Key evidence
- (ev_001) Four-way claim labeling discipline: supported (mapped to source cards), labeled inference, labeled speculation, or explicitly unresolved.
- (ev_002) The swarm is disposable; the durable asset is the evidence bundle (source cards, claims, claim ledger, report, reviews, telemetry).
- (ev_003) Cheap/free models extract; expensive models synthesize.
- (ev_004) Governance is a runtime gate: keys, sensitive sources, model routing, and writeback targets checked before execution.
- (ev_005) External tool mapping: GPT Researcher, PaperQA2 (agentic RAG with in-text citations), LiteLLM (cost-based routing + budgets), Agent Council (Approve/Concern/Block).
- (ev_006) MVP CLI surface prioritized: rf init, rf capture, rf plan, rf verify, rf writeback first.

## Limitations
- Self-authored internal design document, not independent validation; descriptions of external tools cite upstream docs as of mid-2026 and may drift.
