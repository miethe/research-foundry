# Source Card: CrochetBench: Can Vision-Language Models Move from Describing to Doing in the Crochet Domain?

## Summary
CrochetBench is an academic benchmark (Univ. of Notre Dame) that evaluates whether vision-language models can move beyond describing crochet to generating executable procedures, using a dataset of 6,085 Yarnspirations patterns (55 categories, ~98% image coverage) released only as GPT-generated structured JSON annotations to respect copyright. It adopts the CrochetPARADE DSL as an intermediate representation with syntactic/structural validation plus execution and DINO image-similarity scoring. Results show current VLMs largely fail at executable generation — only 5-8% project-level valid programs and DINO similarity of just 0.10-0.17 — establishing that an explicit executable IR with a validator is essential and ML alone is unreliable.

## Key evidence
- (ev_001) CrochetBench: 6,085 crochet patterns across 55 project categories from Yarnspirations, ~98% image coverage.
- (ev_002) Authors release only GPT-generated structured JSON annotations plus reference URLs and scripts, not raw pattern files (copyright).
- (ev_003) CrochetPARADE DSL is the intermediate representation; it parses input and checks syntactic/consistency errors (e.g., mismatched stitch counts, impossible attachments).
- (ev_004) Functional evaluation uses DINO similarity between the target product image and the rendering from each model's executable program.
- (ev_005) Even the strongest models produce only 5-8% executable programs at the project level.
- (ev_006) DINO similarity for valid renderings stays uniformly low (0.10-0.17), far below a faithful-reproduction threshold.
- (ev_007) Step-level validity is under 15% even in the first two steps for most models.
- (ev_008) Performance sharply drops from surface similarity to executable correctness, exposing limits in long-range symbolic and 3D-aware reasoning.

## Limitations
- Preprint (arXiv, Nov 2025; revised Feb 2026); results reflect the specific model versions/prompting tested and may shift as models improve. Some detailed schema/error-taxonomy fields are sourced from appendix tables rather than verbatim prose.
