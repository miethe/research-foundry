# Source Card: KnitWit B1 Workstream Report (project prior-art synthesis)

## Summary
This internal KnitWit B1 prior-art report synthesizes the academic and hobbyist landscape for crochet pattern <-> 3D mapping and recommends a de-scoped inverse baseline, "Amigurumi Graph Lite" (Geodesic Row Planner), that generates rows around a seed axis and sets stitch count per row from local circumference versus gauge. It catalogs four inverse-mapping families (deterministic geometry mapping, primitives/templates, optimization, ML-assisted) with capability matrices, and argues surfaces of revolution are an analytically solved low-risk first milestone while general-mesh AmiGo-style methods are higher risk (no public code, Matlab/C++, seconds-to-minutes). It also fixes concrete crochetability heuristics and a prioritized set of de-risk experiments.

## Key evidence
- (ev_001) The recommended inverse baseline 'Amigurumi Graph Lite' generates rows around a seed axis, sizing each row from mesh circumference vs gauge, scoped to single-part convex meshes first.
- (ev_002) Concrete heuristic: keep each row within +/-1 of the smooth ratio and no more than ~2x stitches round-to-round (ideally less), splitting big jumps.
- (ev_003) Deterministic geometry-mapping family traces a continuous geodesic/iso-curve path spiraling the shape with curvature-driven inc/dec.
- (ev_004) Four-family inverse taxonomy with capability matrices: deterministic mapping vs primitives/templates vs optimization vs ML.
- (ev_005) For surfaces of revolution the inverse mapping is "essentially a solved problem (analytically)" - lowest-risk first milestone.
- (ev_006) AmiGo-style general meshes are higher risk: no public code, Matlab/C++, seconds-to-minutes runtimes.
- (ev_007) Max-curvature / rate-of-increase de-risk experiment: a >~50% single-round increase => not reliably crochetable, must segment.
- (ev_008) Seed/seam placement effect on symmetry and round-trippability through the forward visualizer are key open experiments.
