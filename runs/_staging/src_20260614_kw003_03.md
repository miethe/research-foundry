# Source Card: CrochetBench: Can Vision-Language Models Move from Describing to Doing in the Crochet Domain?

## Summary
CrochetBench is an arXiv benchmark (Li et al., Notre Dame, Nov 2025; v2 Feb 2026) that evaluates whether vision-language models can move from describing crochet to generating executable procedures, using the CrochetPARADE DSL as an execution-validated intermediate representation. Across four progressive tasks, the authors find that models achieve moderate stitch recognition (F1 ~57-61%) and instruction grounding but collapse on procedural synthesis, producing only ~5-8% executable programs at the project level. The central result is a measurable gap between surface-level understanding and executable precision: syntactic validity does not imply correct procedural structure.

## Key evidence
- (ev_001) Four progressive tasks (recognition, selection, generation, DSL translation) with a 119-item step-level and 100-item project-level CrochetPARADE translation set.
- (ev_002) Even the strongest models produce only ~5-8% executable programs on project-level DSL translation.
- (ev_003) Stitch recognition F1 only ~57-61% (Claude Sonnet 4 60.94%, GPT-4o 58.01%, Gemini 2.5 Flash-Lite 56.83%).
- (ev_004) Performance sharply decreases moving from surface similarity to executable correctness, revealing limits in long-range symbolic reasoning.
- (ev_005) Compilable DSL output rarely matches the intended pattern: syntactic validity does not imply correct procedural structure.
- (ev_006) Fluent, crochet-like descriptions fail to preserve the algorithmic structure required for faithful pattern synthesis.
- (ev_007) Core takeaway: a gap between surface-level understanding and executable precision in creative domains.

## Limitations
- arXiv preprint (v1 Nov 2025, v2 Feb 2026), not yet peer-reviewed in a venue; project-level DSL evaluation set is small (100 items; step-level 119). Per-model F1 values (ev_003) are read from the results table and carried as the summary's numeric claim; no short verbatim excerpt fit within the quote limit, so quote is null for that point.
