# Source Card: WebGL best practices — MDN Web Docs

## Summary
MDN's WebGL best-practices reference catalogs vendor-neutral performance guidance for the WebGL API. It emphasizes minimizing CPU-side overhead by batching geometry into fewer/larger draw calls, avoiding state changes that force browser revalidation (mutating VAOs, FBO attachment bindings), and managing GPU object lifetimes explicitly. The guidance is directly applicable to high-sprite-count 2D rendering on mobile web.

## Key evidence
- (ev_001) Collapsing many objects into fewer, larger draw calls generally improves performance; e.g. paint 1000 sprites with a single drawArrays()/drawElements() call.
- (ev_002) Static unchanging VAOs are faster than mutating a VAO per draw because browsers cache fetch limits for unchanged VAOs but revalidate on change.
- (ev_003) Almost any change to FBO attachment bindings invalidates framebuffer completeness, so set up hot framebuffers ahead of time.
- (ev_004) Delete GPU objects eagerly rather than relying on GC, since the API handle only refers to the underlying object.
- (ev_005) Texture atlasing and degenerate triangle strips keep discontinuous geometry in a single draw call.

## Limitations
- General-purpose WebGL guidance, not specific to any particular engine, framework, or mobile GPU; benchmarks/quantified speedups are not provided, so claimed performance gains are directional rather than measured.
