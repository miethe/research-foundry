# Source Card: Visual Knitting Machine Programming

## Summary
This SIGGRAPH 2019 / ACM TOG paper (Narayanan, Wu, Yuksel, McCann) introduces an "augmented stitch mesh" data structure for visual programming of industrial knitting machines. Each mesh face carries a low-level knitting program fragment, and directed edge labels both track inter-face dependencies and act as a type signature (input/output loop and yarn counts) for that fragment. The system generates knittable meshes from 3D models, supports knittability-preserving edits, and schedules the execution order and location of each face for machine production. It establishes the per-face stable-ID/type-signature model later adapted by 2020 crochet stitch-mesh work, evidencing cross-craft transfer.

## Key evidence
- (ev_001) Authorship and venue: Narayanan, Wu, Yuksel, McCann; ACM TOG Vol. 38 No. 4, Article 63, July 2019.
- (ev_002) The augmented stitch mesh stores low-level operations per-face and encodes face dependencies via directed edge labels.
- (ev_003) System generates knittable meshes from 3D models, edits preserve knittability, and execution order/location of each face is scheduled.
- (ev_004) Each face is a knitting-program fragment consuming loops/yarns from in edges and producing loops/yarns on out edges.
- (ev_005) Edge labels give a type signature (input/output loop and yarn counts) for each program fragment.
- (ev_006) Directed edge labels induce a directed graph on faces (with same-type in/out matching) used to detect dependency cycles.

## Limitations
- Scope is machine knitting (not crochet); the cross-craft transfer to crochet stitch-mesh work is established by the later (2020) paper that adapts this pipeline, not by this source itself.
