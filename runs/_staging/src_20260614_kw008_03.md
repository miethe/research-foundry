# Source Card: SceneView — 3D/AR Android View with ARCore and Google Filament (Kotlin successor)

## Summary
SceneView is a Kotlin/Jetpack-Compose Android library that integrates Google Filament (rendering) and ARCore (AR perception) into a unified 3D/AR view, positioned by its maintainers as the actively-developed successor to Google's archived Java Sceneform. It exposes a Filament-only `SceneView` for pure 3D plus an `ArSceneView`/`ARSceneView` that layers ARCore on top, making AR opt-in. For KnitWit's amigurumi preview this means a Filament-only 3D path can run without pulling in ARCore weight, while the same stack supports AR if needed later.

## Key evidence
- (ev_001) SceneView integrates ARCore and Google Filament into one Android 3D/AR view.
- (ev_002) The project explicitly frames itself as the replacement for Sceneform.
- (ev_003) AR is opt-in — `SceneView` is 3D-only and `ArSceneView` adds ARCore.
- (ev_004) Kotlin foundation lets the library track newer ARCore features faster.
- (ev_005) Canonical SDK tagline: a Jetpack Compose + Filament 3D/AR SDK for Android (also iOS/Web).
- (ev_006) Google archived Sceneform in 2021; SceneView descends from the maintained fork and is the Compose-native path forward.
- (ev_007) `SceneView` renders a Filament 3D viewport; `ARSceneView` is that plus ARCore tracking.

## Limitations
- The standalone doc page reflects an older 1.0.0-era framing ("Compose coming next"); the live GitHub repo shows the current Compose-native state (sceneview 4.x). No published date is on the doc page, so version/recency claims must be cross-checked against the repository. Some current-state quotes (ev_005–ev_007) are sourced from the project's canonical GitHub README rather than the legacy doc page.
