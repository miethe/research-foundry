# Source Card: CrochetPARADE (Crochet PAttern Renderer, Analyzer, and DEbugger) — Manual & Grammar

## Summary
CrochetPARADE is an author-built (Svetlin Tassev) open-source platform whose manual defines a custom formal grammar that parses and correctness-checks any user-provided crochet pattern before constructing a virtual 3D model. Its grammar makes stitch attachment explicit and validatable: stitches attach one-by-one to consecutive previous-row stitches by default, with an `@` operator for coordinate, stitch-type-counted, relative, and label-based attachment plus automatic turn accounting. The tool also flags overly loose/tight stitches (a tension/gauge acceptance signal), exports a 2D SVG that identifies each stitch by type/row/position and 3D files for Blender, runs entirely locally, and is GPLv3 (computation) / CC BY-NC-SA 4.0 (manual).

## Key evidence
- (ev_001) Custom grammar parses and checks patterns for correctness before 3D modeling, explicitly to remove plain-English ambiguity.
- (ev_002) Stitches attach one-by-one to consecutive previous-row/round stitches, with automatic turn accounting.
- (ev_003) `@` attachment operator supports coordinates, stitch-type counting, relative positions, and labels.
- (ev_004) Platform flags overly loose/tight stitches so users can swap before crocheting, reducing blocking (tension acceptance signal).
- (ev_005) Exports a 2D SVG identifying each stitch by type/row/position and 3D files importable into Blender.
- (ev_006) Computation is fully local but scales poorly: tens of thousands of stitches can take minutes or more.
- (ev_007) GPLv3 (with Graphviz exception) for computation; CC BY-NC-SA 4.0 for the manual/grammar description.
