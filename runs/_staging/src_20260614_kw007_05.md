# Source Card: Digital Crochet: Toward a Visual Language for Pattern Description

## Summary
Seitz, Rein, Lincke, and Hirschfeld (HPI; Onward! 2022) propose the first visual, domain-specific, graph-based language for representing crochet patterns, where nodes are stitches/insertion points and directed edges encode "previous", "insertion", and slip-stitch relations. The graph captures fabric structure and inter-stitch dependencies directly (not a linear stitch sequence), enabling a machine-checkable crochetability invariant, structural encodings of increases/decreases, and a unified "layer" abstraction over rows and rounds. A think-aloud user study with six professional crochet designers showed the language and a 2D/3D editor prototype can express both 2D and 3D patterns and surface ambiguities present in standard chart notation.

## Key evidence
- (ev_001) Pattern modeled as a directed graph: nodes = stitches/insertion points; edges = previous / insertion / slip-stitch relations.
- (ev_002) The graph encodes structure and dependencies (not a producing sequence), giving an unambiguous machine-readable IR.
- (ev_003) Crochetability is an edge-ordering invariant: a loop cannot use insertion points from a loop started later — i.e., no working into a loop before it exists.
- (ev_004) Increases = multiple incoming insert edges; decreases = multiple outgoing insert edges (concrete shaping signature).
- (ev_005) Rows and rounds are unified as "layers", stored as a per-node layer-number property.
- (ev_006) Designers wanted per-layer overviews (stitch count per layer, clear layer numbers), motivating per-layer complexity metrics.
- (ev_007) Linear/preview-driven editing made previously hidden chart ambiguities apparent during the user study.
- (ev_008) The editor can verify crochetability before an edit commits, ensuring stitch requirements by construction (validator-gated edit loop).

## Limitations
- Proof-of-concept editor prototype, not a production tool; force-based 3D layout is only a first approximation and lacks physical yarn modeling.
- Qualitative evaluation is small (6 professional designers recruited via myboshi); findings are experiential, not quantitative.
- Some techniques (yarn/color switching, copy-and-paste, non-linear editing) were unsupported at publication; layer counts and stitch-count-per-layer metrics were requested features, not yet implemented.
