# Source Card: API Terms of Use - Our House Rules | Etsy

## Summary
Etsy's official API Terms of Use is a binding contract governing all use of the Etsy API. It grants only a limited, non-sublicensable, non-transferable, non-exclusive and revocable license; imposes strict data-freshness and anti-caching rules (no listing content older than six hours; no storage beyond what is reasonably necessary); mandates data minimization; and prohibits facilitating unauthorized copying of members' products, photos or designs. Critically, Etsy Members (not Etsy) own the content they create, so the API conveys no right to redistribute designers' patterns; developers must secure IP rights independently and remain liable for any infringement.

## Key evidence
- (ev_001) Apps must not show listing content more than six hours stale, nor cache/store content longer than reasonably necessary to serve users; this bars durable local pattern storage.
- (ev_002) The API license is limited, non-sublicensable, non-transferable, non-exclusive, and revocable, solely for integrating applications.
- (ev_003) Prohibited Behavior #6 bans supporting, encouraging, or facilitating unauthorized downloading/copying/reproduction of members' products, photos, or designs, or IP infringement.
- (ev_004) Prohibited Behavior #17 enforces data minimization: request no more than the minimum data needed for the intended application.
- (ev_005) Reverse engineering and accessing internal/legacy APIs or data feeds is prohibited, as is creating extra API keys to circumvent rate limits.
- (ev_006) Etsy Members, not Etsy, own the content they create; the API grants no license to redistribute that content.
