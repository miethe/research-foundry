# Source Card: PaddleOCR-VL: Boosting Multilingual Document Parsing via a 0.9B Ultra-Compact Vision-Language Model

## Summary
PaddleOCR-VL is a 0.9B-parameter vision-language model (NaViT-style dynamic-resolution visual encoder + ERNIE-4.5-0.3B LLM) from Baidu's PaddlePaddle team, built for multilingual document parsing across 109 languages. The paper reports state-of-the-art OmniDocBench v1.5 results (overall 92.56) and claims it outperforms much larger models on document parsing, while emitting structured per-element output (tables, LaTeX formulas, charts, reading order). All metrics are vendor self-reported, and throughput figures are server-side (A100/vLLM), not an edge/phone budget.

## Key evidence
- (ev_001) 0.9B-param VLM: NaViT-style dynamic-resolution visual encoder + ERNIE-4.5-0.3B LLM.
- (ev_002) Supports 109 languages; recognizes text, tables, formulas, and charts.
- (ev_003) OmniDocBench v1.5: overall 92.56; text edit 0.035; Table TEDS 89.76 / TEDS-S 93.52; formula CDM 91.43; reading-order edit 0.043.
- (ev_004) Claims SOTA on page-level parsing and element-level recognition, reportedly beating larger general VLMs.
- (ev_005) Server-side throughput: 1.2241 pages/s, 1881.2 tokens/s on a single A100 via vLLM, ~43.7 GB GPU memory.
- (ev_006) Claims consistent advantages in processing speed and memory efficiency.
- (ev_007) Structured per-element output: LaTeX formulas, structured tables, charts as Markdown, reading-order prediction.

## Limitations
- All benchmark and throughput numbers are vendor self-reported (primary source authored by the model creators); not independently reproduced.
- Throughput (1.2241 pages/s, 1881.2 tokens/s, 43.7 GB) is measured on a single NVIDIA A100 via vLLM - a datacenter GPU budget, not an on-device/phone budget relevant to a mobile crochet-PDF use case.
- Quotes for ev_003 and ev_005 are set to null because the figures appear as table values rather than a short verbatim prose sentence; numbers are transcribed from Table 2 and Table 13.
