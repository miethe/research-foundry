# Source Card: BatchedMesh — three.js docs

## Summary
The official three.js API reference for `BatchedMesh`, a multi-draw batch-rendering mesh that draws many objects sharing one material but with differing geometries or transforms in fewer draw calls. It documents per-instance color and matrix APIs, frustum-culling and overdraw-sorting defaults, and the construction-time capacity limits plus runtime resize/repack methods relevant to a performant mobile WebGL stitch/round renderer.

## Key evidence
- (ev_001) BatchedMesh uses multi-draw batch rendering for many objects sharing a material to reduce draw calls and improve performance.
- (ev_002) setColorAt/getColorAt provide per-instance color (Vector4 for alpha), usable for per-stitch/per-round highlight tinting.
- (ev_003) setMatrixAt sets per-instance transforms; negatively scaled matrices are not supported.
- (ev_004) perObjectFrustumCulled and sortObjects default to true; transparent objects render back-to-front, opaque front-to-back to cut overdraw.
- (ev_005) maxInstanceCount and maxVertexCount fix capacity at construction.
- (ev_006) setInstanceCount/setGeometrySize resize buffers, deleteInstance removes instances, and optimize repacks to reclaim deleted space.

## Limitations
- The fetched documentation did not contain explicit text confirming that instance matrices are stored in a texture and read in-shader; that claim is omitted from this card. Documented behavior reflects the three.js version current at access time (page dated 2026-01).
