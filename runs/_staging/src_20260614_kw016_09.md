# Source Card: AmiGo: Computational Design of Amigurumi Crochet Patterns

## Summary
AmiGo (Edelstein, Peleg, Itzhaky, Ben-Chen; SCF '22, Oct 26-28 2022, Seattle) is a deterministic method that converts a closed triangle mesh plus a single seed point and a stitch width into a human-readable amigurumi crochet pattern. It constructs an intermediate Crochet Graph G=(S, R union C) - ordering stitches by the geodesic distance from the seed (function f) and an optimized perpendicular function g - then emits patterns using only single crochet, increase, and decrease stitches in loop-folded notation. Branching shapes are auto-segmented at saddle points of the distance function and joined with the join-as-you-go method, requiring no sewing; the method was demonstrated on 11 models (365-3,670 stitches, 0.2-6.9 min on an Intel Core i7).

## Key evidence
- (ev_001) Inputs are a closed manifold triangle mesh, a single seed vertex, and a stitch width w; output is human-readable crochet instructions.
- (ev_002) Builds a Crochet Graph G=(S, R union C): row edges order bases within a row, column edges (stems) connect rows.
- (ev_003) Row ordering comes from the geodesic distance function f(v)=d(v,s) from the seed (isolines = rows), with a perpendicular function g ordering stitches within rows.
- (ev_004) Uses only three stitches - single crochet (sc), inc(x), dec(x) - emitted in loop-folded notation such as (sc, inc, 2sc)*3.
- (ev_005) Auto-segments branching shapes at saddle-point isolines and joins segments via join-as-you-go, requiring no additional sewing.
- (ev_006) Evaluated on 11 models: 30-60 rows, 1-8 segments, 365-3,670 stitches, processing 0.2-6.9 minutes on an Intel Core i7.

## Limitations
- The paper's loop-folded notation uses asterisk syntax (sc, inc, 2sc)*3, not the "x3" form in the originating note; the verbatim form is quoted here.
- The method assumes a closed manifold triangle mesh with a single seed; the geodesic distance function's saddles must be resolvable (the implementation tunes the heat-geodesics time parameter to avoid neighboring saddles/extrema), so very detailed meshes may need preprocessing.
- Foundational/stable algorithm (tertiary): canonical for deterministic mesh-to-amigurumi but predates and does not cover modern generative or learning-based pattern methods.
