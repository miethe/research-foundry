# Source Card: Digital Crochet: Toward a Visual Language for Pattern Description

## Summary
Seitz, Rein, Lincke, and Hirschfeld (HPI / University of Potsdam, Onward! '22) propose the first visual, domain-specific, graph-based language for crochet patterns: each stitch is a node, edges (previous / insertion / slip-stitch) encode fabric dependencies, and rows and rounds are unified into a single "layers" property. The structure makes increases, decreases, holes, and a single-yarn "crochetability" constraint explicit, and a projectional editor prototype renders the same graph as 2D charts and a force-based 3D view. A think-aloud study with six professional crochet designers (drawn from a 200-designer myboshi survey pool) found the language could express 2D and 3D patterns and removed ambiguities present in conventional notations.

## Key evidence
- (ev_001) Stitches are graph nodes carrying their stitch type; insertion edges point to the insertion points a stitch is worked into, and each node has exactly one outgoing previous edge.
- (ev_002) Rows and rounds are merged into one concept, "layers," needing no structural change in the graph; the method used depends only on where the next stitch is inserted, and each node stores its layer number.
- (ev_003) Increase = multiple incoming insertion edges to one insertion point; decrease = multiple outgoing insertion edges from one stitch.
- (ev_004) Crochetability/reproducibility is enforced by edge ordering so that no loop ever uses an insertion point that does not yet exist, yielding a single unambiguous yarn path.
- (ev_005) The editor prototype builds patterns step-wise "as if crocheting," which by construction only permits crochetable patterns, and renders 2D charts plus a force-based 3D view.
- (ev_006) Evaluated in a think-aloud study with six professional crochet designers recruited from 25 survey respondents drawn from 200 myboshi designers; the language removed ambiguities present in conventional charts.
- (ev_007) Holes (openings under stitches/chains) are modeled explicitly as nodes via surroundingNode edges, capturing a technique standard crochet charts cannot represent.

## Limitations
- Findings rest on a single research prototype editor and a small qualitative study (six designers); some features (color/yarn changes, non-linear editing, copy-and-paste, custom repeatable stitches) are noted as not yet supported, so claims about tool maturity or generality should be scoped to the 2022 prototype.
