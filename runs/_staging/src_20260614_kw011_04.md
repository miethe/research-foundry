# Source Card: Modeling Crochet Patterns with a Force-directed Graph Layout

## Summary
This peer-reviewed Eurographics (Expressive + WICED 2025) paper by Émile Greer and David Mould (Carleton University) presents a method that takes a written amigurumi crochet pattern in row-by-stitch notation, translates it into a stitch graph (one node per stitch, edges for physical connections), and then renders a 3D model via a force-directed graph layout solved as a non-linear least-squares optimization. It cleanly separates pattern topology (the stitch graph) from geometric rendering (the layout/inflation step), letting designers preview a pattern digitally instead of repeatedly crocheting and unraveling.

## Key evidence
- (ev_001) Written pattern → graph → 3D representation via force-directed layout (the core pipeline).
- (ev_002) Input is row-by-stitch notation, the most common amigurumi notation, converted to a graph before layout.
- (ev_003) Per-stitch node / per-connection edge structure — the stitch graph.
- (ev_004) Edge types carry topology semantics: sequential (horizontal), working (vertical), and constraint edges.
- (ev_005) Geometry is decoupled from topology: layout is a least-squares optimization of edge-length and curvature energies.
- (ev_006) Each node rendered as a sphere, enabling per-stitch instance visibility/highlighting.
- (ev_007) Targets amigurumi; intended to preview patterns before physically crocheting, reducing crochet/unravel cycles.

## Limitations
- Tested only on simple, beginner-level patterns; explicitly noted as too narrow for intermediate/advanced patterns.
- Assumes constant edge length of 1 (does not yet model real per-stitch dimensions from yarn/hook/tension); limited to single-crochet-like stitches; piece attachment handled ad hoc.
