# Source Card: MagicVL-2B: Empowering Vision-Language Models on Mobile Devices with Lightweight Visual Encoders via Curriculum Learning

## Summary
MagicVL-2B is a 2B-class vision-language model optimized for flagship smartphones, built on a sub-100M-parameter SigLIP2-base visual encoder (93M params), a Qwen2.5-1.5B/Qwen3-1.7B LLM backbone, a redesigned dynamic resolution scheme, and a multimodal curriculum learning strategy. The paper reports that it matches state-of-the-art <=2B accuracy on standard VLM benchmarks while substantially cutting on-device latency and power consumption (41.1% total power reduction). It is positioned as a practical solution for running advanced multimodal intelligence directly on-device.

## Key evidence
- (ev_001) SigLIP2-base ViT visual encoder with 93M parameters at 384x384, with Qwen2.5-1.5B or Qwen3-1.7B as LLM backbone.
- (ev_002) On Snapdragon 8 Elite: 0.09s ViT latency, 1.7s LLM latency, 23.9 token/s throughput vs InternVL2.5-2B's 0.90s ViT / 14.3 token/s.
- (ev_003) Benchmark accuracy (<=2B class): DocVQA 89.0, OCRBench 828, AI2D 77.4, MMBench-V1.1_en 73.7, MMStar 57.9.
- (ev_004) Dynamic high-resolution token resizing cuts visual tokens ~37.8% (0.52M vs 0.81M) while improving accuracy (74.5% vs 74.3%).
- (ev_005) Matches state-of-the-art accuracy while reducing on-device power consumption by 41.1%.
- (ev_006) Sub-100M-parameter-encoder VLM using a redesigned dynamic resolution scheme plus multimodal curriculum learning.

## Limitations
- Self-reported results from the model's creators; on-device latency/power and benchmark figures are vendor-measured and may be framed favorably.
- The specific power mAh values (0.689 vs 1.170) and the 91.7% vision-encoder power reduction cited in some summaries were not surfaced in the verbatim text fetched; only the 41.1% total reduction is verbatim-confirmed here. The token-reduction figure in the source body reads 0.52M (not 0.51M).
