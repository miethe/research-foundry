# Source Card: KnitScript: A Domain-Specific Scripting Language for Advanced Machine Knitting

## Summary
KnitScript (Hofmann et al., ACM UIST '23) is a domain-specific scripting language for computationally driven machine knitting that interprets a higher-level authoring layer into low-level Knitout instructions — a concrete instance of the layered DSL-to-IR compilation pattern. It maintains a comprehensive virtual model of the knitting machine, validates each operation against that model and a knit-graph before emitting checkable Knitout, and introduces higher-level abstractions (carriage passes, sheets, layers, yarn management) above the raw assembly-like instruction set. The paper establishes that authoring in a validated DSL and compiling to a checkable IR is the accepted approach in adjacent fiber-craft tooling, motivated by the non-reusability and error-proneness of per-instance Knitout code.

## Key evidence
- (ev_001) KnitScript provides a comprehensive virtual model of knitting machines and automates tedious, error-prone details.
- (ev_002) KnitScript is a higher-level language that interprets into Knitout instructions (DSL-to-IR), adding abstractions like carriage passes, sheets, and layers.
- (ev_003) Knitout is an assembly-like / g-code-like low-level language over individual machine operations, lacking control structures, parameters, or machine-state modeling.
- (ev_004) KnitScript maintains a virtual machine model and introduces new abstractions (carriage passes, yarn management, stitch tracking, lateral transfers) above raw instructions.
- (ev_005) The interpreter validates operations against the virtual model and knit graph, raising errors/warnings before recording valid Knitout — author in validated DSL, compile to checkable IR.
- (ev_006) Knitout code is single-instance and non-reusable; per-instance reimplementation introduces errors and inefficiencies — motivating native structured authoring.
- (ev_007) Published at ACM UIST '23 (October 2023), anchoring a stable 2023 methodological reference.
