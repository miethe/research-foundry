# Source Card: AmiGo: Computational Design of Amigurumi Crochet Patterns

## Summary
AmiGo (Edelstein, Peleg, Itzhaky, Ben-Chen; Technion, SCF '22) is a peer-reviewed method that generates human-crochetable amigurumi patterns from a closed triangle mesh, a single user-specified seed point, and a target stitch width. It constructs an explicit intermediate "Crochet Graph" (vertices = stitch tops/bases, column edges = stems, row edges = within-row connectivity) and translates that graph into a round-by-round written pattern. The system automatically segments the shape into crochetable components joined via join-as-you-go, so no sewing is needed, and derives shaping (increase/decrease placement) procedurally from the graph geometry.

## Key evidence
- (ev_001) Generates instructions from only a closed triangle mesh plus one user-specified seed point.
- (ev_002) A target stitch width w is an explicit input parameter to the instruction-generation function.
- (ev_003) A Crochet Graph (geometry + connectivity) is the intermediate representation translated into the pattern.
- (ev_004) The graph encodes stitches as vertices (tops/bases), column edges (stems), and row edges (within-row connectivity), making the IR inspectable.
- (ev_005) Increases/decreases that accommodate curvature are derived procedurally from geometry, not specified arbitrarily.
- (ev_006) Shapes are auto-segmented into components joined via join-as-you-go, eliminating sewing.

## Limitations
- Stitch width is a required input, but the paper does not document a default value; pattern output depends on the supplied gauge.
- The method targets amigurumi-class closed stuffed shapes; generality to arbitrary garment or open geometries is not claimed here.
