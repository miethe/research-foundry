# Source Card: AmiGo: Computational Design of Amigurumi Crochet Patterns

## Summary
AmiGo (SCF '22, ACM) is a peer-reviewed geometry-processing method that automatically generates human-readable Amigurumi crochet patterns from a closed 3D triangle mesh plus a single user-specified seed point and stitch size. It works by building an intermediate "Crochet Graph" (geometry + connectivity), which is translated into a program via standard code-synthesis tools and then loop-unrolled into a pattern; when crocheted and stuffed the result resembles the input shape. The approach is deterministic/computational (not ML-based), uses only simple stitches, and joins automatically segmented components with the join-as-you-go method so no sewing is needed.

## Key evidence
- (ev_001) Input is a closed triangle mesh plus a single user point; output crocheted-and-stuffed toy resembles the input geometry.
- (ev_002) The pipeline is mediated by a "Crochet Graph" (geometry + connectivity) that is translated into a pattern.
- (ev_003) Shapes are automatically segmented into crochetable components joined via join-as-you-go, requiring no sewing.
- (ev_004) Full inputs are a closed 3D model, a seed point, and a stitch size; instructions use only simple stitches.
- (ev_005) Authors claim the output resembles the input more closely than any prior crochet-generation approach.
- (ev_006) No crochet machine exists to date, motivating a computational pattern-generation pipeline.
- (ev_007) The graph is converted to a program via standard code-synthesis tools then loop-unrolled — a deterministic, non-ML pipeline.
- (ev_008) Documented geometric obstruction: "craters" (negative mean / positive Gaussian curvature) cannot be realized and are preprocessed away.

## Limitations
- Output is "similar" to the input, not exact, and the method is sampling-rate dependent (too few rows loses geometric detail).
- Documented geometric obstructions to crochetability: negative mean curvature ("craters") and branching require preprocessing/segmentation; not all shapes are handled directly.
- The paper conjectures (but does not prove) that the f,g parameterization is bijective, leaving the proof to future work.
