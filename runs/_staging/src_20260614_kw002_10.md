# Source Card: crochetBench (official CrochetBench code + data repository)

## Summary
CrochetBench is the official code-and-data repository for a benchmark that evaluates language models on crochet-pattern understanding and generation. It adopts the CrochetPARADE DSL as an intermediate representation (vendored from Codeberg), ships two JavaScript validators, and structures evaluation around four tasks (Stitch Recognition, Instruction Selection, Instruction Generation, Instruction-to-DSL). For the DSL-generation task it scores outputs with Compilation Success Rate (CSR, all-or-nothing compilation) and Partial Executable Rate (PER, fraction compiling before failure).

## Key evidence
- (ev_001) CrochetBench adopts the CrochetPARADE DSL as its intermediate representation.
- (ev_002) The benchmark vendors CrochetPARADE by cloning it from codeberg.org/crochetparade for validation.
- (ev_003) Two JS validators ship in benchmark_task/: verify_crochet_pattern.js and verify_crochet_pattern_with_history.js.
- (ev_004) Data files include project_level_test.json, step_level_test_*.json, and mc_data.json.zip (natural-language-to-DSL pairs / multiple-choice).
- (ev_005) CSR measures the proportion of generated DSL outputs that compile successfully with the validator (all-or-nothing).
- (ev_006) PER measures the average fraction of a program that compiles successfully before failure.
- (ev_007) Four tasks: A Stitch Recognition (6,009), B Instruction Selection (6,003), C Instruction Generation (6,009), D Instruction-to-DSL (119 step-level / 100 project-level).
