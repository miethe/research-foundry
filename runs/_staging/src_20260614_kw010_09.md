# Source Card: CrochetPARADE Manual (Pattern Renderer, Analyzer, and Debugger)

## Summary
The CrochetPARADE Manual documents a browser-based crochet pattern renderer, analyzer, and debugger that builds an interactive 3D stitch graph from a pattern. It animates construction stitch-by-stitch, encodes working order and stitch relationships via colored connections, exposes per-stitch metadata on hover, and flags over/under-stretched stitches as a tension/gauge signal. Critically, it visualizes how stitches attach and where tension fails, but provides no feature explaining why stitch or shaping counts change.

## Key evidence
- (ev_001) Animates pattern creation by revealing/hiding stitches one-by-one ('a' / ctrl+a), a precedent for row-by-row playback and ghost-next-row overlay.
- (ev_002) Blue arrowed connections show working direction; red connections show remaining stitch relationships -- the "how it attaches" view.
- (ev_003) Highlights a row/stitch (ctrl+f), highlights by label (ctrl+d), and hides after a row/stitch (ctrl+h) -- checkpoint and isolation primitives.
- (ev_004) The 's' key flags over/under-stretched stitches (blue = loose, red = tight beyond ~15%) -- a tension/gauge confidence signal.
- (ev_005) Hover reveals row, stitch number, and stitch type; shift+click pins a persistent info box -- an inline per-stitch rationale affordance.
- (ev_006) No feature explains why a count/decrease changed or analyzes inc/dec distribution rationale -- the explainability whitespace for KnitWit.

## Limitations
- Single-author official tool documentation (Svetlin Tassev, CC BY-NC-SA 4.0, updated 2026-01-07); describes intended behavior, not an empirical evaluation. A separate "Scatter increases and decreases" generator option distributes shaping during generation but is not a post-hoc explainability feature, so it does not fill the ev_006 gap.
