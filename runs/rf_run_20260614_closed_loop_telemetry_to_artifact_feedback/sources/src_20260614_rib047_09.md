# Source Card: MLOps: Continuous delivery and automation pipelines in machine learning

## Summary
This Google Cloud Architecture Center document establishes a canonical MLOps maturity model (Levels 0, 1, and 2) and enumerates the conditions under which production ML pipelines should retrain models. It defines five distinct retraining triggers (on demand, on schedule, on new-data availability, on performance degradation, and on data-distribution change) and defines Continuous Training (CT) as automatic in-production retraining on fresh data driven by live pipeline triggers. It deliberately specifies no universal time interval or numeric performance threshold, treating retraining cadence as use-case dependent.

## Key evidence
- (ev_001) Five retraining triggers are defined for production ML pipelines, framed as use-case dependent.
- (ev_002) "On demand" = ad hoc manual run; "on a schedule" = systematically available labeled data (daily/weekly/monthly).
- (ev_003) "On model performance degradation" retrains on noticeable degradation, with no numeric threshold given.
- (ev_004) "On significant changes in data distributions" (drift) fires on observed feature-distribution shifts.
- (ev_005) Continuous Training (CT) = automatic in-production retraining on fresh data via live triggers.
- (ev_006) MLOps Level 0 deploys a new model version only a couple of times per year.
- (ev_007) MLOps Level 0 separates model-creating data scientists from model-serving engineers (disconnection between ML and operations).
- (ev_008) MLOps Level 1 adds automated data validation (gate before retraining) and model validation (gate before promotion).

## Limitations
- Vendor-authored (Google Cloud) architecture guidance; framing favors automated, pipeline-based MLOps and Google Cloud tooling, so its maturity ladder reflects a particular ideal rather than an industry standard.
- The document deliberately specifies no universal time interval or numeric performance threshold for triggering retraining, so it cannot be cited for any specific cadence or threshold value.
