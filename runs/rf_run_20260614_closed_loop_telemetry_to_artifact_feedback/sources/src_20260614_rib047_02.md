# Source Card: AI Agent Observability: Tracing, Testing, and Improving Agents

## Summary
This LangChain resource page describes how agent observability (via LangSmith) instruments multi-step agents and closes a production-to-improvement loop. It establishes what is captured per trace (tool-call provenance, model inputs/outputs, token counts, latency, and memory read/write for stateful agents) and how those traces flow into an Insights Agent, auto-build evaluation datasets, route to Annotation Queues for human feedback, and feed regression testing.

## Key evidence
- (ev_001) Tool-call provenance: which tools selected, arguments, results, and per-call duration.
- (ev_002) Cost/latency attribution can pinpoint a sub-task using 80% of tokens or adding 3s of tool-call latency.
- (ev_003) Per-trace signals: models, inputs, output completions, input/output token counts, tool-call latency.
- (ev_004) Stateful agents record memory reads/writes and how state influenced subsequent decisions.
- (ev_005) Production traces flow into an Insights Agent that surfaces agent-behavior patterns.
- (ev_006) Rules auto-add criteria-matching traces to datasets and route cases to Annotation Queues / online evals.
- (ev_007) Regression testing locks in gains so fixed bugs stay fixed.
- (ev_008) The closed loop: capture traces, analyze patterns, build datasets, evaluate, drive improvements.
