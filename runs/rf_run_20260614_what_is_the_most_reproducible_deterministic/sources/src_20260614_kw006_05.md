# Source Card: Computing Stitches and Crocheting Geometry

## Summary
A peer-reviewed CAAD Futures 2017 paper (Çapunaman, Bingöl, Gürsoy; Springer CCIS vol. 724, pp. 289-305) that formalizes the craft of crocheting 3D objects as a computer algorithm implemented in Grasshopper for Rhino. Stitch directions and connectivity are derived from the NURBS UV parameterization of an input surface, and gauge/hand-effect variables are captured empirically via physical 10-by-10-stitch tension swatches. The algorithm extends crochet beyond axially symmetric revolved surfaces to non-symmetrical 3D objects, emitting a conventional human-readable text pattern.

## Key evidence
- (ev_001) The algorithm requires a UV-parameterized surface as input (stronger precondition than a single seed point).
- (ev_002) Method relies on NURBS UV division to transfer 3D geometry into a crochet pattern.
- (ev_003) Six 10-by-10-stitch swatches capture the crafter's grip on the yarn.
- (ev_004) Gauge split into determinate (yarn weight, hook size) and indeterminate (crafter's grip) variables.
- (ev_005) Generates crochet patterns for non-symmetrical 3D objects, not just revolved/axisymmetric ones.
- (ev_006) Two-stage method: analytical systematization, then formalization as a computer algorithm.

## Limitations
- Springer landing page is paywalled; method-level verbatim quotes were taken from the authors' own ResearchGate-hosted full-text PDF of the identical paper. Abstract and bibliographic metadata verified directly from Springer.
- Scope gap vs amigurumi: no in-the-round geodesic shaping and no explicit per-round increase/decrease-rate bound; producibility depends on parameterization quality plus swatch-measured gauge.
