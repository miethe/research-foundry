# Source Card: autoknit (re-implementation of "Automatic Machine Knitting of 3D Meshes")

## Summary
The `textiles-lab/autoknit` GitHub repository is a public-domain, open-source re-implementation of the SIGGRAPH 2018 paper "Automatic Machine Knitting of 3D Meshes." Its README documents a four-stage pipeline — Constraints, Peeling/Linking, Scheduling, and Knitout — that turns an OBJ mesh plus user-defined constraints into machine knitout instructions, with a distinct intermediate file format (.cons, .st, .js, .k) emitted at each stage. The README also records concrete maturity caveats, most notably that scheduling works for small cases only.

## Key evidence
- (ev_001) The codebase is placed in the public domain, making the pipeline and intermediate formats reusable for IP-safe work.
- (ev_002) autoknit is an open re-implementation of "Automatic Machine Knitting of 3D Meshes" that does not exactly match the paper's original code.
- (ev_003) Inputs are an OBJ mesh plus user-defined constraints saved to a .cons file.
- (ev_004) Constraints must cover all boundaries and can also be placed to control knitting direction, with a region-cut option for boundary-free meshes.
- (ev_005) Stitch geometry is set by obj-scale, stitch-width, and stitch-height relative to the scaled object.
- (ev_006) The pipeline is Constraints -> Peeling/Linking -> Scheduling -> Knitout with a distinct file format per stage (.cons, .st, .js, .k).
- (ev_007) Peeling/Linking builds a row-column graph from the mesh; peeling/tracing are "mostly working," linking is "working."
- (ev_008) Scheduling works for small cases only and has no greedy fallback (optimal-only).

## Limitations
- This README describes the open re-implementation, which explicitly "does not match the code used in the paper exactly"; algorithmic/quantitative details (and the helix-free quad-dominant remeshing / stitch-cell geometry) live in the underlying ACM TOG 37(3) SIGGRAPH 2018 paper (Narayanan, Albaugh, Hodgins, Coros, McCann) and are NOT directly citable from this source. The "working for small cases only" scheduling note is a real maturity caveat to carry into any productization claim.
