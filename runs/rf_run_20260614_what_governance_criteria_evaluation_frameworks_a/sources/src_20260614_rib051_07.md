# Source Card: Update the Approval Status of a Model - amazon-sagemaker-developer-guide (GitHub)

## Summary
This official AWS SageMaker developer guide page documents the model-version approval workflow in the SageMaker Model Registry. It establishes the three ModelApprovalStatus values (PendingManualApproval as the registration default, Approved, Rejected), how status is changed via the update_model_package Boto3 API or SageMaker Studio, and which status transitions trigger CI/CD deployment actions under SageMaker provided project templates.

## Key evidence
- (ev_001) Approval status is updated to Approved (meets requirements) or Rejected (does not), and Approved can initiate CI/CD deployment.
- (ev_002) PendingManualApproval->Approved and Rejected->Approved each initiate CI/CD deployment; PendingManualApproval->Rejected does nothing.
- (ev_003) Approved->Rejected initiates CI/CD to deploy the latest model version that still holds an Approved status.
- (ev_004) Status is changed via the update_model_package API call using ModelPackageArn and ModelApprovalStatus.
- (ev_005) Status can be set manually (Boto3 / Studio) or automatically via a pipeline condition step evaluating performance.
- (ev_006) ModelApprovalStatus is set to PendingManualApproval at registration (the default starting state).

## Limitations
- The described transition-to-action mappings apply specifically when using SageMaker provided project templates; custom CI/CD configurations may behave differently. Page is published by AWS (2025) and reflects the documented API as of that revision.
