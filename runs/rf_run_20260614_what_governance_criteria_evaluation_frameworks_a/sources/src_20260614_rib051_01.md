# Source Card: Protected prompt labels - Langfuse Changelog

## Summary
This Langfuse changelog entry (2025-04-02) introduces "protected prompt labels," a governance control that locks production prompt versions. When a label such as `production` is marked protected, lower-privilege `viewer` and `member` roles can no longer modify or delete it, while only `admin` and `owner` roles can change it. Protection status is configured in project settings and is gated to Team (Cloud) and Enterprise plans.

## Key evidence
- (ev_001) Protected prompt labels give admins/owners the ability to prevent labels from being modified or deleted, improving control over prompt deployment.
- (ev_002) Viewer and member roles cannot modify or delete a protected label, preventing changes to the production prompt version.
- (ev_003) Admin and owner roles can still modify or delete a protected label, deliberately changing the production prompt version.
- (ev_004) Protection status is managed by admins and owners in the project settings.
- (ev_005) Released 2025-04-02 (author Hassieb); available on all Team (Cloud) and Enterprise (Cloud and Self-Hosted) plans.

## Limitations
- Brief changelog entry; full mechanics are deferred to Langfuse's prompt-management documentation, and the feature is gated to Team/Enterprise plans so behavior may not apply to all deployments.
