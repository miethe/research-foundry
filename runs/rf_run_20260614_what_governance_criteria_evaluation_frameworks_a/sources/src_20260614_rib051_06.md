# Source Card: Model Registration Deployment with Model Registry - Amazon SageMaker AI

## Summary
This first-party AWS documentation page describes the Amazon SageMaker Model Registry, a capability for cataloging production models, managing model versions and approval status, and defining a staging construct that models progress through for the model lifecycle. Models are organized into Model (Package) Groups (each trained model registered as a new version), which can be grouped into Collections. The registry integrates with SageMaker ML pipelines and supports automated CI/CD deployment.

## Key evidence
- (ev_001) The Model Registry catalogs models for production, manages versions, manages approval status, and deploys models.
- (ev_002) It supports a lifecycle staging construct and automated deployment via CI/CD.
- (ev_003) Models are cataloged in Model (Package) Groups, with each trained model registered as a new version.
- (ev_004) Model Groups can be organized into higher-level Model Registry Collections.
- (ev_005) A typical workflow registers a model version in the Model Group for each ML pipeline run.
- (ev_006) The registry associates training metrics/metadata and surfaces model lineage for traceability and reproducibility.
