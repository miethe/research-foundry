# Source Card: A/B Testing (Prompt Management) - Langfuse Docs

## Summary
This is the first-party Langfuse documentation page describing how A/B testing of LLM prompts works within Langfuse Prompt Management. A/B testing is set up by labeling distinct prompt versions (e.g., `prod-a` and `prod-b`); the application randomly alternates between the labeled versions at runtime while Langfuse tracks per-version performance metrics. Comparison is done in the Analyze Results view across latency, cost, token usage, and evaluation/quality scores, with a pointer to the separate Experiments feature for dataset-level benchmarking.

## Key evidence
- (ev_001) Langfuse implements prompt A/B testing by labeling prompt versions, e.g. `prod-a` and `prod-b`.
- (ev_002) The application randomly alternates between labeled versions while Langfuse tracks performance metrics.
- (ev_003) Langfuse tracks per-version response latency, cost, token usage, and evaluation metrics.
- (ev_004) Documented comparison metrics: latency and token usage, cost per request, quality evaluation scores, and custom user-defined metrics.
- (ev_005) For dataset-level benchmarking of complete application behavior, the docs direct users to Experiments rather than A/B testing.
