# Source Card: Manage prompts - LangChain / LangSmith Docs

## Summary
LangSmith's prompt management documentation describes a Git-like versioning model where prompts have a commit history that can be referenced by tags. Named deployment environments (Staging and Production) map to specific commits, with promotion semantics that are additive (promoting to Production does not clear Staging). Code references prompts by tag (e.g. `:production`), decoupling version selection from code changes; the `staging` and `production` tag names are reserved for environment management.

## Key evidence
- (ev_001) LangSmith Environments are named deployment targets (Staging and Production) assigned to specific prompt commits.
- (ev_002) Environments let teams track which prompt version is active per environment and promote commits between them.
- (ev_003) Promoting a commit to Production does not remove it from Staging.
- (ev_004) The 'staging' and 'production' tag names are reserved for environment management and disabled in the freeform tag picker.
- (ev_005) Each freeform tag references exactly one commit but can be reassigned to a different commit.
- (ev_006) Prompts are pulled by tag in code (client.pull_prompt("joke-generator:production")), so version selection changes without code edits.
- (ev_007) Commit tags reference a specific commit, so the active prompt version can change without modifying code.
