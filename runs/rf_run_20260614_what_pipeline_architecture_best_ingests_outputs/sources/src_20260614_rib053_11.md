# Source Card: Research Foundry — Service API & Artifact Contract

## Summary
This authoritative internal contract specifies the exact Python function signatures and artifact field shapes for Research Foundry's parallel service build. It defines the normalized artifact chain — source_card (§6.8), extraction_card (§6.9), claim_ledger (§6.10), and evidence_bundle (§6.11) — that an ingestion/outputs pipeline must populate, including the per-evidence (ev_NNN) provenance model and per-claim source attribution. A core invariant is determinism-by-default: ingestion and normalization run without network or API keys, with all LLM/network behavior opt-in and gracefully degrading.

## Key evidence
- (ev_001) source_card front matter shape: source{title, source_type, locator, accessed_at}, trust, usage, and extracted_points[] each with evidence_id/locator/summary/quote.
- (ev_002) source_type enum and unknown default for source_rank — the dimensions a research focus must populate.
- (ev_003) extraction_card carries per-fact {evidence_id, text, locator, confidence, quote_available} plus contradictions_or_cautions.
- (ev_004) claim_ledger maps each claim to sources[{source_card_id, evidence_id, relation, locator}] — RF's native per-claim attribution.
- (ev_005) evidence_bundle aggregates source/extraction cards and claims with counts and governance.approved_for_writeback.
- (ev_006) Determinism-by-default: no network/keys required on the standard path; LLM/network is opt-in and degrades gracefully.
- (ev_007) Deterministic ingestion emits up to 8 evidence points; degraded fetch yields a single needs_content placeholder.
- (ev_008) Claim status enum supported|inference|speculation|unsupported with contradiction/inference logs and unresolved_questions.

## Limitations
- This is an internal coordination contract (sensitivity: personal/internal); it documents the intended RF service/artifact design rather than an external standard, so it is authoritative only for Research Foundry's own pipeline and not citable as a general-industry specification.
