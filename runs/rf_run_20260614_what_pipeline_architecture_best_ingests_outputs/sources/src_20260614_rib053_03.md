# Source Card: Grounding with Google Search | Gemini API

## Summary
Google's official Gemini API documentation specifies how Google Search grounding returns provenance metadata. The `groundingMetadata` object bundles `webSearchQueries`, `groundingChunks` (each web source's `uri` and `title`), `groundingSupports` (which map response text segments to source chunks for inline citations), and `searchEntryPoint` (HTML/CSS Search Suggestions whose display the Terms of Service requires). It also documents billing per search query for Gemini 3 models and the migration from the older `google_search_retrieval` tool to the current `google_search` tool.

## Key evidence
- (ev_001) groundingChunks is an array of web sources, each with a uri and title.
- (ev_002) groundingSupports connects model response text to the sources in groundingChunks.
- (ev_003) Each support links a text segment (startIndex/endIndex) to groundingChunkIndices, enabling inline citations.
- (ev_004) searchEntryPoint contains HTML/CSS to render the required Search Suggestions.
- (ev_005) Gemini 3 projects are billed per search query the model decides to execute.
- (ev_006) Older models use google_search_retrieval; current models use the google_search tool.

## Limitations
- The exact mandate to display Search Suggestions resides in the linked Terms of Service rather than verbatim on this page; this card cites the doc's pointer to those requirements.
- The source documentation notes example chunk `uri` values are `vertexaisearch.cloud.google.com` redirect endpoints; resolving them to true publisher URLs requires a follow-the-redirect step not captured as a standalone verbatim quote here.
