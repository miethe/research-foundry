# Source Card: GroundingMetadata (Vertex AI / Gemini REST reference, v1beta1)

## Summary
This Google Cloud REST API reference defines the GroundingMetadata response object returned when grounding is enabled on a Gemini/Vertex AI generation. It specifies the nested schema that maps generated text spans to their evidence: groundingChunks hold the source web.uri and web.title; groundingSupports link a byte-indexed content Segment (startIndex/endIndex) to one or more chunks via groundingChunkIndices, with a parallel confidenceScores array. It also documents the auxiliary debug/retrieval fields (webSearchQueries, searchEntryPoint, retrievalMetadata).

## Key evidence
- (ev_001) Each grounding chunk's web object carries a string uri (source URL) and string title (page title) linking back to the source.
- (ev_002) groundingSupports.groundingChunkIndices is an integer array indexing into GroundingMetadata.groundingChunks to identify supporting sources for a segment.
- (ev_003) confidenceScores is a 0.0-1.0 float array parallel to groundingChunkIndices; empty/ignored for Gemini 2.5+.
- (ev_004) A segment's startIndex (inclusive) and endIndex (exclusive) are byte offsets delimiting the supported content span.
- (ev_005) GroundingMetadata additionally returns webSearchQueries, searchEntryPoint, and retrievalMetadata for the retrieval/search source.
- (ev_006) Worked example: groundingChunkIndices [1, 3] means groundingChunks[1] and [3] are the sources for that segment's claim — the canonical (span -> chunk indices -> URL) inline-citation pattern, structurally equivalent to mapping a claim text span to a source_card evidence_id.

## Limitations
- No publication or last-updated date is displayed on the page; published_at is recorded as null.
- This is the v1beta1 beta API surface; field names/shapes may change before GA.
- confidenceScores semantics differ by model generation (populated for Gemini 2.0 and earlier; empty for 2.5+).
