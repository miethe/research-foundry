# Source Card: Introduction to deep research in the OpenAI API

## Summary
This OpenAI Cookbook article documents the Deep Research API, which is accessed through the `responses` endpoint and returns a structured, citation-rich report. It defines two dated models (o3-deep-research-2025-06-26 and o4-mini-deep-research-2025-06-26), the citation annotation contract (start_index/end_index, title, url), and the shape of `response.output` as a sequential list of typed steps (reasoning, web_search_call, code_interpreter_call) with the final report as the last item. It also covers the available tools: web_search_preview, code_interpreter, and MCP-based tools for proprietary knowledge bases.

## Key evidence
- (ev_001) Two dated Deep Research models: o3-deep-research-2025-06-26 (quality) and o4-mini-deep-research-2025-06-26 (lightweight/faster).
- (ev_002) Citation annotations carry start_index/end_index, title, and full source url.
- (ev_003) Intermediate steps stored in response.output; the type field identifies each step kind.
- (ev_004) Final report is the last output item (response.output[-1].content[0].text); annotations at the same item.
- (ev_005) Deep Research and the Responses API support MCP-based tools for private knowledge stores / third-party services.
- (ev_006) web_search_preview is required; code_interpreter is optional for data parsing and charts.

## Limitations
- This is documentation/contract reference, not benchmark data; performance, accuracy, and cost claims are not established here. API field names and model version dates are current as of the 2025-06-25 publication and may change in later API revisions.
