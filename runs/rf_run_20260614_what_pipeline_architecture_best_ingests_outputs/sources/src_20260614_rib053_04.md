# Source Card: Gemini Deep Research Agent | Gemini API

## Summary
Google's first-party Gemini API documentation specifies the Deep Research Agent: an autonomous plan/search/read/iterate research workflow that produces detailed, cited reports. It defines two preview model IDs (`deep-research-preview-04-2026` and `deep-research-max-preview-04-2026`), the asynchronous `Interaction` object (id, status, output_text, steps, error) used to poll background tasks, the default tool set (Google Search, URL Context, Code Execution, extensible via MCP and File Search), and a safety recommendation to review the response's citations to verify sources. The feature is in preview via the Interactions API (itself in public beta).

## Key evidence
- (ev_001) Two preview versions: deep-research-preview-04-2026 (speed) and deep-research-max-preview-04-2026 (comprehensiveness).
- (ev_002) Agent autonomously plans, executes, and synthesizes multi-step research into detailed, cited reports.
- (ev_003) Background tasks return a partial Interaction with an id; status transitions in_progress -> completed/failed.
- (ev_004) Interaction exposes status, output_text (final report), and error fields.
- (ev_005) The steps array holds typed steps whose content includes text and image items.
- (ev_006) Google recommends reviewing the citations in the response to verify sources (web-content risk mitigation).
- (ev_007) Feature is in preview, available exclusively via the Interactions API (not generate_content).
- (ev_008) Default tools: Google Search, URL Context, Code Execution; extensible via MCP servers and File Search.

## Limitations
- Preview/beta status: the doc explicitly marks the agent as in preview and the Interactions API as in public beta, so model IDs, schemas, and field shapes may change.
- This page does not mention the Gemini app (web) Deep Research export-to-Google-Docs/PDF or Workspace-file grounding; that claim was not present in the source text and is not asserted here.
- Field names like `citations` are referenced in the safety section but the page does not provide a full formal schema for the Interaction object beyond code examples.
