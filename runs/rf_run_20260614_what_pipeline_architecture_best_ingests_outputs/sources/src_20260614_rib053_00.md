# Source Card: Deep research | OpenAI API

## Summary
OpenAI's "Deep research" API guide documents two Responses-API models, `o3-deep-research` and `o4-mini-deep-research`, that browse, analyze, and synthesize many sources into a citation-rich report. The response output is a typed array of steps (web_search_call, code_interpreter_call, mcp_tool_call, file_search_call) terminating in a `message` whose `output_text` carries inline citations as `annotations` objects with `url`, `title`, `start_index`, and `end_index`. This is the authoritative contract for normalizing deep-research outputs into evidence with inline source linkage.

## Key evidence
- (ev_001) Two deep research models (o3-deep-research, o4-mini-deep-research) synthesize hundreds of sources into an analyst-level report via the Responses API.
- (ev_002) The output array lists the web search, code interpreter, and remote MCP calls the model made to reach the answer.
- (ev_003) Typed output items include web_search_call (with an `action`), code_interpreter_call, mcp_tool_call, file_search_call, and message.
- (ev_004) The `message` item holds the model's final answer with inline citations.
- (ev_005) Each citation annotation on `output_text` carries url, title, start_index, and end_index (integer character offsets into the answer text).
- (ev_006) Inline citations should be rendered as clearly visible, clickable links in the consuming UI.
