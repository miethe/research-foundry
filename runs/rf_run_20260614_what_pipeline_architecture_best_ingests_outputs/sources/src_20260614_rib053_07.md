# Source Card: C2PA Technical Specification (Content Credentials) v2.4

## Summary
The C2PA v2.4 Technical Specification (April 2026) is the primary normative standard for Content Credentials. It defines the Manifest as the binding of assertions, a single signed Claim, and a Claim Signature, produced by a non-human claim generator. It specifies how AI involvement is recorded (digitalSourceType values, the c2pa.actions assertion, and a new v2.4 c2pa.ai-disclosure assertion) and distinguishes hard cryptographic bindings from soft fingerprint/watermark bindings.

## Key evidence
- (ev_001) A C2PA Manifest binds one or more assertions (including content bindings), a single claim, and a claim signature.
- (ev_002) A Claim is a digitally signed, tamper-evident structure referencing assertions and content-binding information.
- (ev_003) Claim v2 split the assertions list into created_assertions and gathered_assertions for the Trust Model.
- (ev_004) The claim generator is the non-human (hardware/software) actor producing the claim and signature.
- (ev_005) v2.4 added the c2pa.ai-disclosure assertion for machine-readable AI transparency.
- (ev_006) AI/ML provenance is encoded via digitalSourceType values such as trainedAlgorithmicData.
- (ev_007) Hard bindings use cryptographic hashes to uniquely identify the asset for tamper detection.
- (ev_008) Soft bindings use fingerprints or invisible watermarks to match derived content even when bits differ.
