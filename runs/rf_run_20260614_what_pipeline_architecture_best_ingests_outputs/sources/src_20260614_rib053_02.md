# Source Card: Chat Completions | Perplexity Sonar API Reference

## Summary
The official Perplexity Sonar Chat Completions API reference documents the response schema that an ingestion pipeline must parse. Responses expose a top-level `citations` array (source URLs) and a richer `search_results` array of objects (title, url, date, last_updated, snippet, source). The `date`/`last_updated` fields supply per-source publication provenance usable for dedup and freshness logic, and the `source` enum distinguishes web results from attachments. The request `model` field selects among four Sonar models: sonar, sonar-pro, sonar-deep-research, sonar-reasoning-pro.

## Key evidence
- (ev_001) The response includes a top-level citations array of source URLs used to generate the response.
- (ev_002) The response also returns a search_results array of objects used for context in the response.
- (ev_003) Each search result carries a date (publication date) and last_updated field for per-source provenance, freshness, and dedup.
- (ev_004) Each search result's source field marks whether the result came from the web or an attachment.
- (ev_005) The request model field selects among sonar, sonar-pro, sonar-deep-research, and sonar-reasoning-pro.
- (ev_006) Each search result includes a snippet field with a text excerpt from the result page.

## Limitations
- The source brief claimed sonar-pro returns roughly double the citations of sonar; this per-model citation count is not stated in the API reference page text and was omitted to avoid an unverifiable claim.
- Documentation is undated beyond the year; exact field schema may evolve with API versions.
