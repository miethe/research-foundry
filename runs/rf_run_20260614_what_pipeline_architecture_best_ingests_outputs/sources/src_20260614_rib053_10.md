# Source Card: Perplexity API Changelog

## Summary
The first-party Perplexity API changelog documents that the legacy `citations` field has been fully deprecated and removed in favor of a richer `search_results` field carrying title, URL, and publication-date per source. Later entries add detailed per-request cost reporting (input/output/request/total) and reaffirm OpenAI API compatibility, both of which directly inform RF ingest design: source-card field mapping, dedup-friendly structured sources, and CCDash-style cost telemetry.

## Key evidence
- (ev_001) The `citations` field is fully deprecated and removed; applications must use `search_results`, which adds titles, URLs, and publication dates.
- (ev_002) `search_results` entries expose title/url/date, mapping onto RF source.locator.url and source.title.
- (ev_003) `search_results` was added for direct, transparent access to the search results the models actually used.
- (ev_004) API responses now include detailed per-request cost information.
- (ev_005) Cost is broken into input_tokens_cost, output_tokens_cost, request_cost, and total_cost.
- (ev_006) OpenAI compatibility is preserved via `/v1/responses` alias and an OpenAI-compatible `GET /v1/models`.
