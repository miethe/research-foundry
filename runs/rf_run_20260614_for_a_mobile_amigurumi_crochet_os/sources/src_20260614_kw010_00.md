# Source Card: CrochetBench: Can Vision-Language Models Move from Describing to Doing in Crochet Domain?

## Summary
CrochetBench (arXiv:2511.09483, Li et al., Nov 2025) is an academic benchmark testing whether vision-language models can move beyond describing crochet visuals to generating executable, compilable patterns expressed in the CrochetPARADE DSL. Across stitch classification, instruction grounding, and natural-language/image-to-DSL translation, model performance collapses as the metric shifts from surface similarity to executable correctness, exposing weaknesses in long-range symbolic reasoning and 3D-aware procedural synthesis. The paper's error taxonomy (Turning Issues, Multiplier Issues, Runtime Errors) and its note that valid human patterns can be penalized by under-specification directly motivate why a structured, machine-checkable pattern format matters for an amigurumi/crochet OS.

## Key evidence
- (ev_001) CrochetBench evaluates the describe-to-do shift via fine-grained procedural crochet reasoning requiring compilable procedures.
- (ev_002) Performance drops sharply from surface similarity to executable correctness, exposing long-range symbolic reasoning and 3D synthesis gaps.
- (ev_003) CrochetPARADE DSL is the intermediate representation enabling structural validation and execution-based functional evaluation.
- (ev_004) Benchmarked VLMs include GPT-4o, Gemini 2.5 Flash-Lite, Claude Sonnet 4, Qwen2-VL, DeepSeek-VL, BLIP-2, and Gemma 3.
- (ev_005) Turning Issues: models misplace turning/orientation commands; turning is valid only at end of row.
- (ev_006) Multiplier Issues: repeat/scaling multipliers are mis-formatted and not bound to a stitch.
- (ev_007) Under-specification penalizes valid human patterns that rely on implicit conventions the DSL/renderer cannot capture.

## Limitations
- Preprint (arXiv v2, accessed 2026-06-14), not formally peer-reviewed; the third failure category "Runtime Errors" is described as low-level JavaScript parsing failures (e.g., null-read errors) and is captured here only at summary level. Crochet-specific; amigurumi/3D applicability is bounded by the CrochetPARADE DSL's expressiveness.
