# Source Card: StitchFlow: Enabling In-Situ Creative Explorations of Crochet Patterns With Stitch Tracking and Process Sharing

## Summary
StitchFlow is a UIST 2025 research system (KAIST Make Lab) that embeds a motion sensor in a crochet hook to auto-detect stitches in real time, automatically building process documentation while the maker stays immersed in their craft. It targets the friction between making and formally recording a pattern -- pausing to count stitches, take notes, and interpret notations -- which disrupts creative flow. The created designs can be viewed, edited, combined, and exported as traditional written patterns, crochet charts, or shareable in-system flows; a user study with 8 crocheters found it preserved creative flow, enabled spontaneous exploration, and eased pattern sharing.

## Key evidence
- (ev_001) Peer-reviewed Open Access full paper at UIST '25 by Marciniak, Lertjaturaphat, and Bianchi.
- (ev_002) Documentation, progress tracking, and backtracking are named as the tasks that disrupt crocheters' creative flow.
- (ev_003) A motion sensor tracks real-time hand gestures to reconstruct the stitch pattern, removing the need to remember prior steps.
- (ev_004) Auto-built documentation exports as written patterns, crochet charts, or follow-along in-system flows.
- (ev_005) User study with 8 crocheters: preserved creative flow, enabled spontaneous exploration, facilitated sharing.
- (ev_006) Pattern documentation forces makers to pause, count stitches, take notes, and interpret notation -- breaking immersive flow.
- (ev_007) Crochet patterns use two flat formats (written notation and symbol charts); fluency in one does not transfer to the other.
- (ev_008) Crochet has resisted mechanization because no machine can replicate the hand movements needed to manipulate stitches.

## Limitations
- Findings rest on a small user study (n=8), limiting generalizability.
- StitchFlow is a research prototype (sensor-augmented hook + interface), not a shipped consumer product, so claims about real-world adoption or mobile/amigurumi-OS feasibility are not directly established by this source.
