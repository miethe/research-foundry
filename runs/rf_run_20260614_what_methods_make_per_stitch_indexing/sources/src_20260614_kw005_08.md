# Source Card: A Graph Model and a Layout Algorithm for Knitting Patterns

## Summary
This arXiv preprint (Gray, Bell, Kobourov; University of Arizona, June 2024) formalizes a knitting pattern as a stitch graph with two edge types — sequential "yarn" connections and "loop" connections (loops pulled through stitches below) — and recovers stitch-creation order via a Hamiltonian path and physical yarn flow via an Eulerian path. It introduces a Desired Edge Length (DEL) metric (root-mean-square relative edge-length error) as a correctness/drift measure and a two-stage layout (planar Chrobak-Payne embedding then crossing-safe force-directed refinement). On seven real patterns it beats SFDP/ImPrEd/KnitGrid on DEL but at a steep runtime cost (up to ~40,906s on a 1,539-node pattern), motivated explicitly by faster pattern error detection.

## Key evidence
- (ev_001) Two connection types — yarn (sequential) and loop (pulled-through) — form the structural basis for per-stitch indexing.
- (ev_002) Hamiltonian path encodes stitch-creation order; Eulerian path (derived from it) encodes physical yarn flow.
- (ev_003) DEL metric = RMS of relative edge-length error, 0 = perfect preservation; a candidate per-stitch correctness/drift measure.
- (ev_004) Three-force refinement (edge-length, collision, electrostatic) with hard rejection of moves that introduce edge crossings.
- (ev_005) Seven real patterns (90-1183 nodes); proposed method wins on DEL (Antique Diamonds 0.07 vs 0.33/0.55/0.20).
- (ev_006) Steep runtime: Antique Diamonds 25s vs SFDP 0.074s; Triangle-5 (1539 nodes) 40,906s — does not scale to interactive use.
- (ev_007) Motivation: expedite error detection (too many/few stitches, misplaced adornments, unintended shape).

## Limitations
- A single-team arXiv preprint (v1, not stated as peer-reviewed); the DEL metric is proposed by the authors, not an external standard.
- Runtime figures (e.g., 40,906s) come from the authors' own benchmark table and reflect their unoptimized exact-length implementation, not a productized/interactive system.
- The ev_006 numeric runtimes are sourced from Table 4 but no single short verbatim sentence captures them, so its quote is null (values are transcribed from the table, not paraphrased prose).
