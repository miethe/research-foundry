# Source Card: CrochetPARADE Manual (Crochet PAttern Renderer, Analyzer, and DEbugger)

## Summary
The CrochetPARADE Manual is the official documentation for a crochet-pattern renderer/analyzer/debugger authored by Svetlin Tassev. It defines a per-stitch indexing scheme — (row, per-row stitch | global stitch number) with zero-based counters — and the deterministic traversal-to-geometry rules (sequential attachment to the previous row, automatic turn handling, even/odd attachment direction) that map a textual DSL pattern onto a 3D physics model. It also specifies tension/drift detection thresholds (~15% over/under-stretch flagging, ~10% physics convergence), inside-out/left-handed mirroring via a random start seed, and DSL increase/decrease syntax plus navigation controls.

## Key evidence
- (ev_001) Stitches are indexed as (row, per-row stitch | global), e.g. (4,23|410), with row and stitch counters starting from 0.
- (ev_002) The global stitch counter counts non-stitches like 'sk' but excludes internal stitches in bobbles/popcorns.
- (ev_003) Stitches are auto-attached one-by-one to consecutive previous-row stitches; turning at row end is handled automatically.
- (ev_004) Even stitch index attaches forward in the base row's crochet direction; odd index attaches to the previous stitch (reverse order).
- (ev_005) Stitches over/under-stretched by more than ~15% of baseline height are flagged (blue loose, red stretched); engine converges to within ~10% of requested lengths.
- (ev_006) Inside-out/left-handed models are corrected by changing the random start seed via 'DOT: start=2' (or other values).
- (ev_007) DSL syntax: sc2inc = two single crochets into one stitch; sc2tog = a decrease combining two; each new line is a new row/round.
- (ev_008) Row/stitch highlighting via ctrl+f; stitch-by-stitch reveal via the 'a' control.

## Limitations
- The ~15% and ~10% thresholds and inside/outside seed behavior reflect default settings and the tool's physics engine; results may vary with non-default settings. The manual is a single-author official doc (CC BY-NC-SA 4.0); no independent validation of accuracy claims is provided within it.
