# Source Card: Modeling crochet patterns with a force-directed graph layout

## Summary
Greer and Mould (Carleton University, Expressive/WICED 2025) present a tool that converts a written row-by-stitch amigurumi crochet pattern into a connectivity graph - one node per stitch, edges between adjacent and worked-into stitches - and computes a force-directed 3D layout (Isenburg et al.) that inflates the planar graph by balancing per-edge length against local curvature. The resulting 3D models match the shape and size of hand-crocheted reference objects on simple beginner-level patterns, letting designers preview a pattern without physically crocheting it. The approach is amigurumi-oriented but the authors argue it extends to clothing and other crochet styles.

## Key evidence
- (ev_001) Written pattern -> graph -> force-directed layout yields a 3D model matching hand-crocheted shape and size without physical crocheting.
- (ev_002) One node per stitch; edges encode sequential (horizontal) and worked-into (vertical) physical connections.
- (ev_003) Layout uses Isenburg et al.'s algorithm: inflate planar graph via per-edge length + minimized curvature to maximize volume (stuffing analogy).
- (ev_004) Non-linear least-squares optimization minimizing two energy functions: edge length and per-node curvature.
- (ev_005) Models were accurate in shape and size to real-world counterparts on simple patterns (similar, not identical).
- (ev_006) Constraint edges (springs) model tension/dimpling effects; pulling-in recreated by linking first and last node, paralleling how a crocheter pulls the yarn.
- (ev_007) Amigurumi-oriented but extensible to clothing and other crochet styles.
- (ev_008) Layout is slow: small pieces converge in <5s; large pieces (700-800 stitches) failed to converge after 2+ minutes.

## Limitations
- Tested only on simple, beginner-level amigurumi patterns and single-crochet-like stitches; explicitly too narrow for intermediate/advanced patterns.
- Assumes uniform edge length of 1, which does not capture real stitch dimensions (yarn tension, hook size, weight, stitch type), causing height/shape discrepancies.
- Force-directed layout is computationally expensive and may not converge on large patterns; flattened (non-stuffed) pieces are out of scope since the energy function maximizes volume.
