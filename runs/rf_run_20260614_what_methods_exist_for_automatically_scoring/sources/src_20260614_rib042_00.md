# Source Card: RAGAS: Automated Evaluation of Retrieval Augmented Generation

## Summary
RAGAS (Retrieval Augmented Generation Assessment) is a reference-free framework, introduced by Shahul Es, Jithin James, Luis Espinosa-Anke, and Steven Schockaert (arXiv 2309.15217, Sep 2023), for automatically scoring RAG pipelines without ground-truth human annotations. It defines LLM-prompted metrics along three dimensions — Faithfulness, Answer Relevance, and Context Relevance — so RAG systems can be evaluated in faster cycles. The framework is open source and integrates with LlamaIndex and LangChain.

## Key evidence
- (ev_001) RAGAS is a reference-free framework for evaluating RAG pipelines without ground-truth human annotations.
- (ev_002) It targets three dimensions: retrieval of relevant/focused passages, faithful exploitation by the LLM, and generation quality.
- (ev_003) Its stated goal is faster RAG evaluation cycles by removing the human-annotation dependency.
- (ev_004) Faithfulness holds when answer claims can be inferred from the retrieved context.
- (ev_005) Answer Relevance rewards directly addressing the question; penalizes incomplete/redundant answers.
- (ev_006) Context Relevance rewards retrieving only the information needed to answer the question.
- (ev_007) Metrics are computed by prompting an LLM (gpt-3.5-turbo-16k in the paper), fully self-contained and reference-free.
- (ev_008) RAGAS integrates with LlamaIndex and LangChain and is available as open source.

## Limitations
- arXiv preprint: v1 dates to 2023-09; a revised v2 was posted 2025-04, so specific numbers may differ between versions. Metrics in v1 were validated on the authors' own WikiEval dataset; broad generalization is not established within the paper.
