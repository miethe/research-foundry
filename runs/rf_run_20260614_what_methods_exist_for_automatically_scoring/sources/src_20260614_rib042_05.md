# Source Card: G-Eval - DeepEval (official docs)

## Summary
The official DeepEval documentation describes G-Eval, an LLM-as-a-judge metric framework derived from the paper "NLG Evaluation using GPT-4 with Better Human Alignment." G-Eval uses chain-of-thought to score LLM outputs against any custom criteria: when only criteria are given it auto-generates evaluation steps, then requests a 1-5 score and refines it using output-token probabilities (weighted summation, normalized to 0-1). DeepEval positions it as the recommended, versatile way to build task-specific subjective metrics, complementing fixed system-specific metrics like RAG and agent metrics.

## Key evidence
- (ev_001) G-Eval is an LLM-as-a-judge, chain-of-thought framework for scoring outputs against any custom criteria, from the "NLG Evaluation using GPT-4..." paper.
- (ev_002) Given only criteria, it auto-generates evaluation_steps via CoT before scoring.
- (ev_003) It asks the judge LLM for an integer score from 1 to 5 (5 better than 1).
- (ev_004) It normalizes the score using output-token probabilities via a weighted summation.
- (ev_005) DeepEval states this probability step was introduced in the paper to minimize bias in LLM scoring.
- (ev_006) It is positioned as the versatile, custom, subjective metric used alongside RAG/agent metrics.
