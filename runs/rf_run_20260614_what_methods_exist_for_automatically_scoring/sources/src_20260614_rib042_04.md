# Source Card: The RAG Triad - TruLens (official docs)

## Summary
This official TruLens documentation page defines the RAG triad: three reference-free evaluations — Context Relevance, Groundedness, and Answer Relevance — placed along the edges of the retrieval-augmented-generation architecture to detect hallucination. It explains that Groundedness works by decomposing the response into individual claims and independently checking each against the retrieved context, and that passing all three gives bounded confidence the app is hallucination-free only up to the accuracy of its underlying knowledge base.

## Key evidence
- (ev_001) The RAG triad = three evaluations (context relevance, groundedness, answer relevance); satisfactory scores together give confidence the app is hallucination-free.
- (ev_002) The triad was designed to evaluate hallucinations along each edge of the RAG architecture (motivated by failed/irrelevant retrieval).
- (ev_003) Context Relevance checks each retrieved chunk is relevant to the input query.
- (ev_004) Groundedness separates the response into individual claims and searches the retrieved context for per-claim supporting evidence.
- (ev_005) Answer Relevance verifies the final response actually answers the user's question.
- (ev_006) The hallucination-free guarantee is bounded: only valid up to the limit/accuracy of the knowledge base.
