# Source Card: RAGChecker: A Fine-grained Framework for Diagnosing Retrieval-Augmented Generation

## Summary
RAGChecker is a fine-grained, claim-level evaluation framework for retrieval-augmented generation that emits a suite of diagnostic metrics spanning overall correctness (precision, recall, F1), retriever quality (claim recall, context precision), and generator behavior (context utilization, relevant/irrelevant noise sensitivity, hallucination, self-knowledge, faithfulness). It works by decomposing responses and reference answers into claims with an LLM extractor and checking entailment with an LLM checker (Llama3-70B via RefChecker). A meta-evaluation reports higher correlation with human judgments than baselines such as RAGAS, with a 90.95% human agreement rate.

## Key evidence
- (ev_001) RAGChecker provides diagnostic metrics for both the retrieval and generation modules of a RAG pipeline.
- (ev_002) Scoring is built on claim-level entailment: extract claims from response and ground truth, then check them against reference texts.
- (ev_003) The extractor and entailment checker are both Llama3-70B implemented via the open-source RefChecker framework.
- (ev_004) Hallucination = incorrect claims unsupported by any chunk; Self-Knowledge = correct claims unsupported by retrieval (parametric vs retrieved knowledge).
- (ev_005) RAGChecker beats RAGAS in human correlation: overall 61.93 vs 48.31, correctness 49.66 vs 41.07, completeness 60.67 vs 53.16 (Pearson).
- (ev_006) Meta-evaluation shows significantly better human correlation than other metrics; human agreement rate 90.95% within tolerance.
- (ev_007) Evaluated across 8 RAG systems, 10 domains, and 4,162 queries.

## Limitations
- Self-reported gains: the authors propose the very metric they meta-evaluate, and comparisons against RAGAS are reported by them.
- Metric reliability is bounded by the LLM claim extractor/checker (Llama3-70B via RefChecker); extraction or entailment errors propagate into every downstream score.
