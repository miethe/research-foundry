# Source Card: Faithfulness - Ragas (official docs)

## Summary
The official Ragas documentation defines the Faithfulness metric, used to evaluate RAG systems, as the fraction of claims in a generated response that can be inferred from the retrieved context, yielding a 0-to-1 score. It specifies a three-step computation: extract claims, cross-check each against context, then apply the ratio. It also documents a FaithfulnesswithHHEM variant that replaces the LLM-based cross-check with Vectara's HHEM-2.1-Open T5 hallucination classifier for cheaper, open, production-oriented scoring with configurable device and batch size.

## Key evidence
- (ev_001) Ragas defines Faithfulness as factual consistency of a response with retrieved context, scored 0-1, higher is better.
- (ev_002) Score = (claims supported by context) / (total claims in response).
- (ev_003) Three-step process: identify claims, check inferability per claim, compute the ratio.
- (ev_004) Worked Einstein example: 1 of 2 statements inferable -> faithfulness = 0.5.
- (ev_005) FaithfulnesswithHHEM uses Vectara HHEM-2.1-Open (T5 hallucination classifier) for the cross-check step; free, small, open-source, production-efficient.
- (ev_006) HHEM variant exposes device and batch_size configuration; defaults to CPU with batch size 10.
