# Source Card: List of available metrics - Ragas

## Summary
The official Ragas stable documentation enumerates the framework's evaluation metrics, split into a Retrieval Augmented Generation set (Faithfulness, Context Precision, Context Recall, Context Entities Recall, Noise Sensitivity, Response Relevancy, plus Multimodal Faithfulness/Relevance) and an agent/tool-use set (Topic adherence, Tool call Accuracy, Tool Call F1, Agent Goal Accuracy). It specifies that input requirements differ across metrics — Faithfulness is reference-free while Context Recall always requires a reference — and notes that LLM-based metrics may use one or more LLM calls per score. Noise Sensitivity is defined as a robustness signal capturing how often a system errs when fed relevant or irrelevant retrieved context.

## Key evidence
- (ev_001) RAG metric set: Faithfulness, Context Precision, Context Recall, Context Entities Recall, Noise Sensitivity, Response Relevancy, plus Multimodal Faithfulness/Relevance.
- (ev_002) Noise Sensitivity measures error frequency over relevant and irrelevant retrieved documents (distractor/robustness signal).
- (ev_003) Noise Sensitivity scores 0-1, lower is better, with a selectable irrelevant-context mode.
- (ev_004) Agent/tool-use metrics added: Topic adherence, Tool call Accuracy, Tool Call F1, Agent Goal Accuracy.
- (ev_005) Faithfulness is reference-free, scoring factual consistency of response vs. retrieved context.
- (ev_006) Context Recall always requires a reference (ground truth), so reference requirements vary across metrics.
- (ev_007) LLM-based metrics may use one or more LLM calls per score — the basis for cost-per-verified-claim accounting.
