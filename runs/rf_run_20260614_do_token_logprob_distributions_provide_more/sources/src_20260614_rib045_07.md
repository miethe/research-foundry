# Source Card: Create a model response (OpenAI Responses API Reference)

## Summary
This is OpenAI's official API reference for the Responses API "create a model response" method. It documents that token logprobs are opt-in via the request `include` array value `message.output_text.logprobs`, and that the returned logprobs are nested on the assistant message's `output_text` (ResponseOutputText) content rather than under a top-level `choices.logprobs` field as in the Chat Completions API. Each logprob object exposes `token`, `bytes`, `logprob`, and a `top_logprobs` array whose nested entries each carry their own `token`, `bytes`, and `logprob`.

## Key evidence
- (ev_001) Logprobs are requested via the `include` parameter value `message.output_text.logprobs`.
- (ev_002) Returned logprobs objects contain `token`, `bytes`, `logprob`, and `top_logprobs`.
- (ev_003) Nested `top_logprobs` entries each contain `token`, `bytes`, and `logprob`.
- (ev_004) Logprobs attach to the assistant message `output_text` (ResponseOutputText), not a top-level `choices.logprobs`.

## Limitations
- The reference page is a live, undated API document; the `published_at` field could not be confirmed and is recorded as null. Field structures reflect the Responses API as accessed 2026-06-14 and may evolve; the `choices.logprobs` contrast is drawn from the differing Responses-vs-Chat-Completions request shapes documented by OpenAI.
