# Source Card: OpenAI compatibility

## Summary
Ollama's official OpenAI-compatibility documentation defines which OpenAI Chat Completions features and request fields the `/v1/chat/completions` endpoint supports. The feature matrix marks chat completions, streaming, JSON mode, reproducible outputs, vision, tools, and reasoning/thinking control as supported, but explicitly leaves Logprobs unchecked. The supported request fields list documents reasoning controls (reasoning_effort, reasoning) but includes no logprobs or top_logprobs field — making the OpenAI-compatible logprobs capability contested relative to version-specific release claims.

## Key evidence
- (ev_001) The OpenAI-compatible /v1/chat/completions docs explicitly mark Logprobs as unsupported with an unchecked checkbox.
- (ev_002) Supported features for /v1/chat/completions cover chat completions, streaming, JSON mode, reproducible outputs, vision, tools, and reasoning/thinking control, with only Logprobs unchecked.
- (ev_003) Supported request fields include model, messages, frequency_penalty, presence_penalty, response_format, seed, stop, stream, stream_options, temperature, top_p, max_tokens, and tools.
- (ev_004) reasoning_effort and reasoning are documented supported fields with effort values high, medium, low, none.
- (ev_005) No top_logprobs field is documented; explicitly unsupported request fields are tool_choice, logit_bias, user, and n.
- (ev_006) Documented OpenAI-compatible endpoints include /v1/chat/completions, /v1/completions, /v1/models, /v1/embeddings, /v1/images/generations, and /v1/responses.

## Limitations
- No visible published or updated date on the page, so recency relative to specific Ollama releases cannot be confirmed.
- The documented support matrix conflicts with v0.12.11 release notes claiming logprobs support, so the OpenAI-compatible logprobs cell is version-dependent/contested and should be cross-checked against the running binary's version.
