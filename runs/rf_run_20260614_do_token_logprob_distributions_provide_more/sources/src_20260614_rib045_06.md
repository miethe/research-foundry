# Source Card: Using logprobs (OpenAI Cookbook)

## Summary
This OpenAI Cookbook example, authored by OpenAI staff James Hills and Shyamal Anadkat (published 2023-12-20), documents the Chat Completions `logprobs` and `top_logprobs` API parameters and their token-level output. `logprobs` is a boolean that returns the log probability of each output token; `top_logprobs` is an integer (0-5) that returns that many most-likely alternative tokens at each position and requires `logprobs=true`. It enumerates five concrete use cases (classification, retrieval/Q&A evaluation, autocomplete, token highlighting/bytes, and perplexity) with worked examples using gpt-4o, gpt-4o-mini, and gpt-4.

## Key evidence
- (ev_001) The logprobs parameter is a boolean returning the log probability of each output token in message content.
- (ev_002) top_logprobs is an integer 0-5 for the number of most-likely tokens per position, requiring logprobs=true.
- (ev_003) Five enumerated use cases: classification, retrieval (Q&A) evaluation, autocomplete, token highlighting/bytes, and perplexity.
- (ev_004) Each token includes a bytes field (ASCII/UTF-8 encoding) useful for reproducing emojis and special characters.
- (ev_005) logprobs give a per-class probability, enabling user-defined classification/confidence thresholds.
- (ev_006) Perplexity is calculated by exponentiating the negative average of the logprobs.
