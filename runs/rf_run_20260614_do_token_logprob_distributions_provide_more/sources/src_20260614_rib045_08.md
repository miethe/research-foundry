# Source Card: Unlock Gemini's reasoning: A step-by-step guide to logprobs on Vertex AI

## Summary
This official Google Developers Blog tutorial (July 16, 2025) explains how to obtain and use token log probabilities (logprobs) from Gemini models on Vertex AI. It documents the API controls (`response_logprobs=True` and an integer `logprobs` parameter from 1 to 20), the structure of the returned `logprobs_result` (chosen tokens and top alternatives), and demonstrates use cases such as confidence-based classification triage and quantitative RAG grounding/confidence scoring, all using the gemini-2.5-flash model.

## Key evidence
- (ev_001) A logprob is the natural log of the probability the model assigned to a token.
- (ev_002) Logprob scores closer to 0 indicate higher model confidence.
- (ev_003) `response_logprobs=True` returns log probabilities for the chosen output tokens.
- (ev_004) The `logprobs` parameter accepts an integer 1-20 for top alternative tokens per step.
- (ev_005) Examples use the gemini-2.5-flash model.
- (ev_006) A small gap between the top two log probabilities signals classification ambiguity.
- (ev_007) Averaging answer log probabilities yields a RAG grounding/confidence score.

## Limitations
- Vendor-authored tutorial; demonstrates capabilities on Vertex AI specifically and uses gemini-2.5-flash, so parameter behavior and ranges may differ across other Gemini surfaces or model versions. Claims about confidence/ambiguity reflect the author's recommended interpretation, not an independent empirical benchmark.
