# Source Card: llama.cpp server README (tools/server)

## Summary
The official README for the llama.cpp HTTP server documents how to obtain per-token probability distributions from generation. Its native POST `/completion` endpoint exposes an `n_probs` request option that returns top-N token probabilities, surfaced in a `completion_probabilities` response array whose nested `top_logprobs` entries carry `id`, `logprob`, `token`, and `bytes`. Setting `post_sampling_probs=true` swaps `logprob` for a 0.0-1.0 `prob` (and `top_logprobs` for `top_probs`), and these probability features are documented for `/completion`, not for the OpenAI-compatible `/v1/chat/completions` endpoint.

## Key evidence
- (ev_001) n_probs > 0 makes /completion return top-N token probabilities per generated token under the sampling settings.
- (ev_002) For temperature < 0, tokens are sampled greedily but probabilities are still computed via plain softmax over the logits.
- (ev_003) completion_probabilities is length-n_predict; each item's top_logprobs holds at most n_probs entries with id/logprob/token/bytes.
- (ev_004) post_sampling_probs=true replaces logprob with prob (0.0-1.0) and top_logprobs with top_probs.
- (ev_005) The number of elements in top_probs may be fewer than n_probs.
- (ev_006) These probability fields are documented under /completion only; /v1/chat/completions just defers to OpenAI docs plus /completion-specific features like mirostat.

## Limitations
- README tracks the `master` branch and carries no publication date, so wording and field names may drift from any pinned release; verify against the server version actually deployed. The absence of probability-field documentation under `/v1/chat/completions` is an inference from its omission in that section, not an explicit statement that the feature is unavailable there.
