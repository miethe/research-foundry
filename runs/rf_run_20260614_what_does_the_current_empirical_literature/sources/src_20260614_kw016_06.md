# Source Card: KnitWit Research Plan (local seed, project primary context)

## Summary
This is the KnitWit project's primary internal research-plan seed (authored by ChatGPT 5.2, updated 2026-04). It defines two research tracks - a forward Pattern->3D visualizer and an inverse 3D-object->pattern generator - and commits the project to structural geometry before yarn physics and an amigurumi-first inverse pipeline. It establishes a Crochet IR with explicit validation rules, names inverse-engine preconditions (watertight mesh + seed point + stitch size) and output constraints (no impossible stitch-count transitions, bounded inc/dec rate), and proposes acceptance metrics for both topological correctness and shape match.

## Key evidence
- (ev_001) The deep dive is split into two tracks: forward Pattern->3D preview and inverse 3D-object->pattern synthesis.
- (ev_002) 3D preview is scoped as structural geometry first (graph/mesh), and the inverse track as amigurumi-first with watertight mesh + seed point + stitch size.
- (ev_003) Crochet IR validation rules: stitch-count invariants per round, increase/decrease legality, and piece-closure constraints.
- (ev_004) Inverse-engine preconditions are a closed/watertight triangle mesh plus a user-selected seed point and target stitch size/gauge.
- (ev_005) Inverse output must guarantee no impossible stitch-count transitions and a bounded increase/decrease rate.
- (ev_006) Proposed inverse acceptance metrics: shape distance vs target mesh plus pattern-complexity metrics (max inc/dec rate, total rounds, stitch-count variance).
- (ev_007) Forward visualizer topological correctness requires a consistent stitch graph with no broken loops and correct per-round stitch counts.
- (ev_008) Primary risks flagged: the 'PDF importer trap' (restrict to structured authoring/import templates) and gauge/tension variability.

## Limitations
