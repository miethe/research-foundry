# Source Card: CrochetBench (official code & data repository)

## Summary
CrochetBench is the official benchmark repository (by Peiyu-Georgia-Li) for evaluating multimodal LLMs on fine-grained procedural reasoning in crochet, moving from "describing to doing." It defines four tasks (stitch recognition, instruction selection, instruction generation, and instruction-to-DSL translation) and uniquely uses the CrochetPARADE DSL with a compilation-based validator so a generator/validator can be functionally evaluated rather than only trained. Data is released as structured JSON test files plus patterns organized by project type, with a Copyright Permission notice covering the original pattern sources.

## Key evidence
- (ev_001) CrochetBench evaluates multimodal LLMs on fine-grained, low-level procedural reasoning in crochet.
- (ev_002) Models must recognize stitches, select instructions, generate instructions, and produce compilable procedures.
- (ev_003) Adopts the CrochetPARADE DSL as an intermediate representation with functional/program-execution evaluation.
- (ev_004) Task A (CrochetBench-A) is stitch recognition scored by F1/Precision/Recall at test size 6,009.
- (ev_005) Task D requires compilable CrochetPARADE programs scored by Compilation Success Rate — usable to validate a generator/validator.
- (ev_006) Data ships as structured JSON test files plus patterns organized by project type.
- (ev_007) A Copyright Permission section indicates permission was required for the original pattern sources.
- (ev_008) Task D validation depends on the external CrochetPARADE repo; data is collected via a project-type CLI scraper.
