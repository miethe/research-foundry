# Source Card: KnitWit Design Spec (local seed, project primary context)

## Summary
This is KnitWit's own LLM-authored design memo (ChatGPT 5.2, ~Apr 2026) that frames the product as "Ravelry + interactive pattern runner + CAD-lite for amigurumi." It establishes a four-level 3D capability ladder (L0 to L3), names interactive structural 3D plus guided assembly as the differentiation wedge against existing pattern/tracker apps, sketches a Crochet intermediate representation (IR) for playback and 3D mapping, and flags pattern IP/copyright as the top business risk. It recommends an amigurumi-first MVP and defers inverse 3D-to-pattern synthesis to Phase 3.

## Key evidence
- (ev_001) Four-level capability ladder L0 (2D + counters) -> L1 (structural 3D preview) -> L2 (editable 3D, consistent stitch semantics) -> L3 (3D object -> pattern synthesis).
- (ev_002) Differentiation wedge is interactive 3D + guided assembly, since the market already has strong pattern/tracking apps (Ribblr, My Row Counter).
- (ev_003) First 3D is structural geometry, not yarn-physics; must be messaged as "structural preview" to avoid implying photoreal accuracy.
- (ev_004) Crochet IR sketch: units/gauge/hook/yarn-weight, pieces[], rows[] as stitch ops with counts, per-row expected-count constraints/checkpoints, and assembly join/placement markers.
- (ev_005) Pattern IP/copyright is the #1 business landmine, arguing for designer opt-in / licensing / user-private imports.
- (ev_006) Amigurumi-first positioning (pieces, assembly, shaping, symmetry checks) is where 3D guidance delivers the most user value.
- (ev_007) Inverse 3D-object-to-pattern generation is deferred to Phase 3; computationally grounded in peer-reviewed work but unproven as a delightful product.
