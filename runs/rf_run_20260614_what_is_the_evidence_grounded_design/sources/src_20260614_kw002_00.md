# Source Card: CrochetPARADE Manual (grammar specification) — Svetlin Tassev

## Summary
The CrochetPARADE manual is the authoritative specification of a custom, unambiguous grammar for describing crochet patterns that the platform parses, checks, and renders into 2D/3D models. It defines line-based row/round semantics, repeat blocks with angle markers, automatic attachment and turn handling, a labeling/attachment-point system, counter variables, and a raw shorthand for defining new stitches (including increases and disjoint-piece markers). The manual is CC BY-NC-SA and the code GPLv3, while the grammar itself is stated to be in the public domain.

## Key evidence
- (ev_001) Rows and rounds are not distinguished by a keyword; each new text line is a new row/round.
- (ev_002) Repeat blocks support `>`/`<` angle markers for early end / late start of iterations, plus `...` line wrap.
- (ev_003) Consecutive auto-attachment to the previous row, with turning handled automatically.
- (ev_004) Counter variables initialized via `$k=0$` with `++`/`--`/`prev`/`next` and index algebra.
- (ev_005) Increase stitches use a `Ninc` shorthand (e.g. `sc2inc`) packing multiple top nodes onto one base.
- (ev_006) `start_anew` uses a `skip` connection length to create disjoint pieces (new part of the project).
- (ev_007) Manual is CC BY-NC-SA / code GPLv3, but the grammar itself is public domain.

## Limitations
- The manual is descriptive prose with informal/placeholder section headings (e.g. a section literally titled "tttttt"), not a formal BNF grammar.
- The text reviewed does not contain a comprehensive built-in stitch keyword list, an explicit "Ninc/Ntog" naming rule, nor `start_a_new_chain` / `TRANSFORM_OBJECT` constructs; only `sc2inc` and decrease examples like `dc2tog` appear, so the increase/decrease convention is inferred from examples rather than a stated rule.
