# Source Card: Modeling Crochet Patterns with a Force-directed Graph Layout

## Summary
This Eurographics Expressive/WICED 2025 paper by Emile Greer and David Mould (Carleton University) presents a method that translates a written row-by-stitch crochet pattern into a graph and computes a force-directed graph layout to produce a 3D amigurumi model accurate in shape and size. Stitches become nodes with horizontal (sequential) and vertical (working) edges plus optional constraint edges for non-stitch shaping; a non-linear least-squares layout (Ceres Solver) blends edge-length and curvature energies via a lambda parameter to "inflate" the planar graph like stuffing. The work is open source and open access, with explicitly stated scope limits.

## Key evidence
- (ev_001) Graph IR: each stitch is a node; horizontal edges = sequential connection, vertical edges = working connection.
- (ev_002) Constraint edges encode non-stitch shaping (e.g. pulling tight to dimple); apple uses a trial-and-error edge length of 6.
- (ev_003) Rows and rounds are treated identically; first-row sc nodes share an anchor ring node, later rows work into the previous row.
- (ev_004) Layout = non-linear least-squares via Ceres Solver minimizing (1-lambda)*edge-length + lambda*curvature, lambda=0.65 default (0.9 for bunny head/body).
- (ev_005) Limitations: constant edge length 1 (swatch measurement needed), only beginner/sc-like stitches tested, ad hoc piece-joining.
- (ev_006) Open-source code at github.com/EnbyMonkey/modelingamigurumi.git; open-access CC-BY article.

## Limitations
- Scope is narrow: tested only on beginner-level, single-crochet-like patterns; flattened (non-stuffed) pieces and intermediate/advanced stitches are out of scope. The constant-edge-length-1 assumption limits dimensional accuracy without per-yarn swatch measurement, and piece-joining is currently ad hoc rather than systematic.
