# Source Card: Digital Crochet: Toward a Visual Language for Pattern Description

## Summary
This peer-reviewed ACM SIGPLAN Onward! '22 paper (Seitz, Rein, Lincke, Hirschfeld; Hasso Plattner Institute / University of Potsdam) proposes the first visual, domain-specific, graph-based language for representing crochet patterns. Stitches/loops are modeled as graph nodes ("insertion points") and the worked-into relationship is captured by directed edges (previous / insertion / slip-stitch), with increases and decreases encoded as edge multiplicity and rows/rounds unified into a single "layers" abstraction. It contributes a prototype editor (2D editing, 3D force-directed viewing) and a qualitative think-aloud study with six professional crochet designers that found the structured representation surfaced ambiguities present in standard chart notation.

## Key evidence
- (ev_001) The paper proposes the first visual, graph-based, domain-specific language for unambiguous crochet pattern representation, plus a 2D/3D editor prototype and a user evaluation.
- (ev_002) A stitch is modeled as a graph node (an "insertion point"); the language encodes pattern structure and dependencies rather than a linear stitch sequence.
- (ev_003) Edges encode the worked-into relationship: "previous" edges trace yarn origin (exactly one outgoing per node) and "insertion" edges point to the insertion points of the current stitch.
- (ev_004) Increases = multiple incoming insertion edges to one insertion point; decreases = multiple outgoing insertion edges from one stitch node.
- (ev_005) Rows and rounds are unified into a single "layers" abstraction (stored as a per-node property) so flat and circular work share one graph structure.
- (ev_006) The crochetability invariant forbids using an insertion point that is only created in the future ("no use before create"); 2D/3D views use a force-based / 3D force-directed layout heuristic.
- (ev_007) Six professional crochet designers: all but one reproduced the pattern in 2D; four cited the shape preview as beneficial and several valued the 3D view for understanding shape (e.g., amigurumi proportions).

## Limitations
- Evaluation is a qualitative think-aloud study with only six designers (recruited from a 25-person formative-survey pool); it is not a quantitative accuracy or productivity benchmark.
- The editor and its force-based 3D layout are described as a prototype/approximation; the authors note the force-based layout is only an approximation of the actual 3D form and that the moving layout disoriented some users.
