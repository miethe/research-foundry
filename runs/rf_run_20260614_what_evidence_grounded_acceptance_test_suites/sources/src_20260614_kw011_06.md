# Source Card: Digital Crochet: Toward a Visual Language for Pattern Description

## Summary
This peer-reviewed Onward!/SPLASH 2022 paper (Seitz, Rein, Lincke, Hirschfeld; Hasso Plattner Institute, Univ. Potsdam) proposes the first visual, domain-specific, graph-based language for representing crochet patterns. It frames existing crochet pattern languages as defective because they are either ambiguous or limited in expressiveness, and positions a formal graph-based intermediate representation as the enabler of machine validation and domain-specific tooling. The authors validate the approach with a prototype editor (author in 2D, preview in 3D) and a qualitative user study of six professional designers, showing the language expresses both 2D and 3D patterns and removes ambiguities found in standard chart notations.

## Key evidence
- (ev_001) Proposes the first visual, domain-specific, graph-based language for crochet pattern representation.
- (ev_002) Existing crochet pattern languages are constrained because they are either ambiguous or limited in expressiveness.
- (ev_003) User study shows the language expresses both 2D and 3D patterns and removes ambiguities in current standard notations.
- (ev_004) Demonstrated via a prototype editor: author patterns in 2D, view them in 3D (the KnitWit author-then-preview loop).
- (ev_005) Existing graphical notations lack a formal, structured representation of stitch relations, limiting tool support.
- (ev_006) Even standardized crochet charts are ambiguous because insertion points rely only on the visual direction of stitch symbols.
- (ev_007) The user study used six professional crochet designers recruited from a formative survey of myboshi designers.
- (ev_008) Participants confirmed that even small, well-arranged state-of-the-art charts contained concrete ambiguities and missing information.

## Limitations
- Evaluation is a small (n=6) qualitative think-aloud user study with professional designers, not a quantitative benchmark; findings are about expressiveness and disambiguation, not measured accuracy or speed. The language and editor are described as a prototype/foundation ("Toward"), with some generation features (e.g., repetitions across layers, absolute insertion-point references) not yet supported.
