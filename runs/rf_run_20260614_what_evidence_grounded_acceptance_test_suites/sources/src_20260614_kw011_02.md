# Source Card: CrochetBench: Can Vision-Language Models Move from Describing to Doing in Crochet Domain?

## Summary
CrochetBench (Li, Huang, Hua, Chawla; arXiv:2511.09483v1, Nov 2025) is the first executable benchmark for procedural crafts, evaluating multimodal models on crochet via a four-task ladder (recognition, comprehension, generation, and instruction-to-DSL translation). It distinguishes describing from doing and grounds acceptance in functional executability: model outputs are translated into the CrochetPARADE DSL and scored by whether they compile against a validator that checks stitch-count consistency, structural correctness, and physical feasibility. This supplies a concrete, evidence-grounded template for acceptance-test families (compilation success, fine-grained error taxonomy, validator-based physical checks) directly reusable for evaluating generated crochet/knit instructions.

## Key evidence
- (ev_001) CrochetBench shifts evaluation from describing to doing: recognize stitches, select structurally appropriate instructions, generate compilable procedures.
- (ev_002) Evaluation centers on instructional fidelity - step-wise, compilable instructions respecting symbolic, numerical, and topological structure.
- (ev_003) A four-task ladder (A-D) progresses from recognition through comprehension and generation to executable synthesis.
- (ev_004) Acceptance for DSL outputs = Compilation Success Rate via the CrochetPARADE validator (executable correctness, not surface similarity).
- (ev_005) Fine-grained error analysis over four failure categories: syntax structure, stitch definition, labeling/reference, structural/formatting.
- (ev_006) The validator checks consistency errors (mismatched stitch counts, impossible attachments) and flags over-/under-stretched stitches - directly reusable acceptance checks.
- (ev_007) Reference patterns sourced from Yarnspirations (6,085 patterns, 55 categories, 98.77% with images), normalized to JSON.
- (ev_008) Performance decays sharply moving from surface-level fidelity (BLEU/ROUGE) to structural validity (compilation), motivating execution-grounded metrics.

## Limitations
- Provided source brief contained inaccuracies corrected here: the paper does NOT name "four validation check families," "dimensional accuracy," or a "CrochetParade" pattern repository - CrochetPARADE is a DSL renderer/analyzer/debugger (crochetparade.org), and the pattern source is Yarnspirations (not "YarnInspirations.com").
- Project-level DSL evaluation uses a small sample (100 items) with low absolute validity rates; authors caution against fine-grained statistical comparisons.
- Compilation-based acceptance cannot assess visual fidelity unless paired with image-render verification; Task C metrics rely on string overlap and may underestimate semantically correct outputs.
