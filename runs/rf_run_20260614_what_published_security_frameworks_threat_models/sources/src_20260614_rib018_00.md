# Source Card: LLM06:2025 Excessive Agency — OWASP Top 10 for LLM Applications

## Summary
This is the OWASP Gen AI Security Project's 2025 Top-10-for-LLM-Applications entry on Excessive Agency: the risk that an LLM-based system granted agency (the ability to call functions or interface with other systems via extensions) performs damaging actions from unexpected, ambiguous, or manipulated outputs. OWASP attributes the vulnerability to three root causes — excessive functionality, excessive permissions, and excessive autonomy — and prescribes least-privilege controls including minimum-necessary extension permissions, per-user authorization/security scope, avoidance of open-ended extensions, OAuth with minimum scope, and human-in-the-loop approval of high-impact actions.

## Key evidence
- (ev_001) OWASP defines Excessive Agency as the vulnerability enabling damaging actions from unexpected, ambiguous, or manipulated LLM outputs, regardless of the malfunction's cause.
- (ev_002) Three root causes: excessive functionality, excessive permissions, excessive autonomy.
- (ev_003) Control: limit LLM-extension permissions to other systems to the minimum necessary.
- (ev_004) Control: track user authorization and security scope so actions execute in that specific user's context with minimum privileges.
- (ev_005) Control: avoid open-ended extensions (e.g., run a shell command, fetch a URL) in favor of granular functionality.
- (ev_006) Control: human-in-the-loop approval required for high-impact actions.
- (ev_007) Control: OAuth with minimum scope for an extension that reads a user's code repo.
