# Source Card: MITRE ATLAS Credential Access (AML.TA0013) — tactic and techniques

## Summary
This Startup Defense reference page restates MITRE ATLAS tactic AML.TA0013 (Credential Access), defined as adversary efforts to steal account names, passwords, or API keys for AI systems. It enumerates the constituent techniques, including AI-specific vectors such as AML.T0083 (Credentials from AI Agent Configuration), AML.T0082 (RAG Credential Harvesting), and AML.T0098 (AI Agent Tool Credential Harvesting), alongside conventional ones like AML.T0055 (Unsecured Credentials), AML.T0090 (OS Credential Dumping), and AML.T0106 (Exploitation for Credential Access). It is useful as a published threat-model taxonomy for agentic/GenAI credential-access risks.

## Key evidence
- (ev_001) AML.TA0013 Credential Access covers adversary efforts to steal account names, passwords, or API keys for AI systems.
- (ev_002) AML.T0083 'Credentials from AI Agent Configuration': accessing credentials of other tools/services from an AI agent's configuration.
- (ev_003) AML.T0082 'RAG Credential Harvesting': using LLM access on the victim system to collect credentials.
- (ev_004) AML.T0098 'AI Agent Tool Credential Harvesting': using AI-agent access to retrieve data from available agent tools.
- (ev_005) AML.T0055 'Unsecured Credentials': finding and obtaining insecurely stored credentials on compromised systems.
- (ev_006) AML.TA0013 also includes AML.T0090 'OS Credential Dumping' and AML.T0106 'Exploitation for Credential Access'.
