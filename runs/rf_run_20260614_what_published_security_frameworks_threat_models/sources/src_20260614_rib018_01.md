# Source Card: Agentic AI - Threats and Mitigations (OWASP Agentic Security Initiative)

## Summary
This is the OWASP Agentic Security Initiative's (ASI) first published guide on agentic AI security, a threat-model-based reference covering emerging agentic threats and mitigations under a CC BY-SA 4.0 license. It frames agentic AI around LLM-based reasoning, persistent or session memory, and tool use, then enumerates a reference threat model (including Confused Deputy and the T3 "Privilege Compromise" threat) with mitigation playbooks. The landing page advertises v1.0 (2025-02-17); the PDF served at the same URL is the revised Version 1.1 (December 2025).

## Key evidence
- (ev_001) First-in-a-series ASI guide providing a threat-model-based reference of agentic threats and mitigations.
- (ev_002) Agentic capability model: LLM reasoning, session-based short-term or persistent long-term memory, and tool use.
- (ev_003) Confused Deputy vulnerability: an over-privileged agent tricked into unauthorized actions on the user's behalf.
- (ev_004) Threat T3 Privilege Compromise: exploiting permission-management weaknesses, often via dynamic role inheritance/misconfiguration.
- (ev_005) Recommends clear identity flows, strict RBAC, and a zero-trust model for agent access.
- (ev_006) Mitigation: limit AI credential persistence; credentials temporary and short-lived.
- (ev_007) Mitigation: granular RBAC/ABAC and no cross-agent privilege delegation unless explicitly authorized.
- (ev_008) Mitigation: session isolation of memory so agents do not carry knowledge across user sessions.

## Limitations
- The document self-describes as a "first report" with planned follow-up role-based guides; taxonomy labels differ from the later OWASP Top 10 for Agentic Applications (e.g., "Privilege Compromise"/"Confused Deputy" vs. "Identity and Privilege Abuse").
- Version ambiguity: landing-page metadata says v1.0 (Feb 2025) while the linked PDF self-identifies as Version 1.1 (December 2025).
