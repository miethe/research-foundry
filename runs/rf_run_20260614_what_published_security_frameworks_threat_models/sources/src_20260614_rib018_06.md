# Source Card: LLM02:2025 Sensitive Information Disclosure — OWASP Top 10 for LLM Applications

## Summary
This is the primary OWASP GenAI Security Project entry for the LLM02:2025 "Sensitive Information Disclosure" risk in the 2025 OWASP Top 10 for LLM Applications. It defines the sensitive-data classes at risk (PII, financial, health, confidential business data, security credentials, legal documents), enumerates common vulnerabilities and example attack scenarios, lists prevention/mitigation strategies (sanitization, least-privilege access, federated learning, differential privacy), and maps the risk to MITRE ATLAS taxonomy techniques.

## Key evidence
- (ev_001) OWASP defines LLM02:2025 sensitive information as spanning PII, financial details, health records, confidential business data, security credentials, and legal documents.
- (ev_002) LLMs embedded in applications risk exposing sensitive data, proprietary algorithms, or confidential details through output, leading to unauthorized access, privacy violations, and IP breaches.
- (ev_003) PII leakage is a listed common vulnerability: PII may be disclosed during interactions with the LLM.
- (ev_004) Proprietary algorithm exposure is exemplified by the Proof Pudding attack (CVE-2019-20634), where revealing training data enables inversion attacks.
- (ev_005) Mitigation: limit model access to external data sources and securely manage runtime data orchestration.
- (ev_006) The risk maps to MITRE ATLAS AML.T0024.000/.001/.002 (training-data membership inference, model inversion, model extraction).
- (ev_007) The lead mitigation is data sanitization to keep user data out of the training model, with strict input validation.

## Limitations
- The page itself does not explicitly state the 2023→2025 renumbering (the claim that this was "LLM06" in 2023 must be sourced from the OWASP edition-comparison material, not this entry).
- The page's MITRE ATLAS mappings concern model inversion/extraction and training-data membership inference (AML.T0024.*), not an explicit "RAG credential harvesting" technique; do not attribute that specific framing to this source.
- Community-authored consensus document; descriptive guidance rather than empirical measurement.
