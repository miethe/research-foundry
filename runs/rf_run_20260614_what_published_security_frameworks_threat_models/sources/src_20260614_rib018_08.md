# Source Card: MCP Security Best Practices (2025-11-25)

## Summary
This is the official Model Context Protocol (2025-11-25) Security Best Practices specification, complementing the MCP Authorization spec. It enumerates concrete attack vectors against MCP implementations — confused deputy, token passthrough, SSRF, session hijacking, local server compromise, and scope inflation — and prescribes normative (MUST/SHOULD) mitigations for each. It establishes hard rules such as the explicit prohibition on token passthrough and the requirement for per-client consent in OAuth proxy servers.

## Key evidence
- (ev_001) Token passthrough is explicitly forbidden: an MCP server MUST NOT accept tokens not issued for it.
- (ev_002) The confused-deputy attack requires four concurrent conditions (static client ID, dynamic registration, third-party consent cookie, no per-client consent).
- (ev_003) MCP proxy servers MUST maintain a per-user registry of approved client_id values and check it before forwarding.
- (ev_004) Scope minimization: start minimal (e.g. mcp:tools-basic), elevate via WWW-Authenticate scope challenges, avoid wildcard/omnibus scopes.
- (ev_005) Token passthrough risks include security-control circumvention, broken audit trails, and trust-boundary/lateral-access violations.
- (ev_006) Local MCP servers should run sandboxed with minimal default privileges and restricted FS/network access.
- (ev_007) Local HTTP-transport servers should require an auth token or use unix domain sockets / restricted IPC.

## Limitations
- These are normative best practices and threat models from the protocol spec, not empirical incident data; effectiveness depends on correct implementation. Scoped to the 2025-11-25 spec revision.
