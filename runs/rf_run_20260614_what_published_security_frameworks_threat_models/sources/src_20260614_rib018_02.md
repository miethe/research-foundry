# Source Card: MCP02:2025 – Privilege Escalation via Scope Creep (OWASP MCP Top 10)

## Summary
This OWASP MCP Top 10 (2025) entry defines "privilege escalation via scope creep" as the gradual expansion of narrowly scoped MCP agent/tool permissions—through convenience or configuration drift—until an agent holds administrative privileges, a risk amplified by autonomous, multi-system agents operating without human review. The page enumerates a set of remediation controls including least privilege by design, per-agent identity and credential binding, expiry-based/JIT access, policy-as-code enforcement, automated entitlement reviews with drift detection, and separation of duties with human approval flows. These controls directly mirror Research Foundry's governance model of per-profile keys, scoped access, and policy gating.

## Key evidence
- (ev_001) OWASP defines scope creep as agent permissions expanding over time via convenience or drift until the agent holds broad/admin privileges.
- (ev_002) Least Privilege by Design: define minimal per-agent permissions before deployment; map documented actions to explicit scopes.
- (ev_003) Per-Agent Identity & Credential Binding: unique agent identities, credentials bound to agent/session, no shared service accounts.
- (ev_004) Expiry-Based & JIT Access: issue time-limited scopes/tokens; require revalidation for long-running tasks.
- (ev_005) Policy-as-Code & Automated Enforcement: encode policies (Rego/OPA/Terraform IAM) and enforce in CI/CD.
- (ev_006) Automated Entitlement Reviews & Drift Detection: periodic audits for scope expansion; alert on permission increases.
- (ev_007) Separation of Duties & Approval Flows: separate grant authority from deploy authority; require human approval for non-routine grants.

## Limitations
- Controls are specified at the policy/standard level without reference implementations or empirical efficacy data; this page is a threat-and-control taxonomy, not a measurement study.
