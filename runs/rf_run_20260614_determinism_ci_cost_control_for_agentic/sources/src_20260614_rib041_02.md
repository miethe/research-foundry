# Source Card: Advanced usage (seed and system_fingerprint) - OpenAI API

## Summary
OpenAI's "Advanced usage" API guide documents that Chat Completions are non-deterministic by default and that reusing a fixed integer `seed` while holding all other request parameters constant yields only "(mostly) deterministic" outputs. Determinism is further conditioned on OpenAI's own backend configuration, which it surfaces through the `system_fingerprint` field: when that value changes, identical requests may produce different outputs. The guide frames reproducibility as best-effort, never a hard guarantee.

## Key evidence
- (ev_001) Chat Completions are non-deterministic by default; identical requests can return different outputs.
- (ev_002) A reused integer `seed` across requests produces (mostly) deterministic outputs.
- (ev_003) Determinism requires all other parameters (e.g., prompt, temperature) to be identical across requests.
- (ev_004) OpenAI's own model-configuration changes can break determinism; tracked via `system_fingerprint`.
- (ev_005) A differing `system_fingerprint` signals outputs may differ despite a fixed seed and params.
- (ev_006) OpenAI explicitly qualifies the behavior as "(mostly) deterministic," not guaranteed.

## Limitations
- No publication date is visible on the page; OpenAI may revise the guidance over time.
- The section is brief and describes intended/best-effort behavior, not a measured or contractual guarantee; the source does not quantify the residual non-determinism rate, nor explicitly list `top_p` among the parameters to hold constant.
