# Source Card: Introducing Pytest and Vitest integrations for LangSmith Evaluations

## Summary
A LangChain product-announcement blog post (Jan 22, 2025) introducing beta integrations that let teams author LLM evaluations directly inside standard test frameworks: Pytest in Python (via the `@pytest.mark.langsmith` decorator) and Vitest/Jest in TypeScript (via `ls.describe()` blocks), with inputs/outputs/feedback logged to LangSmith. It positions test frameworks as a CI-native path for evals: they naturally express pass/fail criteria and raise assertion errors, so score regressions can gate a CI pipeline, and a dedicated GitHub Action was promised as forthcoming.

## Key evidence
- (ev_001) Integrations shipped in beta with v0.3.0 of the LangSmith Python and TypeScript SDKs (Jan 22, 2025).
- (ev_002) Python tracks a test in LangSmith via the `@pytest.mark.langsmith` decorator.
- (ev_003) TypeScript tracks test cases by wrapping them in an `ls.describe()` block.
- (ev_004) Test frameworks naturally support pass/fail criteria and assertion errors in CI, catching regressions early.
- (ev_005) LangSmith supports logging feedback and comparing results over time to prevent regressions.
- (ev_006) A dedicated GitHub Action for configuring eval runs in CI was announced as forthcoming.

## Limitations
- Vendor product-announcement blog, not neutral technical documentation; framing is promotional.
- Features described as shipping "in beta" with v0.3.0; behavior and API surface may have changed in later SDK versions.
- The GitHub Action was forthcoming ("in the coming weeks") at publish time and is not described as released in this source.
