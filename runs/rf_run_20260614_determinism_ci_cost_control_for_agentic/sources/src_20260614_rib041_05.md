# Source Card: openai/evals — A framework for evaluating LLMs and LLM systems

## Summary
openai/evals is OpenAI's open-source (MIT) framework and registry for evaluating LLMs and LLM-based systems. It supports both deterministic ground-truth eval templates (Match, Includes, FuzzyMatch, JsonMatch) and model-graded evals that delegate judgment to a model for open-ended outputs, and it generalizes to prompt chains and tool-using agents via the Completion Function Protocol. Completion functions and the eval registry are configured as YAML and invoked through the oaieval CLI, with registry data stored in Git-LFS.

## Key evidence
- (ev_001) Evals is a framework for evaluating LLMs or systems built using LLMs.
- (ev_002) The Completion Function Protocol extends evals to prompt chains or tool-using agents.
- (ev_003) Completion functions standardize text/Chat inputs and list-of-string outputs, generalizing model completions.
- (ev_004) Basic eval templates provide deterministic ground-truth checks for low-variation responses.
- (ev_005) Model-graded evals delegate judgment to a model for high-variation, open-ended responses.
- (ev_006) Completion functions register as YAML under evals/registry/completion_fns and run via the oaieval CLI by registered key.
- (ev_007) The evals registry is stored using Git-LFS.

## Limitations
