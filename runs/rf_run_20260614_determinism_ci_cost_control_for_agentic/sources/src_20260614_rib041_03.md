# Source Card: How to make your completions outputs consistent with the new seed parameter

## Summary
This official OpenAI Cookbook recipe is the canonical how-to for the `seed` + `system_fingerprint` reproducibility feature. It instructs developers to send the same seed integer and identical request parameters to obtain consistent Chat Completion outputs, and to monitor the `system_fingerprint` response field as a signal that backend configuration changed. Throughout, it frames determinism as a best-effort, "(mostly)" identical behavior rather than a hard guarantee.

## Key evidence
- (ev_001) Seed makes the system attempt deterministic sampling so same seed + same parameters should return the same result.
- (ev_002) Set the seed to any integer but reuse the same value across requests.
- (ev_003) Keep all other parameters (prompt, temperature, top_p, etc.) identical across requests.
- (ev_004) Matching seed + parameters + system_fingerprint yields outputs that are mostly identical.
- (ev_005) Outputs can still differ even when parameters and system_fingerprint match, due to inherent model non-determinism.
- (ev_006) The feature is framed as delivering only "(mostly) consistent" outputs, i.e. best-effort, not guaranteed.

## Limitations
- The source frames reproducibility as best-effort ("mostly identical"), not a hard guarantee. Original 2023-11 article scoped the feature to beta models (gpt-4-1106-preview, gpt-3.5-turbo-1106); model coverage has since broadened, so model-availability specifics are time-bound.
