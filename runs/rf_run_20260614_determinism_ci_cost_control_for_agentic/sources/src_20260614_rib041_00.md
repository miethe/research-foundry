# Source Card: Defeating Nondeterminism in LLM Inference

## Summary
This Thinking Machines Lab engineering blog (Horace He et al., Sep 2025) argues that the dominant cause of nondeterminism in LLM inference is the lack of "batch invariance": because server load varies the batch size nondeterministically, reduction order in kernels changes, producing different bitwise results even at temperature 0. The authors implement batch-invariant RMSNorm, matmul, and attention kernels and demonstrate that doing so makes 1000 Qwen3-235B completions bitwise identical, at a measured 1.6x-2.1x inference slowdown on Qwen-8B.

## Key evidence
- (ev_001) The root cause is nondeterministic batch-size variation from server load, not concurrency or floating point alone.
- (ev_002) The popular concurrency-plus-floating-point hypothesis is incomplete; atomic adds are usually absent from the forward pass.
- (ev_003) Qwen3-235B at temperature 0 produced 80 unique completions over 1000 runs, first diverging at token 103.
- (ev_004) Batch-invariant kernels made all 1000 completions identical.
- (ev_005) Only three reduction ops need batch invariance: RMSNorm, matmul, and attention.
- (ev_006) Floating-point non-associativity is the underlying mathematical residual the technique works around.
- (ev_007) Deterministic vLLM on Qwen-8B cost 26s (default) vs 42-55s deterministic, a 1.6x-2.1x slowdown.

## Limitations
- Engineering blog, not peer-reviewed; presents the authors' own experiments and implementation.
- Performance numbers come from a single model/hardware configuration (Qwen-8B, 1000 sequences) and may not generalize.
- The technique addresses greedy/temperature-0 determinism; sampling at temperature > 0 remains intentionally nondeterministic.
