# Source Card: Messages - Anthropic Claude API Reference

## Summary
This is the official Anthropic Claude API reference for the Messages endpoint. It documents the available sampling controls: `temperature` (default 1.0, range 0.0-1.0, lower for analytical/multiple-choice tasks), `top_p` (nucleus sampling), and `top_k` (top-K sampling), the latter two flagged "recommended for advanced use cases only." Most relevant for determinism and CI cost-control work, the page explicitly states that even at `temperature` 0.0 results will not be fully deterministic, and it exposes no seed/deterministic-seeding parameter at all.

## Key evidence
- (ev_001) temperature defaults to 1.0, ranges 0.0-1.0; use closer to 0.0 for analytical/multiple-choice tasks, closer to 1.0 for creative tasks.
- (ev_002) Documentation explicitly states results will not be fully deterministic even at temperature 0.0.
- (ev_003) top_p (nucleus sampling) is documented but recommended for advanced use cases only.
- (ev_004) top_k samples only from the top K options to remove long-tail responses; also recommended for advanced use cases only.
- (ev_005) The API exposes only temperature/top_p/top_k for sampling and documents no seed/deterministic-seeding parameter.
