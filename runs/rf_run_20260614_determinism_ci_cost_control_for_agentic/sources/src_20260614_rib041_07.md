# Source Card: FakeListLLM - LangChain core documentation

## Summary
The official LangChain core API reference documents `FakeListLLM`, a fake language model "for testing purposes" that returns a predefined `responses: list[str]` "in order." It extends the `LLM` base class and therefore inherits the standard Runnable interface (sync and async `invoke`/`batch`/`stream`/`generate` variants), letting it drop into chains and agents in place of a real model. An internal counter `i` advances through the scripted responses on each invocation, enabling deterministic, network-free unit tests.

## Key evidence
- (ev_001) FakeListLLM is LangChain's fake LLM intended for testing purposes.
- (ev_002) The `responses` attribute is a `list[str]` returned in order, giving deterministic scripted outputs.
- (ev_003) FakeListLLM extends the `LLM` base class, conforming to the standard language-model interface and substitutable for a real LLM.
- (ev_004) Via BaseLLM inheritance it exposes the full Runnable surface, including async variants (invoke/ainvoke, batch/abatch, stream/astream, generate/agenerate).
- (ev_005) An internal index `i` is incremented after every invocation, advancing through the responses; exposed primarily for testing.
- (ev_006) A `sleep` attribute sets per-response delay in seconds; ignored by FakeListLLM but used by subclasses to simulate latency.
- (ev_007) FakeListLLM has been available since langchain_core v0.1 and is present in the current reference.
