# Source Card: Optimize Your App with Experiments (Quickstart: Datasets & Experiments) — Arize Phoenix

## Summary
This Arize Phoenix get-started guide ("Optimize Your App with Experiments," the target of the datasets-and-experiments quickstart URL) defines an experiment as a structured comparison between versions of an application run over the same stored dataset inputs and evaluation criteria. The documented flow — identify failures from explanations, update the application/prompt, then `client.experiments.run_experiment(dataset, task, evaluators)` over the saved dataset — supports iterative, measured improvement by testing new versions on the exact same inputs that previously failed. It establishes the dataset-as-fixed-input-set + experiment pattern for systematic, reproducible model/prompt evaluation.

## Key evidence
- (ev_001) Phoenix defines an experiment as a structured comparison between application versions using identical inputs and evaluation criteria.
- (ev_002) Running experiments in code against a stored dataset lets teams verify whether changes actually improve quality.
- (ev_003) Experiments rerun the same inputs through different versions and compare results side by side so improvements are measured, not assumed.
- (ev_004) An experiment task is a function that takes each dataset example and produces an output by running the application/model on the input.
- (ev_005) Running the experiment over the saved dataset tests the new version on the exact same inputs that previously failed.
- (ev_006) Experiments are launched via client.experiments.run_experiment(dataset, task, evaluators); Phoenix logs results automatically.
- (ev_007) Datasets/experiments scale to comparing multiple prompt or model variants and tracking quality over time.
