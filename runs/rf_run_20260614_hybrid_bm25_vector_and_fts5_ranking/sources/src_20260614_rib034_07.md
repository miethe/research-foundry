# Source Card: Announcing Rerank-v3.5 — Cohere Release Notes

## Summary
Cohere's changelog announces rerank-v3.5, a reranking model with a 4096-token context length and claimed state-of-the-art performance on multilingual retrieval, reasoning, and BEIR plus several application domains (Finance, E-commerce, Hospitality, etc.). The release ships alongside breaking API changes: `model` becomes required, `max_chunks_per_doc` is replaced by a `max_tokens_per_doc` parameter (default 4096), and list-of-objects `documents` input is removed.

## Key evidence
- (ev_001) rerank-v3.5 has a 4096-token context length, positioned for multilingual retrieval and reasoning.
- (ev_002) Vendor-claimed SOTA on BEIR and across Finance, E-commerce, Hospitality, Project Management, and Email/Messaging retrieval.
- (ev_003) The `model` parameter is now required.
- (ev_004) `max_chunks_per_doc` replaced by `max_tokens_per_doc` (per-doc truncation cap, default 4096).
- (ev_005) Passing a list of objects for the `documents` parameter is no longer supported.

## Limitations
- Performance/"SOTA" claims are vendor-asserted in a release note with no benchmark tables or methodology shown on this page; treat as marketing-grade evidence pending independent benchmark confirmation.
