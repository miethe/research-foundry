# Source Card: Reciprocal rank fusion — Elasticsearch Reference

## Summary
Elastic's official Elasticsearch reference page defines Reciprocal Rank Fusion (RRF) as a method for combining multiple result sets with different relevance indicators into a single ranked result set, and documents the exact scoring formula (summing 1.0 / (k + rank) across queries). It establishes that RRF requires no tuning and that the combined relevance signals need not be related to one another. It also specifies the `rank_constant` (k) parameter: higher values give lower-ranked documents more influence, the value must be >= 1, and it defaults to 60.

## Key evidence
- (ev_001) RRF combines multiple result sets with different relevance indicators into one ranked result set.
- (ev_002) RRF requires no tuning and the relevance indicators need not be related to each other.
- (ev_003) The RRF score for a document is the sum of 1.0 / (k + rank(result(q), d)) over each query q where d appears.
- (ev_004) k is the ranking constant; rank(result(q), d) is d's rank within result(q), starting from 1.
- (ev_005) The rank_constant (k) controls per-query influence, higher values favor lower-ranked docs, must be >= 1, and defaults to 60.

## Limitations
- Vendor product documentation: defaults (e.g., k=60) and parameter semantics describe Elasticsearch's specific implementation; other RRF implementations may differ. The page attributes the underlying method to the Cormack et al. (Waterloo) RRF paper but does not itself argue RRF combines rankings vs. raw scores in explicit comparative terms.
