# Source Card: Is Semantic Chunking Worth the Computational Cost?

## Summary
This arXiv preprint (Qu, Bao, Tu; Vectara and UW-Madison) systematically evaluates semantic chunking against simpler fixed-size chunking across document retrieval, evidence retrieval, and retrieval-based answer generation tasks on BEIR and RAGBench datasets. It finds that fixed-size chunking matches or beats semantic chunking on realistic data (e.g., HotpotQA, MSMARCO), while semantic chunking only wins on artificially stitched high-topic-diversity documents (e.g., Miracl). The headline conclusion is that semantic chunking's added computational cost is not justified by consistent gains, and fixed-size chunking is the more efficient and reliable practical default.

## Key evidence
- (ev_001) Computational cost of semantic chunking is not justified by consistent gains.
- (ev_002) Fixed-size chunking led document-retrieval F1@5 on non-stitched data (HotpotQA 90.59%, MSMARCO 93.58%).
- (ev_003) Breakpoint-based semantic chunking won only on stitched high-diversity data (Miracl 81.89% vs fixed 69.45%).
- (ev_004) Authors caution real-world topic diversity is lower, eroding semantic chunkers' edge.
- (ev_005) Evidence-retrieval differences were minimal (ExpertQA: fixed 47.11% vs clustering 46.87%).
- (ev_006) Answer-generation BERTScore differences were negligible and inconclusive.
- (ev_007) Fixed-size chunking recommended as the more efficient, reliable practical default.
- (ev_008) Three chunkers compared across three tasks on BEIR and RAGBench datasets.

## Limitations
- v1 preprint, not formally peer-reviewed at access time; semantic chunking's measured advantage depends on artificially stitched high-topic-diversity documents and the specific chunkers/embedding model used, so results may not generalize to all RAG pipelines.
