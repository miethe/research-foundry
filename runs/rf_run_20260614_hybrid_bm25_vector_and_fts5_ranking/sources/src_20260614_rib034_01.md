# Source Card: SQLite FTS5 Extension

## Summary
The official SQLite FTS5 documentation defines the extension's ranking semantics. The built-in bm25() auxiliary function scores full-text matches as a real number where numerically lower values are better matches, and per-column relevance weights are supplied as trailing arguments (defaulting to 1.0 for any unweighted column). FTS5 also exposes a hidden "rank" column equal to bm25() with no arguments, whose default mapping (including column weights) can be configured and persisted via the 'rank' configuration option.

## Key evidence
- (ev_001) bm25() returns a real value where numerically smaller means a better match; sort with ORDER BY bm25(ft).
- (ev_002) Column weights are trailing args in left-to-right column order; unspecified columns default to weight 1.0.
- (ev_003) The hidden "rank" column equals bm25() with no trailing arguments by default.
- (ev_004) ORDER BY rank is logically equivalent to ORDER BY bm25() and may be faster, especially with LIMIT.
- (ev_005) Default ranking can be persisted via INSERT INTO ft(ft, rank) VALUES('rank', 'bm25(10.0, 5.0)').
