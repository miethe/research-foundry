# Source Card: sqlite-utils Python API — Full-text search

## Summary
The sqlite-utils Python API documentation (by Simon Willison) is the canonical reference for adding SQLite full-text search to a table. It establishes that `enable_fts()` indexes a caller-supplied list of named columns as separate searchable fields, defaults to FTS5 (with an explicit FTS4 opt-out), supports Porter stemming via `tokenize="porter"`, and that `table.search()` returns rows ordered by relevance with an `include_rank=True` option exposing the BM25 score. Per-column BM25 weighting is not surfaced by this API and must be applied via raw FTS5 rank configuration.

## Key evidence
- (ev_001) enable_fts() indexes a list of named columns as separate searchable FTS fields.
- (ev_002) tokenize='porter' enables Porter stemming so 'running' matches 'run'.
- (ev_003) enable_fts() defaults to FTS5; fts_version='FTS4' opts into FTS4.
- (ev_004) table.search() returns rows ordered by relevance, most relevant first.
- (ev_005) include_rank=True adds a 'rank' column with the BM25 ranking score (FTS5 only).
- (ev_006) search() ordering is only an order_by column (default = relevance); no per-column bm25() weight parameter is exposed.
