# Source Card: Digital Crochet: Toward a Visual Language for Pattern Description

## Summary
This peer-reviewed Onward! 2022 (SPLASH/ACM) paper by Seitz, Rein, Lincke, and Hirschfeld proposes a visual, domain-specific, graph-based language for representing crochet patterns, where graph nodes are "insertion points" (connected loops) rather than stitches one-to-one. It demonstrates the language through a prototype editor that creates patterns in 2D and views them in 3D using force-based layouting; the 3D view is built on the vasturiano/3d-force-graph library (Three.js + WebGL, D3 variant physics). A qualitative user study with six professional designers found the language removed ambiguities present in textual/chart notation and that designers were enthusiastic about live 3D shape preview, but identified force-layout instability as the dominant UX problem.

## Key evidence
- (ev_001) Both 2D and 3D views use force-based layouting as a heuristic; the 3D view is built on the 3D force-directed graph library (Three.js/WebGL, D3-variant physics).
- (ev_002) The 3D force-based layout is explicitly only an approximation; an exact model would need physical stitch/yarn modeling as done for knitting.
- (ev_003) Force-layout-induced movement made it hard for users to orient themselves; a stable layout algorithm is named as a desirable open problem.
- (ev_004) Graph nodes are insertion points (connected loops), not stitches one-to-one (a slip stitch may produce no insertion-point loop).
- (ev_005) Increases/decreases are encoded as multiple incoming vs outgoing insertion edges; rows and rounds are unified into "layers" stored as a node property.
- (ev_006) Designers showed amazement at the live 3D visualization, but rotation/instability of the rendered chart was a main complaint.
- (ev_007) The language removes ambiguities present in standard notations, but the prototype was not released.

## Limitations
- Prototype editor was not publicly released; findings are from a qualitative think-aloud study with only six professional designers recruited via the myboshi pattern company. The 3D layout is acknowledged by the authors as an approximation, not a physical simulation of the crocheted form.
