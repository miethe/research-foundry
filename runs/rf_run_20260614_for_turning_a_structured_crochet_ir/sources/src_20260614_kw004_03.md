# Source Card: CrochetPARADE Manual (Crochet PAttern Renderer, Analyzer, and DEbugger)

## Summary
The CrochetPARADE manual documents an official, client-side crochet pattern renderer/analyzer/debugger by Svetlin Tassev. Placement of stitches is computed via an iterative force-directed/physics relaxation (the DOT command) rather than a closed-form deterministic layout, and the tool exposes per-stitch fidelity and navigation aids (tension coloring, zero-based hover labels, highlight-by-label). It defines a compact textual stitch grammar with turn-based attachment-direction semantics and exports to 3D/vector formats.

## Key evidence
- (ev_001) Placement uses iterative force-directed relaxation; learning_rate default 0.1 fails to converge above ~0.15, with a default of 500 iterations.
- (ev_002) Pressing 's' colors stitches blue (too loose) or red (too stretched by >~15% of baseline height).
- (ev_003) The engine converges only to within ~10% of requested stitch lengths under default settings.
- (ev_004) Even/odd turn counting governs whether the next stitch attaches forward (even) or in reverse (odd).
- (ev_005) sc2inc = two single crochets into one stitch (increase); sc2tog = decrease combining two single crochets.
- (ev_006) Row and stitch numbering is zero-based.
- (ev_007) Runs fully client-side; tens-of-thousands-of-stitch patterns can take minutes or more and may be sluggish on old hardware.

## Limitations
- Manual is marked "work in progress" (last updated 2026-01-07); parameter defaults and behavior may change. Placement is stochastic (random seed) and only ~10%-accurate to requested lengths, so it is not a closed-form deterministic geometry source.
