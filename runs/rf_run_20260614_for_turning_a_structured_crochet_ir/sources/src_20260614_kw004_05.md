# Source Card: modeling-amigurumi (EnbyMonkey) — Connectivity Shapes amigurumi modeler

## Summary
A public GitHub C++ project that models simple amigurumi crochet patterns from a structured text grammar (one row per line, magic-circle start, sc/inc/dec stitches, bracket repetition) by embedding the resulting stitch graph in 3D via the "Connectivity Shapes" force-directed graph layout. It documents a concrete, parseable crochet IR and a working graph-layout pipeline built on Eigen3 and Ceres Solver/SuiteSparse. The README is candid about limits: the layout is poor for small patterns, slow for large ones, and cannot model joined limbs.

## Key evidence
- (ev_001) Input grammar: one row per line, first line must be 'mc'; valid stitches sc/inc/dec; 'fo' closes the final row.
- (ev_002) Grammar supports stitch multipliers and bracket repetition, e.g. '3[2sc,inc]'.
- (ev_003) C++ (~98%), built with CMake >= 3.15, depends on Eigen3 and Ceres Solver (SuiteSparse for Ceres).
- (ev_004) Uses the "Connectivity Shapes" force-directed graph layout to embed the stitch graph in 3D.
- (ev_005) Stated limitation: layout works poorly on small patterns and is quite slow on larger ones.
- (ev_006) Does not support joining patterns (joined amigurumi limbs).

## Limitations
- Author-stated performance/scope caveats: poor layout quality on small patterns, slowness on large patterns, and no support for joined limbs — material feasibility caveats for real-time/mobile use.
- README provides no real author name, publisher, or explicit release date; the "2025" / "Greer & Mould reference implementation" framing in the task source comes from outside the README, which itself contains no academic citation. Recorded as such; not independently verified here.
