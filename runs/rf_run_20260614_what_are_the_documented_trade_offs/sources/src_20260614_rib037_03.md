# Source Card: Git for Data | Dolt Docs

## Summary
DoltHub's official "Git for Data" introduction establishes Dolt as a MySQL-compatible, version-controlled SQL database that applies Git-style semantics (checkout, diff, merge, log, clone/fetch/push, cherry-pick, revert, blame) to tables instead of files. Write operations are surfaced as SQL stored procedures and read operations as system tables, and Dolt is presented as a drop-in MySQL replacement that combines relational querying with full version-control history.

## Key evidence
- (ev_001) Dolt is positioned as "Git for Data," version-controlling tabular data the way Git version-controls files.
- (ev_002) Dolt's versioning unit is the SQL table rather than the file.
- (ev_003) Dolt is a MySQL-compatible drop-in replacement that also adds version control.
- (ev_004) Write operations are SQL stored procedures; read operations are system tables.
- (ev_005) Dolt positions itself as a Git + MySQL hybrid.
- (ev_006) Dolt's CLI mirrors Git's CLI, with tables as the versioning target instead of files.
