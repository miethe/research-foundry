# Source Card: Git - Git Internals: Git Objects (Pro Git book, 2nd ed.)

## Summary
This section of the Pro Git book establishes Git as a content-addressable filesystem built on a key-value object store: inserted content is hashed to a unique SHA-1 key used for retrieval. It documents how git hash-object computes the SHA-1 over a type/size header plus content, defines the three core object types (blobs for file content, trees for directory structure, commits for snapshots with parents and metadata), and describes zlib-compressed on-disk storage under .git/objects/ keyed by the SHA-1.

## Key evidence
- (ev_001) Git is a content-addressable filesystem / key-value store returning a unique key per inserted content.
- (ev_002) git hash-object stores data in .git/objects and returns its unique key; 'test content' -> d670460b4b4aece5915caf5c68d12f560a9fe3e4.
- (ev_003) SHA-1 is computed over a header (type + space + byte size + null byte) concatenated with the original content.
- (ev_004) Tree objects hold one-or-more entries, each a SHA-1 of a blob/subtree plus mode, type, and filename.
- (ev_005) Commit objects record the top-level tree SHA, parent commits, author/committer + timestamp, and the message.
- (ev_006) Objects are stored zlib-compressed under .git/objects/<first 2 SHA chars>/<remaining 38 chars>.
