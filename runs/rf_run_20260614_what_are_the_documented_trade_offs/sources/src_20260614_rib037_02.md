# Source Card: simonw/git-history — Tools for analyzing Git history using SQLite

## Summary
git-history is a CLI tool by Simon Willison that converts the full Git commit history of a file (JSON, CSV, or TSV) into a SQLite database for analysis. With the `--id` option it de-duplicates records by a unique identifier and reconstructs item-level change history across commits, materializing every captured version into an `item`/`item_version` schema. It embodies the pattern where Git is the durable, diffable source of truth and the SQLite database is a derived, disposable index optimized for querying — typically published via Datasette.

## Key evidence
- (ev_001) git-history's stated purpose is analyzing a file's Git history by loading it into SQLite.
- (ev_002) The `file` command converts a versioned file's history into a SQLite database.
- (ev_003) The `--id` option de-duplicates by unique identifier and tracks per-item changes over time.
- (ev_004) Running with `--id` produces six tables including `item_version` for item-level change history.
- (ev_005) Every captured version of every record is materialized (10 versions x 30 items = 300 rows).
- (ev_006) Distributed via PyPI; installable as a tool via uv (`uv tool install git-history`).
- (ev_007) Output databases are intended for querying/publishing via Datasette (deployed demos exist).
