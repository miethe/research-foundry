# Source Card: About Obsidian — Obsidian Help (local-first plain-text Markdown vault)

## Summary
This is Obsidian's first-party "About Obsidian" help page. It establishes Obsidian as a Markdown editor and knowledge base app whose differentiator is a plain-text, local-first data model: notes live primarily on the user's hard disk, the file system replaces the cloud, and the resulting plain files are directly compatible with Git versioning, Dropbox backup, and disk encryption. Notes are connected via bidirectional internal links and a graph view, and the product ships as a minimal foundation extended by optional building blocks.

## Key evidence
- (ev_001) Obsidian's philosophy is built on storing the knowledge base as plain text, framing it as user-owned rather than locked into a host.
- (ev_002) Data always lives primarily on the user's local hard disk; Sync is only an optional multi-device utility (local-first).
- (ev_003) The plain-file / file-system model is directly compatible with Git versioning, Dropbox backup, and disk encryption.
- (ev_004) Notes are connected via bidirectional internal links and a graph view rather than a database schema.
- (ev_005) Obsidian is positioned as both a Markdown editor and a knowledge base app.
- (ev_006) Obsidian ships a minimal foundation (view/edit/search files) plus optional building blocks, not a monolithic product.
