# Source Card: CrochetBench: Can Vision-Language Models Move from Describing to Doing in Crochet Domain?

## Summary
CrochetBench is an arXiv benchmark (Li et al., Nov 2025) that tests whether vision-language models can move beyond describing crochet to producing executable procedures. It builds a 6,085-pattern, 55-category dataset from Yarnspirations and defines four tasks (stitch recognition, instruction selection, instruction generation, and DSL translation) evaluated for executable correctness via the CrochetPARADE DSL validator. The headline finding is that model performance collapses when evaluation shifts from surface similarity to execution: top models compile only 5-8% of DSL outputs and produce uniformly low visual fidelity.

## Key evidence
- (ev_001) CrochetBench uses the CrochetPARADE DSL as an intermediate representation enabling structural validation and execution-based functional evaluation.
- (ev_002) The benchmark comprises 6,085 crochet patterns across 55 categories from Yarnspirations, normalized to JSON with >98% image coverage.
- (ev_003) Four tasks use distinct metrics, including Valid Pattern Rate (compilation success) and DINO image similarity for DSL translation.
- (ev_004) Even the strongest models produce only 5-8% executable DSL programs.
- (ev_005) Best Task A F1 was Claude Sonnet 4 at 60.94%; best Task C BLEU was Gemini 2.5 Flash-Lite at 4.93%.
- (ev_006) Performance drops sharply from surface similarity to executable correctness; DINO visual similarity is uniformly low at 0.10-0.17.
- (ev_007) Instruction length stratifies complexity by skill level (Beginner ~1,674 chars up to Experienced ~7,689 chars).

## Limitations
- Preprint (not formally peer-reviewed); dataset is sourced from a single site (Yarnspirations) and normalized via a GPT-4o-mini conversion pipeline that may introduce noise. ev_005 and ev_007 carry summaries without short verbatim quotes because the exact figures appear in tables/figures rather than as quotable sentences; the numbers were confirmed against the full HTML (v2).
