# Source Card: AmiGo: Computational Design of Amigurumi Crochet Patterns

## Summary
AmiGo (Edelstein et al., ACM SCF '22) generates human-readable crochet patterns for amigurumi stuffed toys from a closed triangle mesh plus a single user-chosen seed point and a stitch width. It builds a "Crochet Graph" whose rows are isolines of geodesic distance from the seed, auto-segments branching shapes at the geodesic saddle points, joins segments with join-as-you-go (no sewing), and emits sc/inc(x)/dec(x) instructions. Success is evaluated qualitatively only -- the crocheted, stuffed result is reported as "similar" to the input, with no quantitative shape-distance metric.

## Key evidence
- (ev_001) Input is a closed triangle mesh plus one user-specified seed point and a stitch width; no automatic seed selection.
- (ev_002) Evaluation is qualitative ("similar to M") with no Hausdorff/Chamfer/volume metric reported.
- (ev_003) Shape is auto-segmented at saddle points of a geodesic distance function; geodesic isolines become crochet rows.
- (ev_004) Segments are connected via join-as-you-go (crocheted onto the previous segment's last row), needing no sewing and leaving seams invisible.
- (ev_005) Stitch vocabulary is sc, inc(x), dec(x) with x derived from row coupling; no explicit per-round increase/decrease bound is stated.
- (ev_006) Table 2 complexity: Teddy hi-res = 60 rows / 6 segments / 3,670 stitches / 2.5 min; Moomoo = 60 rows / 8 segments / 2,491 stitches / 6.9 min; runtimes 0.2-6.9 min on an i7 desktop.

## Limitations
- Method handles only closed surfaces (must be stuffable); segmentation can produce hard-to-crochet thin segments; single-seed setup gives limited design freedom and does not account for model symmetry, so low-resolution patterns may be non-symmetric. Evaluation is qualitative only -- no quantitative shape-distance scoring is provided, which is an open gap for an evaluation protocol to add.
