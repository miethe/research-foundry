# Source Card: When Facts Expire: Benchmarking Temporal Validity in Knowledge Graphs

## Summary
This CIKM '25 short paper (Soulard, Saïs, Raad; ACM Open Access, pp. 6534-6538) introduces what the authors describe as the first benchmark for temporal fact validation in Knowledge Graphs — i.e., judging whether existing temporal facts remain valid, rather than predicting missing ones. The benchmark is derived from the May 2023 Wikidata dump and converts it into a validation task by generating negative samples that corrupt only a fact's temporal context while leaving subject, predicate, and object unchanged, isolating the temporal-reasoning (staleness) signal. Artifacts are archived on Zenodo (record 15680977) with code on GitHub, and the data ships in multiple configurations varying temporal scope and KG structure.

## Key evidence
- (ev_001) Traditional KGs treat facts as timeless, causing outdated or incorrect inferences.
- (ev_002) Introduces the first benchmark for temporal fact validation; validating existing temporal info was underexplored versus predicting missing facts.
- (ev_003) Derived from Wikidata; supports comparisons across timestamps/intervals and KG density/depth.
- (ev_004) Dataset derived from the Wikidata dump of May 2023, transformed into a temporal-validation task.
- (ev_005) Negative samples corrupt only the temporal context to isolate temporal-reasoning evaluation.
- (ev_006) Multiple configurations vary temporal scope: dense (1900-2023) vs. full/sparse (-1000-2023).
- (ev_007) Artifacts archived on Zenodo (15680977); code on GitHub (SoulardThibaut/WhenFactExpire).
