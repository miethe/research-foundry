# Source Card: Improved Evidence Extraction and Metrics for Document Inconsistency Detection with LLMs

## Summary
This January 2026 arXiv paper (2601.02627v2) by Tan et al. addresses the relatively under-studied problem of LLM-based document inconsistency detection by focusing on evidence extraction — pinpointing the specific sentences that cause an inconsistency rather than producing a binary contradiction yes/no. It contributes new comprehensive evidence-extraction metrics (evidence precision, evidence recall, alongside the existing evidence-hit metric), a redact-and-retry (RnR) framework with constrained filtering that the authors report as the best performer across datasets, LLMs, and metrics, and a new semi-synthetic dataset (ContraDocPaired) derived from ContraDoc to evaluate multi-sentence evidence sets.

## Key evidence
- (ev_001) Paper targets LLM-based document inconsistency detection via evidence-extraction capabilities, a relatively limited research area.
- (ev_002) Two core contributions: new evidence-extraction metrics plus a redact-and-retry + constrained-filtering framework improving over other prompting methods.
- (ev_003) Releases a new semi-synthetic dataset for evaluating evidence extraction, backed by strong experimental results.
- (ev_004) Task = classify presence of inconsistency AND identify the set of inconsistent sentences (precision of evidence localization).
- (ev_005) ContraDocPaired extends ContraDoc to evidence-set size two, fixing the original's size-one-only limitation.
- (ev_006) Evaluated on GPT-4o (strong) and LLaMA3.2-90B (weak) at temperature 0.
- (ev_007) RnR+CF reported as best performer across both datasets, both LLMs, and all metrics.
