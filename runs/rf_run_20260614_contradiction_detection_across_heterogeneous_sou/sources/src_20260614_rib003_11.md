# Source Card: Contradiction Detection in RAG Systems: Evaluating LLMs as Context Validators for Improved Information Consistency

## Summary
This arXiv preprint (Gokul, Tenneti, Nakkiran; March 2025) introduces a synthetic data-generation framework for RAG retrieval conflicts and benchmarks LLMs as "context validators" that detect contradictions in retrieved document sets. Across a 1,867-sample dataset and four model variants, it shows that contradiction detection is hard even for state-of-the-art LLMs, that performance varies sharply by contradiction type, and that chain-of-thought prompting helps some architectures while hurting others.

## Key evidence
- (ev_001) Synthetic dataset of 1,867 samples: 37.49% none, 26.30% self, 19.07% pair, 17.14% conditional.
- (ev_002) Human inter-annotator agreement was only 74%, showing the task is hard for people too.
- (ev_003) Best conflict-detection accuracy was Claude-3 Sonnet+CoT (0.710); Llama-70B basic 0.679; worst Llama-8B basic 0.380.
- (ev_004) CoT is architecture-dependent: +31% for Claude-3 Sonnet (+46% Haiku) but -26% for Llama-70B.
- (ev_005) Difficulty hierarchy: pair easiest (Llama-70B 0.893), conditional moderate, self hardest (0.006-0.456).
- (ev_006) Larger is not always better: Llama-8B beat Llama-70B by ~6% on type detection.
- (ev_007) Models showed high precision but low recall: conservative, missing many real contradictions.

## Limitations
- Dataset is synthetic (generated by Claude-3 Sonnet from HotpotQA), so it may not reflect naturally occurring RAG conflicts; only four model variants (Claude-3 Sonnet/Haiku, Llama3.3-70B, Llama3.1-8B) were tested.
