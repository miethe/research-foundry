# Source Card: Can LLMs be Good Graph Judge for Knowledge Graph Construction?

## Summary
GraphJudge (EMNLP 2025) is a three-module knowledge-graph construction framework: entity-centric text denoising plus triple extraction with a closed-source generator (GPT-4o-mini), knowledge-aware supervised fine-tuning of an open-source LLaMA-2-7B, and a graph-judgement step that binary-classifies each generated triple as correct or incorrect. The architecture deliberately splits cost and roles: a strong generator maximizes recall while a cheap fine-tuned 7B judge filters false triples to raise precision, beating GPT-4o/GPT-4o-mini and the PiVe verification baseline (which uses a 70B LLM) on general and domain-specific datasets while generalizing across corpora.

## Key evidence
- (ev_001) Generator is closed-source GPT-4o-mini (ECTD); judge is a fine-tuned open-source LLaMA-2-7B (KASFT).
- (ev_002) Graph judgement is defined as a triple-classification task with a binary correct/incorrect label per triple.
- (ev_003) REBEL-Sub G-BERTScore-F1 0.5796 vs GPT-4o-mini 0.4289; SCIERC 0.7283 vs 0.6882.
- (ev_004) Removing the graph-judgement module drops REBEL-Sub G-BS-F1 from 0.5796 to 0.4203 (the precision driver).
- (ev_005) Cross-dataset: trained on GenWiki, tested on REBEL-Sub, GraphJudge G-BS-F1 0.5814 vs GPT-4o 0.4163.
- (ev_006) Cost-effective: SOTA with only a fine-tuned 7B LLM vs the 70B LLM used by PiVe.
- (ev_007) The judge eliminated meaningless triples such as <We, suggest, goal> in a SCIERC case study.
- (ev_008) Fine-tuned judge reaches over 90% judgement accuracy on REBEL-Sub and GenWiki.

## Limitations
- The generator is GPT-4o-mini (the upstream brief's "gpt-3.5-turbo" is incorrect and was not used), and the headline metric is G-BERTScore-F1 (a soft semantic match), not exact-match F1 — so reported deltas are larger and not directly comparable to strict-match scores. Baseline values cited here are GPT-4o-mini / GPT-4o, not gpt-3.5-turbo. Evaluation used 2000-point samples per dataset.
