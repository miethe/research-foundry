# Source Card: From Local to Global: A Graph RAG Approach to Query-Focused Summarization

## Summary
This Microsoft Research paper introduces GraphRAG, a method that uses an LLM to build a graph index from source documents — extracting entity nodes, relationship edges, and optional claim/covariate records — then applies Leiden community detection to partition the graph into a hierarchy of communities. Bottom-up community summaries enable global sensemaking (query-focused summarization) over corpora in the ~1M-token range, where it substantially outperforms naive/vector RAG on LLM-judged answer comprehensiveness and diversity. A gleaning loop trades extra LLM calls for higher entity-extraction recall, and a reference implementation is open-sourced.

## Key evidence
- (ev_001) An LLM extracts entity nodes, relationship edges, and optional claim/covariate records from each source text chunk to build a graph index.
- (ev_002) Claims/covariates are extracted as structured factual statements about entities, such as dates, events, and interactions with other entities.
- (ev_003) Leiden community detection partitions the entity graph hierarchically into a nested hierarchy of closely-related-entity communities.
- (ev_004) Community summaries are generated bottom-up, with higher-level summaries recursively incorporating lower-level community summaries.
- (ev_005) The method was evaluated on global sensemaking over corpora in the ~1 million token range.
- (ev_006) GraphRAG beats naive (vector) RAG on LLM-judged comprehensiveness (72-83%) and diversity (75-82%) win rates.
- (ev_007) A gleaning loop re-prompts the LLM to recover missed entities, trading extra LLM calls (cost) for higher recall.
- (ev_008) GraphRAG is released as an open-source reference implementation.

## Limitations
- The claim/covariate extraction (ev_002) carries the structured description/type/temporal-validity detail per the paper's element-instance design, but the verbatim quote captured here covers the factual-statement definition rather than the full schema. Win-rate figures (ev_006) are LLM-as-judge head-to-head preferences, not ground-truth accuracy. Results are from the authors' own evaluation on two ~1M-token datasets (Podcast transcripts, News articles); generalization beyond these conditions is not established.
