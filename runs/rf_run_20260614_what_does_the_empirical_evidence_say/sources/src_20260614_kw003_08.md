# Source Card: Developing CROML (Crochet Obvious Minimal Language)

## Summary
A practitioner blog post documenting the design of CROML, a compact, indentation-agnostic domain-specific language for crochet patterns that an LLM emits and a local regex-based parser deterministically expands into JSON. The author reports that asking an LLM for raw JSON is error-prone (invalid output, missing characters), token-expensive, and slow for patterns with many rows/stitches, and that emitting the compact DSL instead cuts token usage and generation time while being more parse-robust. It provides independent, practitioner-level corroboration for decoupling the human/LLM authoring surface from the validated machine IR.

## Key evidence
- (ev_001) LLM-generated JSON for crochet patterns is often invalid and is token-expensive/slow due to many rows and stitches.
- (ev_002) Author built a custom minimal representation (CROML) specifically to enable easy local parsing instead of direct LLM JSON.
- (ev_003) CROML is indentation-agnostic, parsed via simple regex, and converted to JSON locally — faster than waiting on LLM JSON.
- (ev_004) Emitting CROML instead of JSON significantly reduces token usage and speeds up generation time.
- (ev_005) The DSL is more parse-robust than JSON: a single missing character is not fatal as it is with JSON.
- (ev_006) Existing minimal formats (TOML/YAML) were rejected as too indentation-heavy/unreadable, motivating a purpose-built authoring surface.

## Limitations
- Secondary, single-author, first-person anecdotal source with no quantified token/latency benchmarks; gains are claimed, not measured.
- Crochet-domain-specific; the general principle (compact DSL plus deterministic expansion beats freeform JSON IR) is inferred corroboration, not a controlled result.
