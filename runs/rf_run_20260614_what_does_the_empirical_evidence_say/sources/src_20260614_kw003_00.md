# Source Card: CrochetBench: Can Vision-Language Models Move from Describing to Doing in the Crochet Domain?

## Summary
CrochetBench is an arXiv benchmark (Li, Huang, Hua, Chawla; cs.AI; v1 Nov 2025, v2 Feb 2026) that tests whether vision-language models can move beyond describing crochet imagery to generating executable crochet procedures. It is built on 6,085 real Yarnspirations patterns across 55 categories and uses the CrochetPARADE DSL as an intermediate representation so outputs are validated by compilation/execution rather than text similarity. The central result is that model performance collapses as evaluation moves from surface-level similarity to executable correctness, with larger models actually hallucinating more invalid stitches.

## Key evidence
- (ev_001) CrochetBench reframes evaluation from describing visual content to generating executable crochet procedures (recognize, select, generate compilable output).
- (ev_002) Dataset of 6,085 Yarnspirations patterns, 55 categories, >98% image coverage, beginner-to-expert difficulty.
- (ev_003) Correctness measured by functional executability via the CrochetPARADE validator (syntactic, structural, and execution checks), not string match.
- (ev_004) Task D NL-to-DSL translation has step-level and project-level variants; project-level supplies full NL plus the product image alongside the DSL target.
- (ev_005) Performance drops sharply from textual similarity to executable correctness, exposing weak long-range symbolic reasoning and 3D-aware procedural synthesis.
- (ev_006) Larger models hallucinate more undefined stitches than smaller ones (Qwen2-VL-72B 72.0% vs 7B 13.6%; Gemma-3-27B 42.6% vs 4B 27.4%), so scale does not fix structural validity.

## Limitations
- Preprint (not formally peer-reviewed); dataset is drawn from a single source (Yarnspirations); executable-correctness scoring is only as complete as the CrochetPARADE compiler's stitch grammar coverage.
