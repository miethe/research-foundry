# Source Card: NIST Issues Updated Security Requirements and Assessment Procedures for Protecting Controlled Unclassified Information (CUI)

## Summary
This NIST CSRC announcement marks the May 14, 2024 release of SP 800-171 Revision 3 (and its assessment companion SP 800-171A Revision 3), the framework governing how nonfederal systems and organizations protect Controlled Unclassified Information (CUI). Rev 3 restructures its requirements to align directly with the SP 800-53r5 control catalog, introduces organization-defined parameters (ODPs) for tailoring, adds new tailoring criteria, and supplies outcome-oriented implementation guidance. For sensitivity-aware AI routing, CUI functions as a binding handling category whose presence mandates protective controls — a useful real-world anchor for a "client_approved"-style tier where writeback to uncontrolled targets should be deny-by-default.

## Key evidence
- (ev_001) Final SP 800-171 Rev 3 (and 800-171A Rev 3) released May 14, 2024 for protecting CUI in nonfederal systems and organizations.
- (ev_002) Rev 3 restructured requirements to align directly with SP 800-53r5 controls.
- (ev_003) Rev 3 introduced organization-defined parameters (ODPs) for tailoring handling thresholds.
- (ev_004) Rev 3 added new tailoring criteria and recategorized controls to reduce redundancy and improve clarity.
- (ev_005) Rev 3 provides outcome-oriented guidance to reduce ambiguity and support implementation.
- (ev_006) The 800-171A Rev 3 assessment companion incorporates ODPs to facilitate traceability and usability.

## Limitations
- This is a NIST news announcement summarizing the change set; it does not enumerate the full control count, so the widely cited 110->97 control reduction must be sourced from the SP 800-171r3 document itself or its change analysis, not this page.
