# Source Card: Classification, redaction, and de-identification | Sensitive Data Protection | Google Cloud

## Summary
This first-party Google Cloud documentation describes how Sensitive Data Protection (DLP) classifies and transforms sensitive data. Classification returns structured findings (InfoType, Likelihood, Offset) using built-in infoType detectors spanning US and international identifiers, and results can be routed to BigQuery or Pub/Sub before transformation. De-identification techniques include automatic redaction with placeholders, plus date shifting, generalization/bucketing, and pseudonymization.

## Key evidence
- (ev_001) Classification output is a list of findings organized by InfoType, Likelihood, and Offset.
- (ev_002) Findings carry a Likelihood/confidence rating with values such as VERY_LIKELY, LIKELY, and VERY_UNLIKELY.
- (ev_003) Built-in infoType detectors include US_HEALTHCARE_NPI, EMAIL_ADDRESS, US_DRIVERS_LICENSE_NUMBER, CANADA_BC_PHN, UK_TAXPAYER_REFERENCE, and CANADA_PASSPORT.
- (ev_004) Automatic redaction replaces sensitive matches with a placeholder such as "***".
- (ev_005) Scan results can be saved to a new BigQuery table or published to a Pub/Sub topic for downstream action.
- (ev_006) De-identification transformations include date shifting, generalization/bucketing, and pseudonymization.
