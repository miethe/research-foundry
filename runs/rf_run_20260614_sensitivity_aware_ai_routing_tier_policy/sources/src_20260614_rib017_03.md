# Source Card: InfoType detector reference — Sensitive Data Protection (Google Cloud DLP)

## Summary
Google Cloud's Sensitive Data Protection (DLP) defines a large catalog of built-in infoType detectors — names, emails, government IDs, financial, health, credentials/secrets, demographic, telecommunications — organized into named categories (Finance, Health, Communications, PII, SPII, Demographic, Credential, Government ID, Document, Contextual information). Every infoType carries a default sensitivity score on a three-level ordinal scale (HIGH / MODERATE / LOW) that is editable per infoType and feeds automated data profiling and risk assessment. The companion sensitivity/data-risk page defines HIGH as the presence of highly sensitive data (e.g., credit-card numbers, national identifiers), MODERATE as not-highly-sensitive PII (e.g., emails, phone numbers), and LOW as no detected sensitive data. This is a working model for content-derived (not user-declared) tier assignment that maps cleanly onto a three-tier router.

## Key evidence
- (ev_001) InfoTypes are the unit of detection — DLP uses infoTypes to define what it scans for.
- (ev_002) Detectors are grouped into named categories: Finance, Health, Communications, PII, SPII, Demographic, Credential, Government ID, Document, Contextual information.
- (ev_003) Each infoType has a default sensitivity score shown in a per-infoType table.
- (ev_004) Default sensitivity scores are customizable per infoType (editable default, not fixed).
- (ev_005) HIGH = highly sensitive data such as credit-card numbers and certain national identifiers.
- (ev_006) MODERATE = sensitive-but-not-highly-sensitive PII such as emails and phone numbers.
- (ev_007) LOW = no sensitive information detected and no freeform/unstructured text.
- (ev_008) Aggregate sensitivity combines each infoType's default sensitivity (plus overrides) with the likelihood that highly sensitive infoTypes are present.

## Limitations
- The infoTypes reference page itself enumerates detectors and sensitivity values but does not define HIGH/MODERATE/LOW; those definitions are drawn from the companion "Sensitivity and data risk levels" documentation page (ev_005–ev_008), which is cited in the locators.
- The page references that new infoType detectors and groups are added periodically; the exact total detector count is not stated as a single verbatim number on the page, so the "400+/500+" figure is not quoted here.
- Sensitivity scores are an editable default; an organization's effective tiering may differ from the documented defaults after user overrides.
