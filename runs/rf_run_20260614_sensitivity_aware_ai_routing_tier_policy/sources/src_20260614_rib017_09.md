# Source Card: Data controls in the OpenAI platform

## Summary
OpenAI's first-party developer documentation establishes the platform's default data-handling posture: API data is not used to train models (since March 1, 2023) unless the customer opts in, and abuse-monitoring logs are retained for up to 30 days by default. It documents two gated controls — Modified Abuse Monitoring and Zero Data Retention (ZDR) — that exclude customer content from abuse-monitoring logs, with ZDR additionally forcing the `store` parameter to false for /v1/responses and /v1/chat/completions. These controls are not self-serve: they require prior OpenAI approval, acceptance of additional requirements, and a sales-team eligibility inquiry, after which approved orgs configure them under Settings -> Organization -> Data controls.

## Key evidence
- (ev_001) Default since March 1, 2023: API data is not used to train OpenAI models unless the customer explicitly opts in.
- (ev_002) Abuse-monitoring logs are generated for all API feature usage and retained up to 30 days by default unless longer retention is legally required.
- (ev_003) ZDR forces the `store` parameter to false for /v1/responses and /v1/chat/completions regardless of the request value.
- (ev_004) ZDR excludes customer content from abuse-monitoring logs, mirroring Modified Abuse Monitoring.
- (ev_005) The abuse-monitoring retention controls require prior OpenAI approval and acceptance of additional requirements (not self-serve).
- (ev_006) Customers contact the OpenAI sales team to learn about offerings and inquire about eligibility.
- (ev_007) Approved orgs configure retention via a Data Retention tab under Settings -> Organization -> Data controls.
