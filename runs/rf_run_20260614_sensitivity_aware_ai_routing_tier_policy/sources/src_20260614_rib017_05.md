# Source Card: Considerations for deploying Microsoft Purview Data Security Posture Management (DSPM) for AI

## Summary
This Microsoft Learn product documentation describes the default, one-click policies that Microsoft Purview DSPM for AI creates to govern how users and AI models interact with sensitive data. It establishes that Microsoft enforces AI-routing-style controls as runtime DLP gates keyed on a combination of per-user Adaptive Protection risk tiers (elevated/moderate/minor) and detected sensitive-information types, including blocking Microsoft 365 Copilot from processing items that carry specific sensitivity labels. Network-level detection of AI interactions is delegated to third-party SASE/SSE partners and must be manually integrated.

## Key evidence
- (ev_001) Default DLP policy 'Block sensitive info from AI sites' gives elevated-risk users a block-with-override on pasting/uploading sensitive data to AI apps across Edge, Chrome, and Firefox (all users, test mode).
- (ev_002) 'Block elevated risk users from submitting prompts' policy blocks elevated, moderate, and minor risk users in Edge and forces protected browsers by treating others as unmanaged apps.
- (ev_003) 'Protect sensitive data from Copilot processing' blocks Microsoft 365 Copilot and agents from processing items carrying selected sensitivity labels — binding model eligibility to a data label.
- (ev_004) 'Detect sensitive info shared with AI via network' requires manually adding SASE/SSE integrations; AI-interaction detection depends on the network partner implementation.
- (ev_005) Adaptive Protection is auto-enabled for default policies, using default risk levels for all users/groups to dynamically enforce protection actions.
- (ev_006) Endpoint DLP enforcement is risk-tier-aware: an elevated-risk user is blocked-with-override when pasting credit card numbers into ChatGPT, showing the per-user-risk-tier + SIT decision unit.

## Limitations
- This page documents the classic version of DSPM for AI, which Microsoft states is replaced by a new version with broader reach and different management; enforcement details may differ in the successor product. Page last updated 2026-05-01.
