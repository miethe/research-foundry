# Source Card: LLM06:2025 Excessive Agency

## Summary
This is the OWASP Gen AI Security Project's primary Top 10 for LLM Applications (2025) entry for Excessive Agency, defining the vulnerability as damaging actions taken in response to unexpected, ambiguous, or manipulated LLM outputs. It attributes the risk to three root causes — excessive functionality, excessive permissions, and excessive autonomy — and prescribes a set of mitigations centered on least-privilege design, avoiding open-ended capabilities, downstream authorization, and human-in-the-loop approval for high-impact actions.

## Key evidence
- (ev_001) Excessive Agency enables damaging actions from unexpected, ambiguous, or manipulated LLM outputs regardless of the malfunction cause.
- (ev_002) Excessive functionality: an agent has access to extension functions not needed for intended operation.
- (ev_003) Excessive permissions and excessive autonomy: unneeded downstream access rights, and failure to independently verify/approve high-impact actions.
- (ev_004) Mitigation: limit extensions callable and permissions granted to the minimum necessary.
- (ev_005) Mitigation: avoid open-ended extensions (e.g., shell command, fetch URL) in favor of granular functionality.
- (ev_006) Mitigation: require human-in-the-loop approval of high-impact actions; enforce authorization in downstream systems, not the LLM.
