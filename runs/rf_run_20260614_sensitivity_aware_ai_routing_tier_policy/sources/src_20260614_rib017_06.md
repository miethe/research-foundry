# Source Card: LLM02:2025 Sensitive Information Disclosure

## Summary
This is the OWASP Gen AI Security Project entry for LLM02:2025 (Sensitive Information Disclosure) within the OWASP Top 10 for LLM Applications. It defines the risk of LLMs leaking sensitive data, proprietary algorithms, and confidential details through their output, enumerates the categories of sensitive information at stake, and prescribes layered prevention and mitigation strategies spanning sanitization, access controls, privacy techniques, and advanced cryptographic methods. It directly supports sensitivity-aware routing by recommending least-privilege data access and securely managed runtime data orchestration that gates which external sources/sinks a task may reach.

## Key evidence
- (ev_001) OWASP scopes sensitive information disclosure to PII, financial details, health records, confidential business data, security credentials, legal documents, and proprietary models/training methods/source code.
- (ev_002) Restrict access to sensitive data on a least-privilege basis, granting only data necessary for the specific user or process.
- (ev_003) Limit model access to external data sources and securely manage runtime data orchestration to avoid unintended data leakage (basis for gating reachable sinks).
- (ev_004) Apply data sanitization (scrubbing/masking) to prevent user data from entering the training model, plus strict input validation.
- (ev_005) Use federated learning over decentralized data and differential privacy (noise injection) to limit centralized exposure and resist reverse-engineering of individual data points.
- (ev_006) Use homomorphic encryption for privacy-preserving analysis and tokenization with pattern-matching redaction before processing.

## Limitations
- The page carries no named individual author and no precise publication month (version is the "2025" list designation).
- Guidance is prescriptive and qualitative; it provides no benchmarks, measured efficacy, or quantitative thresholds.
