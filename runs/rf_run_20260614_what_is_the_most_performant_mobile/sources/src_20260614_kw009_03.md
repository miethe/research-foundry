# Source Card: Arm GPU Best Practices Developer Guide (Revision 3.4)

## Summary
This is Arm's vendor-authoritative best-practices guide (Revision 3.4, dated 2025-02) for optimizing rendering on Mali and Immortalis mobile GPUs. It codifies overdraw-reduction strategy around the GPU's hidden-surface-removal hardware: front-to-back opaque rendering plus early ZS testing, Forward Pixel Kill (present since Mali-T620), and the newer Fragment Prepass (Mali-G725 / Immortalis-G925), while flagging blending — especially on floating-point framebuffers — as the operation that disables these optimizations.

## Key evidence
- (ev_001) FPK ships on all Arm GPUs since Mali-T620 and removes occluded fragments that early ZS misses.
- (ev_002) Render opaque objects front-to-back and minimize blended transparent layers to cut overdraw.
- (ev_003) Do not rely on FPK alone; early-ZS testing is always more energy-efficient and works on older GPUs.
- (ev_004) Fragment Prepass (Mali-G725 / Immortalis-G925) removes overdraw regardless of draw order, letting apps drop front-to-back sorting and reduce draw-call CPU cost.
- (ev_005) Split large UI elements into opaque and transparent portions so early ZS / FPK can kill underlying overdraw.
- (ev_006) Do not use blending on floating-point framebuffers (blending disables overdraw-removal optimizations).
- (ev_007) Monitor per-pixel blended-layer counts; high layer counts consume cycles even with simple shaders.
- (ev_008) Batch geometry and avoid many small draw calls to reduce per-draw CPU and binning overhead (no verbatim quote confirmable from binary PDF body).
