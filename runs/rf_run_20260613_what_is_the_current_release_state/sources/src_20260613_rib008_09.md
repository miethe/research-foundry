# Source Card: Monitoring (OpenTelemetry for Claude Code)

## Summary
This official Anthropic documentation page describes how to enable and configure OpenTelemetry (OTel) for Claude Code to track usage, costs, and tool activity. It covers the enabling flag, metrics/logs/traces exporters and their allowed values, default export intervals, OTLP endpoint and auth configuration, and a beta distributed-tracing feature with W3C trace-context propagation through subprocesses and Agent SDK / non-interactive sessions.

## Key evidence
- (ev_001) Telemetry is enabled with CLAUDE_CODE_ENABLE_TELEMETRY=1; exporters OTEL_METRICS_EXPORTER (otlp/prometheus/console/none) and OTEL_LOGS_EXPORTER (otlp/console/none) are optional.
- (ev_002) Default export intervals are 60s for metrics and 5s for logs; shorten only for debugging.
- (ev_003) Distributed tracing is beta; requires CLAUDE_CODE_ENABLE_TELEMETRY=1 + CLAUDE_CODE_ENHANCED_TELEMETRY_BETA=1 + OTEL_TRACES_EXPORTER, linking prompts to API requests and tool executions.
- (ev_004) Bash/PowerShell subprocesses inherit a TRACEPARENT env var (W3C trace context) for end-to-end tracing.
- (ev_005) Agent SDK and -p sessions read TRACEPARENT and TRACESTATE so spans nest under the caller's trace; interactive sessions ignore inbound TRACEPARENT.
- (ev_006) OTLP endpoint configured via OTEL_EXPORTER_OTLP_PROTOCOL, OTEL_EXPORTER_OTLP_ENDPOINT, and OTEL_EXPORTER_OTLP_HEADERS.
- (ev_007) Metrics export via the standard metrics protocol, events via logs/events protocol, traces via the traces protocol; metrics include cost and token usage.

## Limitations
- Distributed traces (and detailed beta tracing) are explicitly a beta feature; flags, gating (e.g., org allowlisting for some detailed spans), and behavior may change. Some env vars are documented as "also accepted" aliases.
