# Source Card: Intercept and control agent behavior with hooks

## Summary
This is the first-party Anthropic Agent SDK documentation describing hooks: callback functions that run in response to agent events (tool calls, session lifecycle, stops, subagent activity). It establishes which hook events are available in the Python vs TypeScript SDKs, how PreToolUse permission decisions and their precedence work, how tool-use IDs and subagent identity fields are exposed, and the patterns for blocking operations, auditing, transforming inputs/outputs, and forwarding notifications.

## Key evidence
- (ev_001) Python SDK supports PreToolUse, PostToolUse, PostToolUseFailure, UserPromptSubmit, Stop, SubagentStart, SubagentStop, PreCompact, PermissionRequest, and Notification; other events (PostToolBatch, MessageDisplay, SessionStart, SessionEnd, Setup, etc.) are TypeScript-only.
- (ev_002) SessionStart and SessionEnd are not Python SDK callback hooks (HookEvent omits them); in Python they work only as shell-command hooks loaded via setting_sources.
- (ev_003) Tool-use ID correlates PreToolUse and PostToolUse for the same call; agent_id/agent_type appear inside subagents (Python: only on PreToolUse/PostToolUse/PostToolUseFailure).
- (ev_004) PreToolUse returns permissionDecision allow/deny/ask/defer with precedence deny > defer > ask > allow; any deny blocks the tool.
- (ev_005) Hooks can block dangerous operations, audit-log, transform inputs (updatedInput) and outputs (updatedToolOutput), and forward Notifications to Slack/PagerDuty.
- (ev_006) Hooks may not fire when the agent hits the max_turns limit because the session ends first.
- (ev_007) Async hooks (async_=True) run side-effect logging without blocking the agent but cannot block, modify, or inject context.

## Limitations
- Documentation reflects SDK behavior as of June 2026; Python/TypeScript hook parity and the available-events table may change in future SDK releases. Page is undated beyond the published month inferred from the source metadata.
