# Source Card: Agent Skills in the SDK

## Summary
This official Claude Agent SDK documentation page specifies how Agent Skills behave when used through the SDK (as opposed to the Claude Code CLI). It establishes that Skills are purely filesystem artifacts (SKILL.md files) with no programmatic registration API, that discovery depends on `setting_sources`/`settingSources` including `'user'` or `'project'`, and that the `skills` option is a context filter rather than a sandbox. It also documents progressive disclosure of skill content, automatic addition of the Skill tool to `allowedTools`, and the SDK-specific handling of the `allowed-tools` frontmatter field.

## Key evidence
- (ev_001) The SDK has no programmatic API for registering Skills; they must be filesystem SKILL.md artifacts (unlike subagents).
- (ev_002) The `skills` option is a context filter ('all' / name list / []), not a sandbox — unlisted skills stay on disk and remain reachable via Read and Bash.
- (ev_003) Skills are discovered only via filesystem setting sources; explicit settingSources must include 'user' or 'project' to keep discovery.
- (ev_004) setting_sources=[] disables skill discovery entirely by omitting the user and project sources.
- (ev_005) Skill metadata is discovered at startup; full SKILL.md content is loaded only when the skill is triggered (progressive disclosure).
- (ev_006) Setting the `skills` option auto-adds the Skill tool to allowedTools; an explicit tools list must include 'Skill'.
- (ev_007) The `allowed-tools` SKILL.md frontmatter field is CLI-only and ignored via the SDK; control tool access through the main allowedTools option.
