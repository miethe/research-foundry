# Source Card: Agent SDK reference — Python

## Summary
Anthropic's first-party Python reference for the Claude Agent SDK. It defines the two core entry points — `query()` (new session per call, returns an `AsyncIterator[Message]`) and the stateful, multi-turn `ClaudeSDKClient` — plus the `@tool()` decorator and `create_sdk_mcp_server()` for in-process MCP. It documents the `ClaudeAgentOptions` and `AgentDefinition` configuration surfaces (including camelCase wire-format field names, model "inherit" support, skills auto-adding the Skill tool, and setting_sources defaults) and flags stability/deprecation details (`max_thinking_tokens` deprecated, `Transport` as a low-level internal API, and the retired `context-1m-2025-08-07` beta).

## Key evidence
- (ev_001) query() creates a fresh session per call and returns an async iterator, with no cross-call memory unless continue_conversation/resume is set.
- (ev_002) ClaudeSDKClient maintains a conversation session across multiple exchanges (stateful, multi-turn).
- (ev_003) Setting the skills option auto-adds the Skill tool to allowed_tools; if tools is also passed, "Skill" must be included.
- (ev_004) Omitting/None setting_sources loads user/project/local settings like the CLI; managed policy loads in all cases.
- (ev_005) AgentDefinition uses camelCase field names to match the TS wire format; snake_case keywords raise TypeError.
- (ev_006) AgentDefinition.model accepts "sonnet"/"opus"/"haiku"/"inherit" or a full model ID, defaulting to the main model.
- (ev_007) max_thinking_tokens is deprecated in favor of the thinking config.
- (ev_008) The context-1m-2025-08-07 beta is retired as of April 30, 2026; newer 4.6/4.7/4.8 models provide 1M context without a beta header.
- (ev_009) Transport is a low-level internal API subject to change.

## Limitations
- Reference doc carries no explicit overall stability designation for the SDK as a whole; only specific surfaces (Transport, max_thinking_tokens, the retired beta) are flagged. Captured as of the 2026-06 docs snapshot; field names and stability flags can change between releases.
