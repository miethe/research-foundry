# Source Card: claude-agent-sdk-python CHANGELOG.md

## Summary
The official changelog for Anthropic's `claude-agent-sdk-python` package documents a still pre-1.0 (0.x) release line where minor versions can carry breaking changes. The pivotal v0.1.0 rebranded the Claude Code SDK to the Claude Agent SDK (renaming `ClaudeCodeOptions` to `ClaudeAgentOptions`, dropping the default Claude Code system prompt, and disabling automatic filesystem settings loading unless `setting_sources` is set). Later 0.1.x/0.2.x releases layer in features such as the top-level `skills` option (v0.1.62), expanded inline subagent definitions (v0.1.51), subagent transcript helpers plus OTel tracing (v0.1.60), and dependency hardening like the `mcp<2.0.0` pin (v0.2.96), with the bundled Claude Code CLI tracking the ~2.1.17x line.

## Key evidence
- (ev_001) v0.1.0 was a breaking rebrand of the Claude Code SDK to the Claude Agent SDK, renaming ClaudeCodeOptions to ClaudeAgentOptions.
- (ev_002) As of v0.1.0 there is no default system prompt and no automatic filesystem settings load unless setting_sources is set explicitly.
- (ev_003) v0.1.62 added a top-level skills option (accepts 'all', a named list, or []).
- (ev_004) Programmatic subagents are defined inline via the agents option; v0.1.51 added AgentDefinition fields disallowedTools, maxTurns, and initialPrompt.
- (ev_005) v0.1.60 added list_subagents()/get_subagent_messages() and OpenTelemetry distributed tracing via the [otel] extra.
- (ev_006) v0.2.96 pinned mcp<2.0.0 and bundled Claude CLI 2.1.172.
- (ev_007) The latest entry is v0.2.101, bundling Claude CLI 2.1.177 — still a 0.x line.

## Limitations
- This is a point-in-time read of `main`; newer releases may supersede these entries. Because the SDK is pre-1.0, minor version bumps may introduce breaking changes, so feature/version mappings should be confirmed against the specific installed version.
