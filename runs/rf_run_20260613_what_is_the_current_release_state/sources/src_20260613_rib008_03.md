# Source Card: Subagents in the SDK (Claude Docs)

## Summary
This first-party Anthropic documentation page specifies how to define and invoke subagents in the Claude Agent SDK via the `agents` parameter and the `Agent` tool. It establishes the parent-to-child context boundary (fresh context, prompt-string-only channel, inherited tool definitions and CLAUDE.md), isolated/parallel execution with only the final message returning to the parent, and version-gated capabilities including nested subagents (v2.1.172), the Task-to-Agent rename (v2.1.63), resumable subagents, and the Workflow tool for large-scale orchestration.

## Key evidence
- (ev_001) Subagents are defined programmatically via the `agents` parameter and invoked through the `Agent` tool; include `Agent` in `allowedTools` to auto-approve invocations.
- (ev_002) Subagents start a fresh context whose only parent channel is the Agent tool prompt string; they inherit tool definitions and CLAUDE.md but not the parent's conversation, tool results, or system prompt.
- (ev_003) Subagents run in isolated context, can run in parallel, and return only their final message to the parent (verbatim as the Agent tool result).
- (ev_004) As of Claude Code v2.1.172, subagents can spawn nested subagents; a background subagent five levels below the main agent cannot spawn further, while foreground subagents can spawn at any depth.
- (ev_005) The Agent tool was renamed from `Task` to `Agent` in v2.1.63; tool_use emits `Agent` but `system:init` and `permission_denials` still use `Task`, so code must check both.
- (ev_006) Subagents are resumable (capture session_id + agentId, pass resume=sessionId); for dozens-to-hundreds of agents the docs point to the Workflow tool (TypeScript Agent SDK v0.3.149+).

## Limitations
- Page lists no named authors; published date inferred as 2026-06 from source metadata. Version-gated claims (v2.1.172, v2.1.63, SDK v0.3.149) are accurate as of the documented release but may change in later versions.
