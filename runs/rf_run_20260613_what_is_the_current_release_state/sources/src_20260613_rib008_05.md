# Source Card: Agent SDK overview

## Summary
Anthropic's official Claude Docs overview page for the Claude Agent SDK presents the SDK as a Python/TypeScript library that exposes the same agent loop, built-in tools, and context management that power Claude Code. It documents authentication (ANTHROPIC_API_KEY plus Bedrock, Claude Platform on AWS, Vertex AI, and Azure/Foundry), restricts third-party use of claude.ai login/rate limits without pre-approval, flags a June 15 2026 change introducing a separate monthly Agent SDK credit for subscription plans, and contrasts the Agent SDK with both the Client SDK (you implement the tool loop) and Managed Agents (Anthropic-hosted REST API with a managed sandbox).

## Key evidence
- (ev_001) Agent SDK positioned as a library giving the same tools, agent loop, and context management as Claude Code, in Python and TypeScript.
- (ev_002) Auth via ANTHROPIC_API_KEY plus third-party providers: Bedrock, Claude Platform on AWS, Vertex AI, and Microsoft Azure (Foundry), each with a CLAUDE_CODE_USE_* env var.
- (ev_003) claude.ai login / rate limits not permitted for third-party Agent SDK products unless previously approved; API key auth required.
- (ev_004) From June 15, 2026, Agent SDK and `claude -p` usage on subscription plans draws from a separate monthly Agent SDK credit.
- (ev_005) Client SDK requires you to implement the tool loop; Agent SDK runs it autonomously.
- (ev_006) Agent SDK runs in your process (session state as JSONL on disk) vs Managed Agents hosted REST API with managed sandbox; common path is prototype on SDK then move to Managed Agents.

## Limitations
- Living documentation with no explicit version/published date (date inferred as 2026-06); env-var labeling differs slightly from the source brief (Microsoft Azure uses CLAUDE_CODE_USE_FOUNDRY=1; a distinct "Claude Platform on AWS" entry uses CLAUDE_CODE_USE_ANTHROPIC_AWS=1).
