# Source Card: Track cost and usage

## Summary
This official Anthropic Claude Agent SDK guide explains how to track token usage and estimate costs across query() calls, steps, and sessions. It establishes that the SDK's cost fields (total_cost_usd / costUSD) are client-side estimates from a bundled price table — not authoritative billing — and documents the message-level usage model, parallel-call deduplication, per-model breakdowns, cache-token accounting, and the 1-hour cache TTL option.

## Key evidence
- (ev_001) total_cost_usd/costUSD are client-side estimates from a build-time price table, not authoritative billing; use the Usage and Cost API or Console for real billing.
- (ev_002) Python ResultMessage exposes total_cost_usd, a cumulative usage dict, and model_usage; per-step usage is on AssistantMessage.usage with message_id.
- (ev_003) Parallel tool calls emit multiple assistant messages sharing one id with identical usage; deduplicate by ID to avoid double-counting.
- (ev_004) Each query() call reports only its own cost; the SDK has no session-level total, so accumulate across calls yourself.
- (ev_005) usage includes cache_creation_input_tokens (higher rate) and cache_read_input_tokens (reduced rate); prompt caching is automatic.
- (ev_006) ENABLE_PROMPT_CACHING_1H extends cache-write TTL from the default 5 minutes to 1 hour.
- (ev_007) Both success and error result messages carry usage and total_cost_usd, so failed conversations still report consumed tokens.

## Limitations
- The SDK cost figures are explicitly estimates and may diverge from actual billing on pricing changes, unknown models, or billing rules the client cannot model; authoritative billing requires the Usage and Cost API or Console. Page is undated beyond a 2026-06 access/publication window.
