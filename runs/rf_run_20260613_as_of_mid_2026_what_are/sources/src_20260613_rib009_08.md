# Source Card: Rate Limits and Usage Tiers — Perplexity API Docs

## Summary
This first-party Perplexity API documentation page defines a six-tier usage model (Tier 0 through Tier 5) gated by cumulative lifetime spend, and publishes per-model requests-per-minute (RPM) rate limits that increase with tier. It distinguishes synchronous Sonar models (sonar, sonar-pro, sonar-reasoning-pro) from the heavier sonar-deep-research model and from async submission/polling endpoints, the latter of which have high limits that do not vary by tier.

## Key evidence
- (ev_001) Six tiers gated by cumulative lifetime spend: Tier 0 $0, Tier 1 $50+, Tier 2 $250+, Tier 3 $500+, Tier 4 $1,000+, Tier 5 $5,000+.
- (ev_002) Tier placement is based on cumulative purchases over account lifetime, not current balance.
- (ev_003) sonar-deep-research RPM scales 5 / 10 / 20 / 40 / 60 / 100 from Tier 0 to Tier 5.
- (ev_004) sonar, sonar-pro, and sonar-reasoning-pro share RPM by tier: 50 / 150 / 500 / 1,000 / 4,000 / 4,000.
- (ev_005) Async retrieval endpoints are tier-independent: GET /v1/async/sonar at 3,000 RPM, GET /v1/async/sonar/{request_id} at 6,000 RPM.

## Limitations
- Vendor documentation is subject to change without notice; figures reflect the page as accessed on 2026-06-13 and a stated publication period of 2026-01. Exact per-tier numbers for sonar-deep-research and the shared sonar models were captured from the tier tables but lack a single short verbatim excerpt (quote set to null for ev_003 and ev_004).
