# Source Card: Configuration - GPT Researcher Documentation

## Summary
The official GPT Researcher configuration documentation defines how the framework is customized via `config.py` and environment variables. It establishes a three-tier LLM scheme (FAST_LLM, SMART_LLM, STRATEGIC_LLM) with distinct default OpenAI models, a default integration suite of OpenAI for LLM calls plus Tavily for web retrieval, and pluggable retriever, embedding, and report-format options set through a `.env` file or exported environment variables.

## Key evidence
- (ev_001) FAST_LLM handles summaries and other fast operations, defaulting to openai:gpt-4o-mini.
- (ev_002) SMART_LLM handles report generation and reasoning, defaulting to openai:gpt-5.
- (ev_003) STRATEGIC_LLM handles research planning and strategy, defaulting to openai:gpt-5-mini.
- (ev_004) The default integration suite pairs OpenAI for LLM calls with the Tavily API for real-time web retrieval.
- (ev_005) RETRIEVER defaults to tavily with six alternative options (duckduckgo, bing, google, searchapi, serper, searx).
- (ev_006) EMBEDDING defaults to openai:text-embedding-3-small with ollama, huggingface, azure_openai, and custom options.
- (ev_007) REPORT_FORMAT defaults to APA and supports formats including MLA, CMS, Harvard, and IEEE.
- (ev_008) Defaults are overridden via env variables in a .env file or manual exports in the local project directory.

## Limitations
- Defaults (e.g., gpt-5 / gpt-5-mini) are versioned to the documentation snapshot accessed 2026-06-13 and may shift as the project updates; verify against the live docs for current values.
